# AI-1. あいまい検索強化（1段目・クライアント完結・AI不使用）

親設計: docs/16_AI連携設計.md §1。前提: specs/ui/tools-directory-ia.md の検索仕様（?q=・Enter遷移）。
担当: オーパス次波（辞書整備はソネット可）。外部送信ゼロのまま実装する（本仕様の範囲に fetch は登場しない）。

## 1. 正規化パイプライン（照合前に両辺へ適用）

1. NFKC 正規化（全角英数→半角、半角カナ→全角）
2. カタカナ→ひらがな折りたたみ
3. 長音・中点・スペース・記号の除去（「ほいく-りょう」=「保育料」のかな部分一致を許す）
4. 小文字化

## 2. 同義語辞書

- 新規ファイル: `data/tables/search-synonyms.json`
  ```json
  { "entries": [
    { "canonical": "保育料", "synonyms": ["保育園の料金", "保育園 お金", "ほいくりょう"] },
    { "canonical": "育児時短就業給付", "synonyms": ["時短給付", "時短の給料", "時短手当"] }
  ] }
  ```
- 位置づけは編集資産（出典不要・verify-seido 対象外）だが、brand-lint の走査対象に含める（煽り語の混入防止）。
- 初期整備: 未解決クエリ蓄積（AI-5）が回り始めたら実クエリ上位から追加する運用。初期値は各ツールの solves を種に 50 語程度をソネットが起票。

## 3. スコアリング

- 対象フィールドと重み: title(3) > solves(2) > description(1)。同義語ヒットは canonical ヒットと同点。
- 完全一致 > 前方一致 > 部分一致の順で加点。スコア降順・同点はカテゴリ順。
- 「ヒットが弱い」の定義（2段目のボタン表示条件。docs/16 §1）: 最高スコアが「部分一致1件分」以下、または 0 件。閾値は定数 1 箇所で管理し、AI-3 実装時に同じ値を参照する。

## 4. 完了条件

- トップ・/tools の両検索が同一ロジックを共有（実装は lib/search/ に純関数＋テスト）
- 「ほいくりょう」「じたん きゅうりょう」「オムツ 何歳」級のかな・話し言葉クエリで期待ツールが上位に出るテストを 10 件以上
- JS 無効時は従来どおり全件一覧（劣化なし）
- Lighthouse Perf 90+ 維持（辞書は数 KB 想定。肥大時は検索チャンク分離）
