対象ツール: [02_ツール一覧](../../docs/02_ツール一覧.md) #55（優先度S・難易度S・YMYL高）
制度データ: [`data/seido/fuyou-kabe.json`](../../data/seido/fuyou-kabe.json)（`asOf: 2026-07-17`）

## 1. 概要

「いくらまで働くと損をしないか」を、**2026年（令和8年）分の実際の制度**で示すツール。

kurashi-keisan.jp の既存 `fuyou-kabe` を**移植し、2025〜26年の改正ラッシュに対応して全面強化**する（[02_ツール一覧](../../docs/02_ツール一覧.md) の棲み分け方針: 移植＝計算エンジンを流用し文脈UIで再構築）。

### 差別化の核心

**「103万円の壁」は2026年分にはもう存在しない。** 令和8年度税制改正により所得税の課税最低限は **178万円** になった（[財務省 令和8年度税制改正の大綱の概要](https://www.mof.go.jp/tax_policy/tax_reform/outline/fy2026/08taikou_gaiyou.pdf) 「所得税の課税最低限を178万円まで特例的に先取りして引き上げる」）。

一方で SERP 上位の記事・ツールの多くは 103万円／150万円／201万円という**旧制度の数値**のまま更新されていない。本ツールの価値は「今年の正しい壁を出すこと」そのものにある。

### このツールが答える問い

1. 私の年収だと、いま**どの壁の手前／向こう**にいるのか
2. 次の壁は**いくら**で、超えると**何が起きる**のか
3. 超えたら手取りは**いくら減る**のか（＝崖なのか坂なのか）
4. **2026年10月**に社会保険の壁が動く。私に影響はあるか

## 2. スコープ

### 含むもの

| 領域 | 内容 |
|---|---|
| 税（本人） | 所得税の課税最低限178万円（令和8・9年分） |
| 税（扶養者） | 扶養控除・配偶者控除の所得要件136万円、配偶者特別控除（〜207万円）、特定親族特別控除（〜197万円）、勤労学生控除（163万円） |
| 社会保険（本人） | 106万円の壁（被用者保険の適用拡大・5要件）、130万円の壁（被扶養者認定）、19〜23歳の150万円特例 |
| 時系列 | 2026年10月の賃金要件撤廃による判定の切替 |

### 含まないもの（明示）

| 除外するもの | 理由 |
|---|---|
| **住民税の壁（100万円の壁）の確定値** | 均等割の非課税限度額は級地ごとに異なり市区町村が条例で定める。一次情報で確定できず `fuyou-kabe.json` の `walls.items[0].amount2026` は `null`。→ 「お住まいの市区町村により異なります」と表示し確定値を出さない（[queue/hoikuryo-backlog.md](../../queue/hoikuryo-backlog.md) §5） |
| 給与以外の所得（事業・不動産・雑・一時） | 合計所得金額の計算が複雑化し、本ツールの想定利用者（パート・アルバイト）の射程外。「給与収入のみの方向けです」と明示 |
| 合計所得2,350万円超の基礎控除の逓減 | 射程外（`fuyou-kabe.json` の `bracketsFY2026to2027` 最終行に未収集として記録） |
| 組合健保の独自認定基準 | 保険者ごとに差異がある。協会けんぽ・日本年金機構の基準に基づく旨を免責文で明示 |
| ふるさと納税・住宅ローン控除等の税額控除 | kurashi-keisan へ送客（[02_ツール一覧](../../docs/02_ツール一覧.md) 送客リスト） |
| 手取りの精密計算 | 都道府県別の健康保険料率が未収集（全国平均9.9%で概算）。「概算です」と明示 |

## 3. 入力項目

| # | 項目 | 型 | 必須 | 既定値 | 制約 | UIヒント／なぜ聞くのか |
|---|---|---|---|---|---|---|
| 1 | 年間の給与収入（見込み） | integer(円) | ✅ | — | 0 〜 30,000,000 | 「額面の年収です。手取りではありません」。交通費は税では非課税だが社保の被扶養者判定では収入に含む場合がある旨を注記 |

| 5 | 勤務先の従業員数 | enum | ✅ | `unknown` | `51plus` / `under51` / `unknown` | 106万円の壁（特定適用事業所＝51人以上）の判定。`unknown` の場合は両方の結果を併記 |
| 6 | 週の所定労働時間 | number(時間) | ✅ | — | 0 〜 60 | 20時間以上が106万円の壁の要件。**所定労働時間であり実労働時間ではない** |
| 7 | 雇用期間の見込み | enum | 任意 | `over2months` | `over2months` / `under2months` | 2か月超の見込みが要件 |
| 8 | 扶養者の年収（配偶者控除の判定用） | integer(円) | 条件付き | — | 0 〜 30,000,000 | #3 が `spouse` のとき必須。**所得者の合計所得金額が900万/950万/1,000万で控除額が変わる**。1,000万円超は配偶者特別控除なし |
| 9 | 試算の基準月 | enum | 任意 | 今日の月 | 2026-04 〜 2027-03 | **2026年10月に賃金要件が撤廃される**ため。既定は現在月 |

### 入力設計の判断

- **「所得割額を直接入力させない」**（保育料ツールと同じ思想）。ユーザーは自分の合計所得金額を知らない。年収から逆算する。
- **#5 に `unknown` を用意する理由**: パート勤務者は自社の従業員数を把握していないことが多い。「わからない」を選べないと入力が止まる。両方の結果を併記して「勤務先に確認してください」と促す。

## 4. 計算式

### 4.0 前提: 令和8年分の税額計算

すべての税計算は `fuyou-kabe.json` の値のみを使う。**ハードコード禁止。**

#### Step 1: 給与所得の金額を求める

```
給与収入 X（円）→ 給与所得 S
```

| X の範囲 | S | 根拠 |
|---|---|---|
| X < 691,000 | 0 | `data.kyuyoShotokuKoujo.specialTableFY2026to2027.rows[0]` |
| 691,000 ≦ X < 741,000 | 0 | 同上（「なし」） |
| 741,000 ≦ X < 2,191,000 | X − 740,000 | `rows[1]`（最低保障額74万円） |
| 2,191,000 ≦ X < 2,193,000 | 1,451,000 | `rows[2]`（定額） |
| 2,193,000 ≦ X < 2,196,000 | 1,453,000 | `rows[3]`（定額） |
| 2,196,000 ≦ X < 2,200,000 | 1,456,000 | `rows[4]`（定額） |
| X ≧ 2,200,000 | **未収集** | `data.kyuyoShotokuKoujo.standardTable`。速算表（220万円超）は未収集。[queue §8](../../queue/hoikuryo-backlog.md) |

**根拠**: `data/seido/fuyou-kabe.json` の `data.kyuyoShotokuKoujo.specialTableFY2026to2027`
**出典**: 国税庁「源泉所得税の改正のあらまし（令和8年4月）」

> ⚠️ **220万円超は未収集**。MVP では**年収220万円未満**を正確に扱い、220万円以上は「詳しい試算は kurashi-keisan へ」と送客するか、速算表を収集してから対応する。壁の大半（178万/136万/106万/130万/150万/163万/197万）は220万円未満に収まるため、MVPの価値は毀損しない。**ただし配偶者特別控除の上限207万円だけは220万円未満に収まらないため、207万円の壁を扱うには速算表の収集が必要。**

#### Step 2: 基礎控除を求める

合計所得金額 = S（給与のみの前提）

| S の範囲 | 基礎控除 | 内訳 | 根拠 |
|---|---|---|---|
| S ≦ 4,890,000 | 1,040,000 | 62万 + 加算42万 | `bracketsFY2026to2027.brackets[0]` |
| 4,890,000 < S ≦ 6,550,000 | 670,000 | 62万 + 加算5万 | `brackets[1]` |
| 6,550,000 < S ≦ 23,500,000 | 620,000 | 62万（加算なし） | `brackets[2]` |

**根拠**: `data.kisoKoujo.bracketsFY2026to2027`
**出典**: 国税庁「源泉所得税の改正のあらまし（令和8年4月）」／財務省「令和8年度税制改正の大綱の概要」（「合計所得金額が489万円以下である場合 42万円／489万円を超える場合 5万円」）

> **検算による裏取り**: 給与所得控除74万 + 基礎控除104万 = **178万** = 大綱の「課税最低限を178万円まで」と一致。給与収入136万 − 74万 = 合計所得62万 = 扶養親族の所得要件と一致。給与収入206万 − 74万 = 132万 = 基礎控除表の132万円と一致。**3点の独立した検算が通っている。**

#### Step 3: 所得税の課税所得

```
課税所得 = max(0, S − 基礎控除 − その他の所得控除)
```

MVP では「その他の所得控除」は勤労学生控除のみ扱う。社会保険料控除は**社保に加入した場合の手取り計算でのみ**使う（§4.3）。

### 4.1 税の壁の判定

| 壁 | 判定式 | 効果 | 根拠 |
|---|---|---|---|
| 178万円（本人の所得税） | X ≧ 1,780,000 | 本人に所得税が発生 | `walls.items[1]` |
| 136万円（扶養控除・配偶者控除） | S > 620,000 ⟺ X > 1,360,000 | 扶養者が扶養控除／配偶者控除を失う | `walls.items[2]` |
| 197万円（特定親族特別控除の消滅） | S > 1,230,000 ⟺ X > 1,970,000 | 親の特定親族特別控除がゼロに | `walls.items[3]` |
| 207万円（配偶者特別控除の消滅） | S > 1,330,000 ⟺ X > 2,070,000 | 配偶者特別控除がゼロに | `walls.items[4]` |
| 163万円（勤労学生控除） | S > 890,000 ⟺ X > 1,630,000 | 勤労学生控除（27万円）を失う | `walls.items[5]` |

> **⚠️ 178万 vs 136万 の関係を UI で必ず説明すること。**
> 本人に所得税がかからないのは178万円まで。しかし**136万円を超えた時点で扶養者（親・配偶者）の税負担が増える**。「自分は非課税なのに世帯の手取りが減る」という直感に反する領域が **136万〜178万** に存在する。ここが2026年の最大の説明ポイント。

### 4.2 「壁」ではなく「坂」であることの明示

**特定親族特別控除**（`data.tokuteiShinzokuTokubetsuKoujo`）と**配偶者特別控除**（`data.haiguushaTokubetsuKoujo`）は、控除額が段階的に逓減する。

特定親族特別控除の逓減（給与収入ベース）:

| 給与収入 | 控除額 |
|---|---|
| 136万円超 159万円以下 | 63万円 |
| 159万円超 164万円以下 | 61万円 |
| 164万円超 169万円以下 | 51万円 |
| 169万円超 174万円以下 | 41万円 |
| 174万円超 179万円以下 | 31万円 |
| 179万円超 184万円以下 | 21万円 |
| 184万円超 189万円以下 | 11万円 |
| 189万円超 194万円以下 | 6万円 |
| 194万円超 197万円以下 | 3万円 |

**根拠**: `data.tokuteiShinzokuTokubetsuKoujo.rows`

→ **手取りが逆転する崖は生じない。** UI では「ここは崖ではなく、なだらかな坂です」と明示し、**働き控えの必要がない**ことを伝える。これが本ツールの最も価値ある出力の一つ。

### 4.3 社会保険の壁の判定

#### 106万円の壁（被用者保険の適用拡大）

**2026年9月30日まで**: 以下の**すべて**を満たすと加入

```
週の所定労働時間 ≧ 20時間                       … shortTimeWorker.weeklyHoursThreshold
かつ 所定内賃金の月額 ≧ 88,000円                 … shortTimeWorker.monthlyWageThreshold
かつ 2か月を超えて使用される見込み                 … shortTimeWorker.employmentPeriodRequirement
かつ 学生でない                                  … shortTimeWorker.studentExclusion
かつ 勤務先の被保険者数 ≧ 51人                    … shortTimeWorker.employerSizeThreshold
```

**2026年10月1日以降**: **賃金要件（8.8万円）が撤廃される**（`amendments` の `effectiveFrom: 2026-10-01`）

```
週の所定労働時間 ≧ 20時間
かつ 2か月を超えて使用される見込み
かつ 学生でない
かつ 勤務先の被保険者数 ≧ 51人
```

**根拠**: `data.socialInsurance.shortTimeWorker` ／ `amendments`
**出典**: 厚生年金保険法第12条第5号（e-Gov）／日本年金機構「短時間労働者に対する健康保険・厚生年金保険の適用拡大」／厚生労働省 社会保険適用拡大特設サイト

> ⚠️ **重要な注意を仕様として固定する**:
> 1. 「106万円」は**法令上の数値ではない**。法令は報酬月額8.8万円以上（8.8万×12=105.6万を丸めた通称）。
> 2. 判定は**年収ではなく月額の所定内賃金**。賞与・残業代・通勤手当・精皆勤手当・家族手当は算入しない（最低賃金法第4条第3項各号）。
> 3. 施行期日を定める政令の公布は 2026-07-17 時点で**未確認**。厚労省・年金機構とも「撤廃**予定**」表記。→ UI では「予定」と表記し、断定しない。

#### 130万円の壁（健康保険の被扶養者）

```
年間収入 < 1,300,000                            … dependentCertification.annualIncomeThreshold
  ただし 19歳以上23歳未満（配偶者を除く） → < 1,500,000   … annualIncomeThreshold19to22
  ただし 60歳以上または障害者              → < 1,800,000   … annualIncomeThresholdElderlyOrDisabled
かつ（同一世帯なら）年間収入 < 被保険者の年間収入 × 1/2      … sameHouseholdRule
```

**根拠**: `data.socialInsurance.dependentCertification`
**出典**: 日本年金機構「健康保険の被扶養者になるための条件」／「19歳以上23歳未満の方の被扶養者認定基準の変更について」

> ⚠️ **税と社保の決定的な違いを UI で必ず対比すること**（`incomeDefinition` の note）:
> - **税**: 1〜12月の暦年の**確定額**
> - **社保**: 認定時点からの**将来12か月の見込み額**（月額108,333円 / 日額3,611円が目安）
>
> さらに **2026年4月1日から**、労働条件通知書等の**労働契約内容から見込まれる収入**で判定する取扱いに変わった。労働契約で見込み難い**時間外労働の賃金等（臨時収入）は年間収入に含まれない**。→ 入力UIで「契約上の所定内賃金ベースで」と注記する。

#### 事業主の証明による例外

一時的な収入変動により130万円以上となっても、**事業主の証明により原則連続2回まで**被扶養者認定が可能（`temporaryIncomeException`、「年収の壁・支援強化パッケージ」）。終了期日は設定されていない（2026-07-17時点で継続中）。

→ 130万円を超える結果になったユーザーには、この救済措置を必ず案内する。

### 4.4 手取りの概算

社会保険に加入した場合の本人負担:

| 項目 | 率 | 根拠 |
|---|---|---|
| 厚生年金保険料 | 18.300% ÷ 2 = **9.15%** | `insuranceRates.employeesPensionRate` |
| 健康保険料（全国平均） | 9.9% ÷ 2 = **4.95%** | `insuranceRates.healthInsuranceRateAverage` |
| **子ども・子育て支援金** | 0.23% ÷ 2 = **0.115%** | `insuranceRates.childcareSupportRate` |
| 雇用保険料（一般の事業） | **0.5%** | `insuranceRates.employmentInsuranceRateWorker` |

> ⚠️ **子ども・子育て支援金（0.23%）は令和8年4月分から新設**。2025年以前の記事・ツールには存在しない。**見落とすと社保加入後の手取りを過大評価する。**
>
> ⚠️ これらの率は `verify.skip` になっている（出典ページに率の記載がないため機械照合不能）。**都道府県別の健康保険料率・標準報酬月額表は未収集**（[queue §8](../../queue/hoikuryo-backlog.md)）。→ 手取りは「概算」と明示し、「お住まいの都道府県により異なります」と注記する。

**標準報酬月額**を使うのが正確だが未収集のため、MVP では月額賃金への直接乗算で概算する。**この近似を実装メモに明記し、標準報酬月額表の収集後に差し替える。**

## 5. 境界値

すべて**給与収入（円）**。`fuyou-kabe.json` の値と1対1で対応。

| 境界 | 値 | どちら側に含まれるか | 意味 |
|---|---|---|---|
| 給与所得の発生 | 691,000 | **以上**で表の対象 | 特例表の下限 |
| 給与所得 > 0 | 741,000 | **以上**で S = X − 740,000 | 74万円の最低保障 |
| 扶養控除の喪失 | 1,360,000 | **超**で喪失（1,360,000 ちょうどは**セーフ**） | S = 620,000 ≦ 620,000 |
| 特定親族特別控除の満額 | 1,590,000 | **超**で63万→61万に逓減 | S = 850,000 |
| 勤労学生控除の喪失 | 1,630,000 | **超**で喪失（1,630,000 ちょうどは**セーフ**） | S = 890,000 ≦ 890,000 |
| 所得税の発生 | 1,780,000 | **超**で課税（1,780,000 ちょうどは課税所得0＝**セーフ**） | 74万+104万 |
| 特定親族特別控除の消滅 | 1,970,000 | **超**で0円（1,970,000 ちょうどは3万円） | S = 1,230,000 |
| 配偶者特別控除の消滅 | 2,070,000 | **超**で0円（2,070,000 ちょうどは3万円） | S = 1,330,000 |
| 給与所得控除の特例表の上限 | 2,200,000 | **未満**が特例表の範囲 | 220万以上は未収集 |
| 106万円（月額） | 88,000 **/月** | **以上**で加入 | 年収ではない |
| 130万円 | 1,300,000 | **未満**で被扶養者（1,300,000 ちょうどは**アウト**） | 「130万円未満」 |
| 150万円（19〜23歳） | 1,500,000 | **未満**で被扶養者 | 「150万円未満」 |
| 180万円（60歳以上・障害者） | 1,800,000 | **未満**で被扶養者 | 「180万円未満」 |
| 賃金要件の撤廃 | 2026-10-01 | **以降**で撤廃 | 基準月で分岐 |

### 9.2 端数処理

- 給与所得の特例表（`specialTableFY2026to2027`）は**定額指定**の行がある。`X − 740,000` で計算してはいけない領域（219万1,000円以上220万円未満）がある。**テストケース #17〜19 で固定。**
- 保険料は円未満の端数処理が保険者ごとに異なる（50銭以下切捨等）。概算表示のため MVP では四捨五入で足りるが、「概算」と明記すること。

### 9.3 罠

1. **`fuyou-kabe.json` の `bracketsFY2026to2027.brackets[0]` は「合計所得489万円以下＝104万円」**。原典（国税庁のあらまし）の表は結合セルで、132万円以下／132万円超336万円以下／336万円超489万円以下の**3行が縦結合**されて104万円になっている。単純なPDFテキスト抽出では「132万円以下=104万円」と誤読しやすい。**財務省大綱の「合計所得金額が489万円以下である場合 42万円」が正しい解釈の裏取り**。この経緯は `brackets[0].note` に記録済み。**触る前に必ず読むこと。**
2. **標準報酬月額を使っていない**（未収集）。月額賃金への直接乗算で概算している。標準報酬月額表を収集したら差し替える（[queue §8](../../queue/hoikuryo-backlog.md)）。
