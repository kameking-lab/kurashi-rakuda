## 1. 概要

### 1.1 何を解くツールか

育休から復職して時短で働くとき、**「毎月いくら手元に残るのか」が1画面で分からない**問題を解く。手元の額は次の4つが同時に動くため、単発の給料計算では出ない。

1. **賃金そのものが減る**（時短の比率に応じて）
2. **育児時短就業給付金が乗る**（2025年4月創設・賃金の10%・非課税）
3. **社会保険料はすぐには下がらない**（標準報酬月額は時短の翌月に自動で下がらない＝時間差）
4. **保育料が引かれる**（★前年＝育休前のフル収入の課税額で決まるため復職直後は高め＝もう一つの時間差★）

本ツールは①〜④を1つのタイムラインに載せ、「時短直後の月」と「随時改定後の月」で手元に残る額を並べて示す。

### 1.2 #78（時短勤務の給料・給付シミュレーター）との違い

[02_ツール一覧](../../docs/02_ツール一覧.md) #78（`jitan-kinmu-simu`）は①〜③まで（[specs/s-tools/04](04-jitan-kinmu-simu.md) §2.2 で**保育料を明示的に射程外**とする）。本ツール #56 は**④の保育料を差し引く**ことが差別化点であり存在理由。ペルソナ2の中核・競合不在（docs/02 の #56 備考）。

## 2. スコープ

### 2.1 含むもの

- ①〜③（賃金・社会保険料・育児時短就業給付・随時改定の前後2シナリオ・8/1改定でのデータ期限切れ停止）の計算。**[specs/s-tools/04](04-jitan-kinmu-simu.md) の実装（`lib/tools/impl/jitan-kyuyo.ts`）にそのまま委譲する。**
- ④保育料を差し引いた「1か月に手元へ残る額（税引前）」の提示（2シナリオ）。
- 復職直後に社会保険料・保育料の2つの時間差が重なることの可視化。
- 育休給付・育児時短就業給付が非課税で翌年度の保育料算定基礎に入らない旨（＝翌年度に保育料が下がりうる）の案内。

### 2.2 含まないもの（明示）

| 含まないもの | 理由 |
|---|---|
| **保育料そのものの計算（年収→所得割→階層→保育料月額）** | 自治体ごとの階層表で決まり全国共通式がない。保育料計算ツール（`hoikuryo`）が担当。推計値を手取りの実数に積み上げると YMYL 上の誤差が複利で乗るため、本ツールは**利用者入力の実額**を差し引く |
| 翌年度（9月〜）の保育料の再計算 | 上記に加え切替時期・金額が自治体差。案内のみ |
| 所得税・住民税の厳密計算 | #78 と同じく税引前まで（[04](04-jitan-kinmu-simu.md) §2.2） |
| 標準報酬月額の等級判定／都道府県別健康保険料率 | 未収集（[04](04-jitan-kinmu-simu.md) §2.2 と同一の概算・注記を継承） |
| 育児休業給付金・出産手当金等（復職前の給付） | #4「産休育休まるごとお金シミュレーター」の担当。本ツールは**復職後**が起点 |

## 3. 入力項目

#78（[04](04-jitan-kinmu-simu.md) §3）の全入力をそのまま継承し、**保育料を1項目追加**する。

| # | 項目 | 型 | 必須/任意 | 既定 | 制約 | なぜ聞くのか |
|---|---|---|---|---|---|---|
| （継承） | [04](04-jitan-kinmu-simu.md) §3 の 1〜12（賃金・時短・子の生年月日・開始日・被保険者要件・都道府県 等） | — | — | — | — | ①〜③の算定。jitan エンジンへそのまま渡す |
| 追加 | `hoikuryoMonthly` 復職後の保育料（1か月） | integer (JPY) | 任意 | 40,000 | 0 以上 `MAX_HOIKURYO_INPUT`(200,000) 以下 | ④の差し引き額。制度計算せず利用者が入力した実額を使う。分からない場合は保育料計算ツールへ誘導 |

- `hoikuryoMonthly` は**制度の数値ではなく利用者入力の実額**である。したがってデータ照合（`verify-seido`）の対象外。

## 4. 計算式

**大原則: `data/seido/*.json` に無い数値をハードコードしない。①〜③は jitan エンジンに委譲するため、本ツールが新たに持つ制度定数は無い。**

### 4.1 ①〜③（委譲）

```
jitan = simulate_jitan(input)   # lib/tools/impl/jitan-kyuyo.ts。[04] §4 の算式そのまま
```

- 8/1改定でのデータ期限切れ停止（[04](04-jitan-kinmu-simu.md) §4.1／§7.1）、区分①②③・限度額・最低限度額（[04](04-jitan-kinmu-simu.md) §4.4）、社会保険料の2シナリオ（[04](04-jitan-kinmu-simu.md) §4.6）はすべて jitan 側の責務。ここでは再実装しない。

### 4.2 ④保育料の差し引き

```
hoikuryo = validateHoikuryo(input.hoikuryoMonthly) が空なら floor(input.hoikuryoMonthly)、そうでなければ 0
各シナリオ s について:
    net(s) = (s.monthlyTotal == null) ? null : s.monthlyTotal − hoikuryo
beforeJitanNet = jitan.beforeJitanTakehome − hoikuryo
```

- `s.monthlyTotal`（保育料を引く前の税引前手取り＋非課税給付）が算定できない月（給付が要確認・対象外・昇給）は `net` も null（保育料だけ引いた額は出さない）。
- 保育料は非課税給付の**後**に引く（給付は非課税なので手取りに加算済み、そこから実費の保育料を引く）。

### 4.3 非課税と翌年度保育料（案内のみ）

根拠: `ikukyu-kyufu.json` の `data.handoriJuwariSoutou.taxExempt.note`（「所得税・住民税ともに非課税。翌年の住民税・保育料の算定基礎にも入らない（＝復職後の保育料が下がる要因になる）」）。この note を UI にそのまま表示し、翌年度の金額は自動計算しない。

## 6. テストケース（`tests/fukushoku-tedori.test.ts`・計24件）

- 委譲: 給付・各シナリオ手取り・checks が jitan と完全一致／基本形で給付24,000・2シナリオ／区分②逓減（4.74%・13,509円）が委譲で効く。
- 保育料の差し引き: 時短直後 220,155 −40,000 → **180,155**／随時改定後 228,684 −40,000 → **188,684**／`net = monthlyTotal − hoikuryo`（複数の保育料で）／保育料0で net = monthlyTotal／2シナリオ差は保育料に依らず8,529円／小数入力は切り捨て／`beforeJitanNet = 255,855 − 40,000 = 215,855`。
- 算定不能: 同月育休給付（要確認）・昇給・子2歳以上で net = null。
- バリデーション: 負・上限超過・NaN でエラー、0/妥当額でエラーなし。
- 期限: 期限日は計算、改定日は停止（scenarios 空）／非課税 note がデータ由来で「保育料」を含む。
- ハードコード禁止: `fukushoku-tedori.ts`・`FukushokuTedori.tsx` に制度の数値・改定日の直書きがない。

## 7. エラー・未確認データの扱い

- ①〜③の未確認データ（等級表・都道府県別料率・随時改定の時期・折半の根拠・給付の端数処理・2歳到達日の定義）は [04](04-jitan-kinmu-simu.md) §7.2 の方針をそのまま継承（jitan が注記を生成）。
- 保育料は利用者入力のため未確認データではないが、**推計はしない**（保育料計算ツール・自治体へ誘導）。
- 「わからない」を「対象外」にしない（[04](04-jitan-kinmu-simu.md) §7.3）方針を継承。

## 8. 免責表示文

`ikukyu-kyufu.json` の `disclaimer` を土台に、本ツール固有として以下を追記する。

> **保育料は自治体ごとの階層表で決まり、全国共通の計算式はありません。** 本ツールは保育料を推測せず、あなたが入力した実額を差し引いています。金額は保育料計算ツールやお住まいの自治体でご確認ください。復職直後は、社会保険料が前の給料のままであること・保育料が前年（育休前）の収入で決まることの2つの時間差が重なり、手取りが想定より少なくなりがちです。育児休業給付金・育児時短就業給付金は非課税で翌年度の保育料の算定基礎に入らないため、収入が下がっていれば翌年度（多くの自治体で9月）の見直しで保育料が下がる場合があります。
