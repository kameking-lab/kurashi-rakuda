# 08a. 児童扶養手当 年収→所得換算補助（仕様。docs/13 A-8 / DIAG-A6）

作成: 2026-07-20 司令塔（フェイブル）。実装はオーパス次波。YMYL のため実装者≠検算者のダブルチェック必須（hoikuryo 8a と同運用）。

## 1. 解決する問題（診断実測）

- 現行フォームは課税証明書の「所得金額」直接入力のみ。初見のひとり親は自分の所得金額を知らないことが多く、年収を入れてしまうと過大所得で誤判定になる（診断では収入 160 万円をそのまま所得欄に入れると「一部支給 24,010 円」、正しく換算した所得 77 万円なら「全部支給 48,050 円」と結果が大きく変わることを確認）。
- 「扶養親族の数」の初期値が 0 のため、児童 2 人でもそのままだと限度額が扶養 0 人の行（208 万円）で判定され、実態（扶養 2 人: 284 万円）より厳しい結果が出やすい。

## 2. 判断: 年収からの概算モードを追加する（安全設計つき）

hoikuryo 8a（年収→所得割推計）で確立した「推計値ラベル＋一次データ由来＋断定しない」パターンが再利用でき、データも既存の `kyuyo-shotoku-koujo.json` と `jidou-fuyou-teate.json` で賄えるため、**追加する**。ただし以下の安全設計を必須とする。

## 3. 仕様

### 3.1 入力モード
- 既定タブ「年収から概算（給与のみ）」／もう1つ「所得金額で入力（課税証明書のとおり・正確）」。
- 概算モードの入力: 給与収入（年額・額面）のみ。給与以外の所得（事業・年金等）がある場合は概算モード対象外とし、所得入力モードへ誘導する文言を常時表示（「給与だけの方向けの概算です」）。

### 3.2 換算ロジック（すべて data 由来・ハードコード禁止)
```
給与所得 = 給与収入 − 給与所得控除（該当年分の表）
児童扶養手当の所得 = 給与所得 − 10万円（給与所得がある場合の控除）
                  − 8万円（社会保険料相当の一律控除）
                  ＋ 受取養育費 × 0.8（既存入力を流用）
```
- 10 万円・8 万円・養育費 8 割は `jidou-fuyou-teate.json` に valueNode として追加収録する（現行データに無ければ。出典: こども家庭庁の所得の計算方法ページ・施行令）。
- ひとり親控除・障害者控除等の諸控除は概算に含めない（受給者本人には寡婦・ひとり親控除が適用されない点も含め、caveat で「控除がある方は所得入力モードへ」と明示。捏造しない原則）。

### 3.3 年分の扱い（重要・データ前提条件）
- 手当の所得判定は 11 月切替: 令和 8 年度の 4〜10 月分 = **令和 6 年(2024)分所得**、11 月分以降 = **令和 7 年(2025)分所得**。
- `kyuyo-shotoku-koujo.json` は現在 tableFY2025（令和 7 年分）と tableFY2026 を収録済みだが、**令和 6 年分（最低保障 55 万円の旧税制）が未収録**。
- **前提タスク（実装 PR に含める）**: 令和 6 年分の給与所得控除表を国税庁の一次資料から `kyuyo-shotoku-koujo.json` に追加（verify.expect 付き）。
- ツールは基準日（既存の today 取得）から該当年分を自動選択し、「令和◯年分の所得（令和8年◯月〜◯月分の手当に適用）で概算」をラベル表示。切替月をまたぐ場合は両方の判定を並記してよい（範囲提示は hoikuryo 8a と同流儀）。

### 3.4 扶養親族数の初期値連動（概算・所得入力の両モード共通）
- 「扶養親族の数」の既定値を「対象児童の人数」に**連動**させる（児童数を変えたら追随）。ユーザーが手動で変更したら連動を解除。
- 現行の補足文「多くの場合は対象児童の数と同じです」は維持し、「16 歳未満の児童も限度額表の人数には数えます」を追記。

### 3.5 表示
- 概算モードの結果には常時「概算」バッジ＋「正確な判定は課税証明書の所得金額で。市区町村の窓口で確認できます」を表示（既存の窓口誘導文言を流用）。
- 概算と所得入力で結果が変わりうる主要因（各種控除・給与以外の所得・養育費）を折りたたみで列挙。

### 3.6 テスト（実装者）と検算（別セッション）
- 境界: 全部支給限度ちょうど／±1 円、一部支給限度ちょうど、扶養 0〜3 人、11 月切替の前後、令和 6 年分と 7 年分で結果が変わる収入帯。
- 検算者は本仕様と一次資料のみから期待値を独立作成し、`tests/jidou-fuyou-teate-kensan.test.ts` に恒久化（fuyo-kabe-kensan-r8.test.ts と同流儀）。診断時の実測値（収入 160 万→所得 77 万→全部支給 48,050 円／収入 250 万・児童 2・扶養 2→一部支給 58,160 円）をアンカーに含めること。

## 4. やらないこと

- 給与以外の所得の概算（事業所得・年金の控除体系はスコープ外。誘導のみ）
- 同居扶養義務者の所得の年収換算（本人分のみ。義務者は現行どおり所得入力）
- 市区町村ごとの端数運用差の再現（既存 caveat を維持）
