# レシピ人数スケール換算（P2-T22）

## 概要

レシピの元の人数分（例: 4人分）と、材料名・分量（数値＋単位）の一覧を入力すると、目標の人数分（例: 6人分）にスケールした各材料の分量を計算して表示するツール。

- docs/02 **#43**（カテゴリ3: 家事・料理、需要 中、難易度B、YMYL 低）。BACKLOG.md queueId: `P2-T22`。
- 依存データ: なし。単純な比例計算（目標人数 ÷ 元の人数 の倍率を各材料の分量に掛けるだけ）であり、制度・統計データへの依存はない純粋な算術ツール（BACKLOG.md P2-T22行「依存データ: なし（#38部品流用）」の通り、#38 調味料換算（ChomiryoKanzan）のUIパターン〈複数行の材料入力・追加/削除〉のみを部品として流用し、比重データそのものは使わない）。
- YMYL区分: 低（金額・制度・医療のいずれにも関わらない生活の目安計算）。

## 入力仕様

| フィールド名 | 型 | 必須/任意 | バリデーションルール |
|---|---|---|---|
| 元の人数（fromNinzuu） | 数値 | 必須 | 0より大きい有限数であること。0以下・負数・非数値は入力エラー |
| 目標人数（toNinzuu） | 数値 | 必須 | 0より大きい有限数であること。0以下・負数・非数値は入力エラー |
| 材料一覧（ingredients） | 配列（各行: 材料名・分量・単位） | 必須（1件以上） | 0件（材料未入力）はエラー。各行の材料名は空文字不可。各行の分量は0より大きい有限数であること |

材料一覧は「追加」ボタンで行を増やし、「削除」ボタンで各行を削除できる動的リストとする（UIパターンはChomiryoKanzan.tsxを参考にしつつ、複数行に拡張）。単位（例: g、ml、大さじ、個、本）は自由入力のテキストとし、単位間の相互換算（大さじ→gなど）は行わない。単位はあくまで「入力された分量をどう丸めて表示するか」を決めるためのヒントとしてのみ使う。

## ロジック仕様

### 1. 倍率計算式

```
ratio = toNinzuu / fromNinzuu
scaledAmount(材料) = amount(材料) × ratio
```

各材料の分量に同じ倍率 `ratio` を掛けるだけの単純比例計算であり、材料ごとに異なる倍率は使わない。

### 2. バリデーション（早期リターン。以下の順に検査し、最初に該当したエラーのみを返す）

```
if fromNinzuu が非数値 or 非有限:
    → エラー「元の人数を入力してください。」
elif fromNinzuu <= 0:
    → エラー「元の人数は0より大きい数値で入力してください。」
elif toNinzuu が非数値 or 非有限:
    → エラー「目標の人数を入力してください。」
elif toNinzuu <= 0:
    → エラー「目標の人数は0より大きい数値で入力してください。」
elif ingredients.length === 0:
    → エラー「材料を1つ以上入力してください。」
else:
    ingredients を先頭から順に検査し、最初に見つかった不正行についてエラーを返す
    - 材料名が空文字（trim後）→ エラー「◯行目の材料名を入力してください。」
    - 分量が非数値・非有限・0以下 → エラー「◯行目の分量を正しく入力してください。」
```

### 3. 丸め処理（読みやすい表示のための丸め。値そのものの倍率計算には丸めを適用しない）

単位の文字列を次の3分類のいずれかに判定し、分類ごとに異なる丸め幅を適用する（`classifyUnit`）。

| 分類 | 該当する単位の例 | 丸め幅 | 理由 |
|---|---|---|---|
| 個数系（count） | 個・本・枚・玉・片・束・丁・尾・切れ・房・株・缶・袋・パック | 整数（1刻み） | 実際に切り分けられない・整数個でしか扱えない食材が多いため |
| 計量スプーン・カップ系（spoon） | 大さじ・小さじ・カップ | 0.5刻み | 大さじ・小さじは半量（0.5杯）まではよく使われる読みやすい単位のため、整数だけに丸めると誤差が大きくなる |
| 重量・体積系（weight_volume）・その他未知の単位 | g・ml・kg・L・cc・（上記以外の任意の文字列） | 小数第1位まで | g/ml等は元々細かい数値を扱うため、小数第1位程度の精度を残す。単位が空文字・未知語の場合もこの丸め幅を既定値として使う |

丸めた結果が0になり、かつ元の分量（スケール前）が0より大きい場合（例: 個数系で倍率が非常に小さいとき）は、「0個」という表示を避けるため、丸め後の最小値を1（個数系）または丸め幅の最小刻み（spoon系は0.5、weight_volume系は0.1）に引き上げる。これにより「必要だが表示上は消えてしまう材料」が発生しないようにする。

### 4. 表示する値

各材料について、元の分量・単位・スケール後の丸め済み分量を表示する。倍率（ratio）そのものも「◯倍」として画面に明示し、丸めによって実際の倍率と表示上の分量比が完全には一致しない場合があることをUI上で示す（丸め処理の説明を根拠・計算式セクションに記載）。

## エッジケース

- **元の人数・目標人数が0または負数**: 入力エラーとして扱う（「0人分のレシピ」「マイナス人数」は意味をなさないため）。
- **元の人数と目標人数が同じ**: 倍率1倍となり、各材料の分量はそのまま（丸め処理のみ適用）表示される。異常ではなく正常系として扱う。
- **材料が1件も入力されていない**: 入力エラーとして扱う（計算対象がないため）。
- **材料名が空文字・空白のみ**: 入力エラーとして扱う。
- **分量が0以下・非数値**: 入力エラーとして扱う（「塩 少々」のような数値化できない分量は本ツールの対象外。数値と単位に分解できる分量のみ入力する運用とする）。
- **倍率が非常に大きい・小さい場合**: 計算自体はクラッシュしない（浮動小数点の範囲内で計算するのみ）。表示上は丸め処理により極端に小さい値が0にならないよう最小値に引き上げる（上記「丸め処理」参照）。
- **単位の表記ゆれ（例: "グラム" と "g"）**: 本ツールは単位文字列を厳密一致では判定せず、代表的な表記（後述の分類表）に部分一致で対応するが、想定外の表記は丸め幅のみ既定値（小数第1位）にフォールバックし、計算自体は継続する（クラッシュしない）。

## YMYL配慮事項

- 本ツールは金額・医療・制度のいずれも扱わない、生活の目安（分量の比例計算）専用のツールである（YMYL区分: 低）。
- 調味料・香辛料・加熱時間・加熱方法は人数（分量）に完全に比例するとは限らない（例: 大人数の調理では塩分濃度を人数分そのまま倍にすると味が濃くなりすぎることが知られている）。本ツールは分量の比例計算のみを行い、加熱時間・調理方法の調整は行わない旨を結果画面に常時表示する。
- 本ツールの計算は算術（比例計算）のみであり、公的機関の統計・制度データへの依存はない。出典として一次情報（官公庁データ等）は本質的に存在しないため、registry.json の `sources` には計算の考え方の一般的な参考情報（一般に公開されている実用サイトの解説）を記載するにとどめ、存在しない公的データを出典として捏造しない。
