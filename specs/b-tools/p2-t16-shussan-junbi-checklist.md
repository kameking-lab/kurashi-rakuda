# 出産準備チェックリスト（予定日逆算）（P2-T16）

## 概要

出産予定日を入力すると、今日の日付と出産予定日の関係から「安定期」「臨月」「産後」のどの時期にいるかを判定し、時期ごとの出産準備の段取り・持ち物の一般的なチェックリスト（今すぐ確認したいこと／臨月までに準備したいこと／入院バッグに入れるもの／産後に向けて準備したいこと）を表示するツール。

- docs/02 **#7**（カテゴリ1: 妊娠・出産、優先度 中、難易度B、YMYL 低）。BACKLOG.md queueId: `P2-T16`。
- 依存データ: なし（日付演算＋自前リスト）。妊娠週数計算に用いる「40週0日＝出産予定日」の考え方のみ、`data/tables/san-fujinka-kijun.json`（Q3-01 出産予定日・妊娠週数計算 / 戌の日計算と共有するネーゲレ概算法系の日数定数のSSOT）の `kihon_nissuu_280`（280日＝40週）を再利用する。新規に妊娠計算ロジック（LMPからの逆算等）は作らず、本ツールが行うのは「出産予定日→臨月開始日（出産予定日−28日）→今日との差分日数」の単純な日付演算のみである。
- チェックリストの項目そのもの（入院バッグの中身、産後の手続きの一般的な項目名など）は、特定の一次情報（官公庁の統計・法令）に基づく数値ではなく、一般的に広く知られた出産準備の目安をまとめた自前のリストである。YMYL区分は低いが、「これは一般的な目安であり、公的機関が定めた基準や統計ではない」ことを画面に明示する（YMYL配慮事項を参照）。

## 入力仕様

| フィールド名 | 型 | 必須/任意 | バリデーションルール |
|---|---|---|---|
| 出産予定日（dueDate） | 日付（YYYY-MM-DD） | 必須 | 実在する日付であること（例: 2月30日のような非実在日はエラー）。過去日・未来日いずれも許可する（過去の予定日を入れて「産後」区分の内容を確認したい場合や、まだ余裕のある未来日を試しに入れる場合を妨げないため、上限・下限の範囲チェックは行わない） |
| 基準日（baseDate） | 日付（YYYY-MM-DD） | 内部固定 | UIでは常に実行時の「今日」（ローカル日付）を使う。calc関数自体はテスト可能なように任意の日付を受け取れる |

## ロジック仕様

### 1. 臨月開始日の算出

```
臨月開始日 = 出産予定日 − 28日
```

28日は「40週0日（出産予定日）− 36週0日（臨月＝妊娠10か月の開始）＝4週間＝28日」から導かれる。40週（280日）の値は `data/tables/san-fujinka-kijun.json` の `kihon_nissuu_280.value` をそのまま参照し、ここに数値をハードコードしない。36週という「臨月」の開始週は、産科領域で妊娠10か月（36週0日〜39週6日）を指す一般に定着した慣用区分であり、本ツール独自の判定基準ではない。

### 2. 時期区分（安定期／臨月／産後）の判定

基準日（今日）と、出産予定日・臨月開始日の前後関係で判定する。

```
if 基準日 < 臨月開始日:
    period = "安定期"
elif 基準日 <= 出産予定日:      # 臨月開始日 <= 基準日 <= 出産予定日
    period = "臨月"
else:  # 基準日 > 出産予定日
    period = "産後"
```

臨月開始日当日は「臨月」に含む（境界は臨月側）。出産予定日当日も「臨月」に含み、出産予定日の翌日から「産後」に切り替える（実際の出産日を入力する項目は持たないため、出産予定日を境に一律に切り替える簡易な近似である。エッジケース節を参照）。

### 3. 日数差分

```
出産予定日までの日数（daysUntilDue） = 出産予定日 − 基準日   # 負の値=経過日数（超過）
臨月開始日までの日数（daysUntilRingetsu） = 臨月開始日 − 基準日 # 負の値=既に臨月開始日以降
overdue = 出産予定日 < 基準日
```

### 4. チェックリストの4区分

以下の4つの区分をどの時期でも常に全件表示する（該当しない時期の区分も参考として表示し続け、非表示にはしない）。各区分には、今の時期区分と対応する場合に `isCurrent: true` を付与する。

| 区分ID | ラベル | 対応する時期区分（isCurrent） | 内容 |
|---|---|---|---|
| `stable` | 今すぐ確認したいこと | 安定期 | 出産する産院を決める、母子手帳・妊婦健診の受診票を確認する、名前の候補を考え始める、産休・育休の手続きを勤務先に確認する、上の子がいる場合の預け先を検討する、里帰り出産をするか検討する |
| `beforeRingetsu` | 臨月までに準備したいこと | 安定期 | ベビー用品（肌着・オムツ・ベビーベッド等）を揃える、チャイルドシートを用意する、里帰りする場合は移動手段・時期を決める、上の子の急な預け先・お願いできる人を決めておく、緊急連絡先リスト（病院・家族・タクシー会社）を作る、陣痛時のタクシー会社の事前登録を検討する |
| `hospitalBag` | 入院バッグに入れるもの | 臨月 | 母子健康手帳・健康保険証・診察券、産褥ショーツ・産褥パッド、授乳しやすい服・授乳用ブラジャー、赤ちゃんの退院時の服・おくるみ・肌着、洗面用具・普段使いのタオル、スマートフォンの充電器、現金・軽食、産院から指定された持ち物リストの確認 |
| `postpartum` | 産後に向けて準備したいこと | 産後 | 出生届の提出（出生日から14日以内）、児童手当の申請、赤ちゃんの健康保険加入手続き、出産育児一時金・出産手当金の手続き確認、里帰りからの帰宅・生活再開の準備、上の子がいる場合の赤ちゃん返りへの対応の情報収集 |

`isCurrent` は次の対応で判定する。

```
stable.isCurrent          = (period == "安定期")
beforeRingetsu.isCurrent  = (period == "安定期")
hospitalBag.isCurrent     = (period == "臨月")
postpartum.isCurrent      = (period == "産後")
```

## データ表・出典

チェックリストの項目自体には官公庁の一次データを用いていない（自前の一般的な準備リスト）。日付演算の基礎となる「40週0日＝出産予定日」の考え方のみ、`data/tables/san-fujinka-kijun.json`（Q3-01・Q3-02と共有）を再利用する。同ファイルの出典・確度についての注記は当該JSONの `$comment`・`openIssues` を参照（280日の値そのものは産科診療で広く定着した計算慣行であり、本サイトの出典許可ドメイン内に単独の一次情報ページを2026-07-17時点で確認できていないため、facts[]では機械照合対象外＝stub運用としている）。

「臨月＝妊娠10か月＝妊娠36週0日〜39週6日」という区分自体も、産科領域で広く使われる慣用的な区分であり、本サイトの出典許可ドメイン内に単独の一次情報ページを確認できていない。次回確認: `data/tables/san-fujinka-kijun.json` の `nextCheckDue`（2027-01-01）に合わせて見直す。

## エッジケース

- **出産予定日が未入力**: 入力エラーとして扱う（「出産予定日を入力してください」）。結果は表示しない。
- **出産予定日として実在しない日付（例: 2026-02-30）**: 入力エラー。
- **基準日 = 出産予定日（予定日当日）**: `daysUntilDue = 0`、`period = "臨月"`（出産予定日当日は臨月に含める。産後への切り替えは翌日から）。
- **基準日 = 臨月開始日（境界日）**: `daysUntilRingetsu = 0`、`period = "臨月"`（臨月開始日当日から臨月に切り替わる）。
- **基準日 = 臨月開始日の前日**: `period = "安定期"`（臨月開始日の前日まではまだ安定期）。
- **出産予定日が基準日より過去（例: 1年前）**: エラーにせず `period = "産後"` として計算する。「産後に向けて準備したいこと」を表示しつつ、他の区分（安定期・臨月までに・入院バッグ）も参考として表示し続ける。実際の出産日が予定日どおりとは限らない点を常時のCalloutで明示する。
- **出産予定日が基準日より大きく先（例: 3年後）**: エラーにせず `period = "安定期"` として計算する。妊娠期間として不自然に長い先の日付でもクラッシュしない（実際の妊娠期間の妥当性チェックは行わない。本ツールは「入力された日付から28日前を引く」単純な演算のみを行うため）。
- **実際の出産日と出産予定日がずれる場合**: 本ツールは出産予定日のみを起点にした簡易な近似であり、実際の出産日を入力する項目を持たない。早産・予定日超過のいずれでも、実際の出産後は「産後に向けて準備したいこと」の内容を参考にしつつ、産院・自治体の案内を優先するよう常時のCalloutで案内する。

## YMYL配慮事項

- 結果画面には常時、次のCalloutを表示する: 「このチェックリストは一般的に知られている出産準備の目安をまとめたもので、公的機関が定めた基準や統計調査の結果ではありません。必要なものは体調・地域・産院の方針によって変わるので、母子健康手帳や産院からの案内もあわせて確認してね。」
- 「必ず用意してください」のような断定・指導的な表現は使わず、「〜したいこと」「〜の目安」という柔らかい言い回しに統一する（docs/09 らっくの口調準拠）。
- 医学的な判断（体調評価・受診の要否）は一切行わない。
- 産後区分では、出生届・児童手当など期限のある公的手続きの「一般的な期限の目安」に触れるが、金額や詳細な要件はこのツール内では計算しない（該当する制度解説記事・他ツールへの案内にとどめる）。
