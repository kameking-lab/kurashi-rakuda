# 産後手続きリスト生成（期限つき）（P2-T18）

## 概要

出産日（出生日）と、国内出生か国外出生かを入力すると、産後にやるべき主要な手続き（①出生届、②児童手当の認定請求〈15日特例〉、③出産育児一時金の請求）について、それぞれの法定期限から逆算した「期限日」を計算し、期限が早い順に並べたチェックリストとして表示するツール。

- docs/02 **#11**（産後手続きリスト生成〈出生届〜児童手当、期限つき〉、優先度A、難易度低）。BACKLOG.md queueId `P2-T18`。
- 依存データ: `data/seido/shussho-todoke-kigen.json`・`data/seido/jido-teate.json`・`data/seido/ikukyu-kyufu.json`（すべて既存）。新規データ収集なし。
- カテゴリ: `pregnancy`。
- YMYL区分: **中**。期限を誤ると、国外出生時の国籍留保届の徒過による日本国籍喪失や、児童手当の15日特例を逃したことによる支給月の遡及不可など実害がある。期限の計算根拠を必ず明示し、断定的すぎる表現（「必ず〜」等の不安を煽る言い回し）は避けつつ、期限日そのものは正確に示す。

## 入力仕様

| フィールド名 | 型 | 必須/任意 | バリデーションルール |
|---|---|---|---|
| 出生日（birthDate） | 日付（YYYY-MM-DD） | 必須 | 実在する日付であること。未入力時は結果を表示しない。今日より1年以上先はソフト警告（明らかな入力ミスの可能性）。今日より2年以上前もソフト警告（対象手続の多くは期限を過ぎている可能性が高い旨を明示） |
| 出生した場所（location） | `"domestic"` \| `"abroad"` | 必須（初期値 `domestic`） | 出生届の届出期間（14日 or 3か月）の分岐にのみ使用 |
| 基準日（today） | 日付（YYYY-MM-DD） | クライアント側で自動取得 | 端末の今日の日付。マウント後にクライアント側で取得する（YoukaigoNinteiDandoriNavi.tsx と同じパターン） |

## ロジック仕様

### 0. 起算日ルール（本ツールの核）

日本の期間計算には、原則（民法第140条: 初日不算入＝期間の初日は算入せず翌日から起算する）と、個別法令による例外（初日算入）が存在する。本ツールが扱う3手続はこの2パターンが混在しており、それぞれの根拠を区別して実装する。

| 手続 | 起算日 | 根拠 |
|---|---|---|
| ①出生届 | **出生日当日**（初日算入） | 戸籍法第43条第1項「届出期間は、届出事件発生の日からこれを起算する」（`shussho-todoke-kigen.json` → `data.deadline.startOfPeriod`）。民法第140条の初日不算入の原則の例外。 |
| ②児童手当（15日特例） | **出生日の翌日**（初日不算入＝原則どおり） | こども家庭庁「15日特例」の説明は「出生・転入の**日の翌日**から15日以内」（`jido-teate.json` → `data.payment.startMonth.note`）。特段の初日算入規定がないため民法の原則どおり翌日起算。 |
| ③出産育児一時金の請求期限 | **出産日の翌日**（初日不算入＝原則どおり） | 「出産日の**翌日**から2年以内」（`ikukyu-kyufu.json` → `data.shussanIkujiIchijikin.claimDeadline`）。健康保険法上の請求権の消滅時効であり、特段の初日算入規定がないため民法の原則どおり翌日起算。 |

★出生届だけが初日算入（出生日を1日目として数える）という例外である★という対比を、UIとコンテンツの両方で明示する。

### 1. 日数ベースの期限計算（出生届〈国内〉・児童手当）

起算日（kisanbi）を1日目として N 日以内、を「期限日 = kisanbi + (N − 1) 日」として計算する（addDays はUTC日付演算、`lib/tools/seido.ts` の `addDays` を使用）。

```
① 出生届（国内）:
    kisanbi = birthDate（出生日当日が1日目）
    days = shussho-todoke-kigen.json → data.deadline.days.value（14）
    deadlineDate = addDays(kisanbi, days − 1) = addDays(birthDate, 13)
    例: 出生日が4月1日 → 期限は4月14日（data.deadline.days.note の例示と一致）

② 児童手当（15日特例）:
    kisanbi = addDays(birthDate, 1)（翌日が1日目）
    days = jido-teate.json → data.application.deadline.value（15）
    deadlineDate = addDays(kisanbi, days − 1) = addDays(birthDate, 15)
    例: 出生日が4月1日 → 翌日4月2日が1日目 → 15日目は4月16日
```

### 2. 月・年単位の期限計算（出生届〈国外〉・出産育児一時金）

月または年を単位とする期間は、民法第143条の一般原則（暦に従って計算し、起算日に応当する日の前日に満了する。ただし最後の月に応当する日がないときはその月の末日に満了する）を適用する。この「応当日方式・月末クランプ」は `FukkiBiKeisan.calc.ts` の `addMonths`/`attainDay`（1歳6か月・2歳到達日の計算）と同じ考え方だが、本ツールでは「応当日が存在しない場合はその月の末日そのものが満了日」（前日にしない）という民法第143条第2項ただし書の本文どおりの分岐を明示的に実装する（後述のテストケース参照）。

```
periodEndAfterMonths(kisanbi, months):
    target = kisanbi の月に months を加えた年月
    dim = target 年月の末日の日
    if kisanbi の日 <= dim:
        応当日 = target 年月の（kisanbi と同じ日）
        return 応当日 の前日
    else:
        # 応当日が存在しない（例: 11月30日起算+3か月→2月に30日がない）
        return target 年月の末日そのもの

③ 出生届（国外）:
    kisanbi = birthDate（出生届は初日算入。★出生届はどちらの場合も起算日は出生日当日★）
    months = shussho-todoke-kigen.json → data.deadline.monthsAbroad.value（3）
    deadlineDate = periodEndAfterMonths(birthDate, 3)
    例: 出生日が4月1日 → 応当日は7月1日 → 前日の6月30日が期限
    例: 出生日が11月30日 → 応当日（2月30日）が存在しない → 2月の末日（28日 or 29日）が期限

④ 出産育児一時金の請求:
    kisanbi = addDays(birthDate, 1)（翌日起算）
    years = 2（ikukyu-kyufu.json → data.shussanIkujiIchijikin.claimDeadline.value「出産日の翌日から2年以内」より。
             このノードは数値フィールドではなく説明文のみのため、テストで文字列に「2年」を含むことを機械照合し、
             データが変わった場合に本定数の追随漏れを検知する）
    deadlineDate = periodEndAfterMonths(kisanbi, years × 12)
    例: 出生日が4月1日 → 翌日4月2日起算 → 24か月後の応当日は2年後の4月2日 → 前日の（2年後の）4月1日が期限
```

### 3. 総合結果

3手続を配列にまとめ、`deadlineDate` の昇順（期限が早い順）にソートして返す。国内出生と国外出生とで出生届の期限が大きく変わるため、ソート順は入力によって変わりうる（国内: 出生届14日 → 児童手当15日 → 一時金2年。国外: 児童手当15日 → 出生届3か月 → 一時金2年、が典型）。

各項目について、`daysRemaining = diffDays(today, deadlineDate)`（負の値＝期限超過）と `overdue`（`today > deadlineDate`）を計算する。

### 4. 制度データの鮮度チェック

3つの `SeidoDataset`（`shusshoTodokeDataset`・`jidoTeateDataset`・`ikukyuKyufuDataset`）のいずれかで `isDataExpired(ds, today)` が真の場合、結果を表示せず更新中である旨を表示する（`YoukaigoNinteiDandoriNavi` と同じパターン）。★`ikukyu-kyufu.json` には育休給付金の賃金日額上限額に関する `expiresOn: "2026-07-31"` の改定予告があり、本ツールが使う出産育児一時金の請求期限（健康保険法上の時効）とは無関係な数値だが、同一データセット内に期限切れの改定が存在する場合は安全側に倒して結果表示を止める（他ツールと同じ設計方針）。★

## エッジケース・注意事項

- **未入力**: 出生日が未入力の場合は結果を表示しない（プレースホルダのみ）。
- **不正な日付**: `validateBirthDate` が実在しない日付（例: `2026-02-30`）や形式不正（例: `2026/07/17`）を弾く。
- **4/1生まれ（出生届の起算日確認）**: 出生日が4月1日の場合、出生届（国内）の期限は4月14日（`shussho-todoke-kigen.json` の note に明記された例示と一致することをテストで確認）。
- **国外出生・月末クランプ**: 出生日が11月30日の場合、出生届（国外）の期限は応当日（2月30日）が存在しないため、その年の2月の末日（うるう年なら29日・平年なら28日）となる。
- **うるう日生まれ（2/29生まれ）**: 出産育児一時金の請求期限（翌日起算・2年後の応当日の前日）で、翌日（3月1日）起算から24か月後の応当日（3月1日）の前日（2月末日）を正しく計算できることを確認する。
- **年末生まれ（12/31生まれ）**: 児童手当の15日特例の期限が年をまたぐ（翌年1月）ことを確認する。
- **国内⇄国外の切り替えでソート順が変わる**: 出生届の期限が15日より早い（国内）か遅い（国外）かで、チェックリストの並び順（期限昇順）が変わることをテストで確認する。
- **極端な未来日／過去日**: 出生日が今日から1年以上先、または2年以上前の場合はソフト警告を表示する（計算は継続する）。
- **制度データの期限切れ**: 3データセットのいずれかで `isDataExpired` が真を返す場合は結果を表示せず、更新中である旨を表示する。
- **健康保険の加入手続き・出生後休業支援給付金等、法定の届出期限日数が一次資料から明確に読み取れない手続には踏み込まない**: 依存データ3件から機械的に導出できる期限のみを扱い、期限を推測・創作しない（`ikukyu-kyufu.json` にある育児休業給付金等の金額データはこのツールでは使用しない）。

## YMYL配慮事項

- 国外出生の場合、出生届とともに国籍留保届をしないと日本国籍を失う場合がある旨を、法務省の警告文言（`nationalityReservationAbroad`）に基づき必ず明示する。
- 出生届は期限を過ぎても市町村長が必ず受理する（過料のリスクはあるが、届出自体を諦める必要はない）旨を必ず明示し、「遅れたら受け付けてもらえない」という誤解を防ぐ。
- 児童手当の15日特例を逃すと、原則どおり「申請した月の翌月分から」の支給になり、出生月の翌月分からの支給を受けられなくなる（遡及されない）旨を明示する。
- 出産育児一時金の請求期限（2年）は健康保険法上の消滅時効であり、直接支払制度を利用した場合は多くのケースで自己請求が不要である旨もあわせて明示し、不要な不安を煽らない。
- 期限日はすべて「〜が目安です」ではなく、法令の条文・数値に基づき断定的に計算するが、「正当な理由があれば過料が科されない」「延着・窓口の休業日等の実務上の変動要因がある」等、機械計算にとどまらない実務上の留保がある場合は明示する。
- 出典3点セット（戸籍法・こども家庭庁・厚生労働省）と最終更新日を画面下部に表示する（`SeidoNotice`）。
