# 陣痛間隔カウンター（オフライン必須）

## 概要

陣痛（子宮収縮）の「始まった」「終わった」を都度タップして記録し、記録から自動的に

- **間隔**（前回の陣痛の開始〜今回の陣痛の開始までの時間）
- **持続時間**（陣痛の開始〜終了までの時間）

を計算して一覧表示するツール。BACKLOG.md **P2-T17**（docs/02 #10、担当 B(ソネット)、依存データなし、仕様レベル「簡易」）。

- 対象ペルソナ: 出産が近い妊婦本人・付き添い（パートナー等）。陣痛が始まってから病院に連絡・入院するまでの間、繰り返し使うことを想定する。
- **YMYL区分: 中**（数値目安を出典つきで案内するが、本ツールは医学的判断・診断を行わない。詳細は下記「YMYL配慮事項」）。
- **オフライン必須**: すべての記録・計算はブラウザ内（クライアントサイド）で完結する。ネットワーク通信（fetch・XHR等）は一切発生しない。記録は `localStorage` に永続化し、タブの再読み込みやブラウザ再起動後も記録が消えないようにする。

## 入力仕様

このツールはフォーム入力ではなく「ボタン操作によるイベント記録」が主入力になる。

| 操作 | 説明 | バリデーション |
|---|---|---|
| 「陣痛が始まった」ボタン | 現在時刻（`Date.now()`）で新しい記録（`endedAt: null`）を追加する | 既に「進行中」の記録（`endedAt === null`）がある間は無効化（誤操作防止）。連打しても2重に開始記録が作られない |
| 「陣痛が終わった」ボタン | 進行中の記録の `endedAt` を現在時刻で確定する | 進行中の記録がない間は無効化 |
| リセットボタン | 全記録を削除する | 誤操作防止のため、押下後に「本当に削除しますか？」の確認ステップ（確認／キャンセルの2ボタン）を挟み、確認を押すまでは削除しない |

内部的に保持する記録の型:

```ts
interface ContractionRecord {
  id: string;
  startedAt: number; // epoch ミリ秒
  endedAt: number | null; // 進行中は null
}
```

## ロジック仕様

すべて `components/tools/impl/JintsuuKankakuCounter.calc.ts` の純関数として実装する（UIから独立、`fetch`/`window`のグローバル副作用なし）。

### 1. 持続時間（`calcDuration`）

```
durationSec(record):
    if record.endedAt is null: return null   # 進行中
    diffMs = record.endedAt - record.startedAt
    if diffMs < 0: return null                # 時計のズレ等による不正値
    return round(diffMs / 1000)
```

### 2. 間隔（`calcInterval`）

「間隔」は前回の陣痛の**開始時刻**から今回の陣痛の**開始時刻**までの経過時間と定義する（多くの産科案内で使われる定義に合わせる）。

```
intervalSec(current, previous):
    if previous is null: return null          # 先頭の記録には間隔がない
    diffMs = current.startedAt - previous.startedAt
    if diffMs < 0: return null                # 記録順序の異常（時計のズレ等）
    return round(diffMs / 1000)
```

記録配列は開始時刻の昇順（`sortByStartedAt`）に並べ替えたうえで、隣接するペアに対して上記を適用する（`buildRows`）。UI表示（新しい順）への並べ替えは表示層の責務とし、ロジック層は常に昇順で扱う。

### 3. 平均（`calcAverages`）

直近 `n`（デフォルト5）件の**有効な**間隔・持続時間のみを対象に単純平均（四捨五入）する。`null`（進行中・不正値・先頭記録）はサンプルから除外し、サンプル数が0件なら平均も `null` を返す。

```
avgIntervalSec  = round(mean(有効な間隔[-n:]))  # 有効サンプルが0件なら null
avgDurationSec  = round(mean(有効な持続時間[-n:]))
```

### 4. localStorage キー設計

| キー | 値 | 備考 |
|---|---|---|
| `rakuda:jintsuu-kankaku-counter:v1` | `ContractionRecord[]` を `JSON.stringify` した文字列 | バージョン番号 `v1` を含め、将来スキーマ変更時は `v2` 等に切り替えて旧データとの衝突を避ける |

- 読み込み: マウント後（`useEffect`）に `localStorage.getItem(STORAGE_KEY)` → `parseRecords()` でデコード。壊れたJSON・型不正な要素は例外を投げずに除外し、残りの正常な記録は保持する（記録全体を失わないため）。
- 書き込み: 記録配列が変化するたびに `serializeRecords()` → `localStorage.setItem(STORAGE_KEY, ...)`。
- SSR対策: サーバー側で `localStorage` は存在しないため、初期状態は空配列とし、クライアントマウント後にのみ読み込む（`Getsurei.tsx` の「マウント後に今日の日付を取得する」パターンに倣う）。

## 出典

「何分間隔になったら病院に連絡すべきか」の目安は、`factory/config/source-allowlist.json` の許可ドメインのうち、`ncchd.go.jp`（国立成育医療研究センター）が自院の産科ページで公開している具体的な連絡目安を出典とする。

- 出典: 国立成育医療研究センター 産科「入院について」（https://www.ncchd.go.jp/hospital/pregnancy/bunben/nyuin.html）
  - 引用: 「初産婦：10分おきの陣痛になったとき」「経産婦：10分〜15分おきの陣痛になったとき」「破水したとき（赤ちゃんを包む膜が破れて羊水が流れます）」＝間隔に関わらず連絡対象

この目安は同センターという一施設の入院連絡基準であり、産婦人科学会が定めた全国一律の診療ガイドライン値ではない。そのため画面上では「一般的な目安の一例」として提示し、実際の連絡タイミングは必ず受診先の産院・病院の指示に従うよう明記する（下記YMYL配慮事項）。

## エッジケース

| # | ケース | 挙動 |
|---|---|---|
| 1 | 記録が0件 | 一覧は空。平均は「記録がまだありません」等の文言で `null` 扱い表示 |
| 2 | 記録が1件のみ（進行中） | 持続時間・間隔とも表示なし（進行中である旨のみ表示） |
| 3 | 記録が1件のみ（終了済み） | 持続時間は計算できるが、間隔は「（比較対象なし）」表示 |
| 4 | 日をまたぐ記録（23:58開始→翌0:02終了） | epochミリ秒の差分で計算するため日付境界の特別処理は不要。持続時間4分として正しく計算される |
| 5 | 端末の時計が記録の途中でずれ、`endedAt < startedAt` になる | 持続時間は `null`（不正値）として扱い、一覧では「—」表示にとどめる（マイナス表示はしない） |
| 6 | 記録の開始時刻が前回の記録より前になる（時計のズレ等） | 間隔は `null` として扱う |
| 7 | 「陣痛が始まった」を連打 | 進行中の記録がある間はボタンを無効化し、2件目の開始記録は作られない |
| 8 | 「陣痛が終わった」を進行中の記録がない状態で呼ぶ | ボタンを無効化しているため通常発生しないが、ロジック層でも無視（no-op）する |
| 9 | localStorage の内容が壊れている（手動改変・別バージョンの形式混入等） | 解析できない要素は無視し、正常な要素だけを復元する。全体が壊れている場合は空配列から再開する |
| 10 | リセットボタン押下 | 確認ステップを経てから全記録を削除する（誤操作防止） |
| 11 | 記録数が平均のサンプル数（5件）を超える | 直近5件のみを平均対象とする（それより古い記録は一覧には残るが平均には使わない） |

## YMYL配慮事項

- 本ツールは陣痛の記録・間隔計算という**計測機能のみ**を提供し、陣痛の強さ・分娩の進行段階・出産の緊急度についての**医学的判断や診断は一切行わない**。
- 「何分間隔になったら病院に連絡すべきか」の数値目安は、上記出典（国立成育医療研究センター）から引用した**一般的な目安の一例**として `Callout` で案内する。固定文言案:
  - 「一般的には、初産婦は陣痛が10分間隔に、経産婦は10〜15分間隔になったら病院・産院に連絡するとされています（国立成育医療研究センター 産科の案内より）。破水した場合は、間隔に関わらずすぐに連絡してください。ただし連絡のタイミングは必ずご自身の受診先の指示を優先してください。」
- 上記目安が「絶対的な基準ではなく、一施設の運用例であること」「かかりつけの産院の指示が優先されること」を必ず併記する。
- 記録データはすべて端末内（localStorage）にのみ保存され、外部送信は一切行わない旨を明記する（プライバシー・オフライン動作の両面）。

## テストケース

| # | 入力 | 期待される出力 |
|---|---|---|
| 1 | 記録なし `[]` | `buildRows([])` = `[]`、`calcAverages([])` の `avgIntervalSec`/`avgDurationSec` は `null`、サンプル数0 |
| 2 | 1件のみ、進行中（`endedAt: null`） | `durationSec` = `null`、`intervalSec` = `null` |
| 3 | 1件のみ、終了済み（開始10:00:00、終了10:00:45） | `durationSec` = 45、`intervalSec` = `null` |
| 4 | 2件（開始10:00:00→終了10:00:40、開始10:07:00→終了10:07:50） | 2件目の `intervalSec` = 420（7分）、`durationSec` = 50 |
| 5 | 開始が前回より前（記録の入力順が前後） | 該当行の `intervalSec` = `null` |
| 6 | `endedAt` が `startedAt` より前（時計のズレ） | `durationSec` = `null` |
| 7 | 日またぎ（開始 23:58:00→終了 翌0:02:00） | `durationSec` = 240（4分） |
| 8 | 進行中の記録がある状態で `startContraction` を呼ぶ | 記録が増えない（no-op） |
| 9 | 進行中の記録がない状態で `endContraction` を呼ぶ | 記録が変化しない（no-op） |
| 10 | `resetRecords()` | `[]` を返す |
| 11 | 7件の完了記録がある状態で `calcAverages(records, 5)` | 直近5件の間隔・持続時間のみで平均を計算する（`intervalSampleSize`/`durationSampleSize` = 5） |
| 12 | `parseRecords("{不正なJSON")` | `[]`（例外を投げない） |

| 15 | `formatSecondsLabel` に 0秒・59秒・60秒・125秒・`null` を渡す | それぞれ `"0秒"`・`"59秒"`・`"1分0秒"`・`"2分5秒"`・`"—"` |
