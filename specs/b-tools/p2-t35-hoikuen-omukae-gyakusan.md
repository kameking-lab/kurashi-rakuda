# 保育園お迎え逆算 勤務可能時間計算（P2-T35）

## 概要

保育園のお迎え締切時刻・職場⇔保育園の移動時間・退勤後の準備時間（バッファ）を入力すると、「何時までに退勤しなければならないか（退勤限界時刻）」を逆算するツール。出勤時刻（勤務開始時刻）もあわせて入力すると、その日の実労働可能時間（時間・分）も算出する。

- docs/02 **#82**（カテゴリ6: 仕事・キャリア、優先度 中、難易度B）。BACKLOG.md queueId: `P2-T35`。
- 依存データ: なし。制度・統計データに一切依存しない、純粋な時刻の加減算のみで完結するツール（BACKLOG.md P2-T35「依存データ」欄の指示どおり）。
- YMYL区分: 低。医療・お金・法的な判断は一切含まない、暦・時刻の演算のみ。ただし「勤務時間」に関わるため、断定的な指導（残業をしないよう指示する等）は行わず、あくまで入力値から機械的に導いた目安であることを明示する。
- 日をまたぐ計算（夜勤・深夜帰宅など）は対象外。入力・計算はすべて同一日内の時刻（0:00〜23:59）として扱う。

## 入力仕様

| フィールド名 | 型 | 必須/任意 | バリデーションルール |
|---|---|---|---|
| お迎え締切時刻（omukaeShimekiri） | 時刻文字列 `HH:mm` | 必須 | `00:00`〜`23:59` の形式であること。時2桁（00〜23）・分2桁（00〜59）。不正な形式（`25:00`、`9:60`、空文字、数値でない文字列等）は入力エラー |
| 職場から保育園までの移動時間（shokubaToHoikuenMin） | 分（整数、0以上） | 必須 | 0以上の数値。負数・非数値（NaN）は入力エラー。小数は四捨五入して分単位で扱う |
| 保育園から職場までの移動時間（hoikuenToShokubaMin） | 分（整数、0以上） | 必須 | 同上。退勤限界時刻の算出そのものには使わないが、出勤時刻が入力された場合の「朝、保育園を出発する目安時刻」の参考表示に使う |
| 退勤後の準備時間・バッファ（bufferMin） | 分（整数、0以上） | 必須 | 0以上の数値。片付け・退勤打刻・職場を出るまでの余裕時間。0を許容する |
| 出勤時刻・勤務開始時刻（shukkinJikoku） | 時刻文字列 `HH:mm` | 任意 | 未入力可（空文字）。入力する場合はお迎え締切時刻と同じ形式検証を行う |

## ロジック仕様（時刻の逆算式・分単位での計算）

すべての時刻は `HH:mm` 文字列を「0時0分からの経過分数（0〜1439の整数）」に変換してから計算する（`parseHHMM`）。日をまたぐ計算は行わないため、計算結果が0未満または1440以上になる場合はエラーとして扱う。

### 1. 退勤限界時刻の算出

```
omukaeMin = parseHHMM(omukaeShimekiri)
taikinGenkaiMin = omukaeMin - shokubaToHoikuenMin - bufferMin

if taikinGenkaiMin < 0:
    → 入力エラー（「お迎え締切時刻に対して移動時間・準備時間が長すぎます。日をまたぐ計算はできません。」）
else:
    taikinGenkaiLabel = formatHHMM(taikinGenkaiMin)  # "HH:mm" 表示
```

「退勤限界時刻」は、お迎え締切時刻から「職場→保育園の移動時間」と「退勤後の準備時間（バッファ）」を差し引いた時刻で、これより後に退勤するとお迎え締切に間に合わない可能性がある、という意味の目安である。

### 2. 実労働可能時間の算出（出勤時刻が入力された場合のみ）

```
if shukkinJikoku が入力されている:
    shukkinMin = parseHHMM(shukkinJikoku)
    workableMin = taikinGenkaiMin - shukkinMin
    if workableMin < 0:
        → 入力エラー（「退勤限界時刻が出勤時刻より前になっています。勤務時間がマイナスになるため、入力内容をご確認ください。」）
    else:
        workableLabel = "◯時間◯分"（workableMin を60で割った商と余り）
```

### 3. 朝の保育園出発目安（参考値。出勤時刻が入力された場合のみ）

```
if shukkinJikoku が入力されている:
    asaShuppatsuMin = shukkinMin - hoikuenToShokubaMin
    possible = asaShuppatsuMin >= 0
    asaShuppatsuLabel = possible ? formatHHMM(asaShuppatsuMin) : 算出不可（前日側にはみ出すため）
```

出勤時刻に間に合うよう、保育園から出発すべき目安時刻を参考情報として表示する。この値がマイナスになる場合（＝出勤時刻までに保育園から職場への移動が間に合わない）は、退勤限界時刻・実労働可能時間の計算自体は成立しているため、この項目のみ「算出できません」と表示しエラーにはしない（メインの2出力をブロックしない設計）。

### 4. 時刻・分数フォーマット関数

```
parseHHMM(s): "HH:mm" 形式（時00〜23・分00〜59の2桁ゼロ埋め）に一致しなければ null を返す。一致すれば 時*60+分 を返す。
formatHHMM(min): 0〜1439の整数を "HH:mm" のゼロ埋め文字列に変換する。
formatDuration(min): 0以上の整数を "◯時間◯分" 形式に変換する（0時間なら "◯分" のみ、0分なら "◯時間" のみ）。
```

## エッジケース

- **移動時間が0分**: 職場が保育園に隣接している等の場合。バリデーションはそのまま通過し、退勤限界時刻＝お迎え締切時刻−バッファとなる。
- **バッファが0分**: 準備時間なしとして扱い、退勤限界時刻＝お迎え締切時刻−移動時間となる。
- **移動時間・バッファがすべて0**: 退勤限界時刻＝お迎え締切時刻そのもの。
- **退勤限界時刻がちょうど0:00（境界値）**: エラーにはしない（0以上なので正常値として扱う）。
- **退勤限界時刻が負（お迎え締切時刻より移動時間＋バッファが大きい）**: 入力エラー。日をまたぐ計算はサポートしない前提のため、「日をまたぐ計算はできません」という趣旨のメッセージを表示する。
- **出勤時刻が退勤限界時刻より後（勤務時間が負になる）**: 入力エラー。「退勤時刻が出勤時刻より前になっています」という趣旨を明示する（BACKLOG.md P2-T35の指示に基づく必須のエラーケース）。
- **不正な時刻文字列**（`25:00`、`9:5`、`abc`、空文字、`24:00`等）: いずれも入力エラー。`24:00`は日付が変わる境界のため、本ツールの時刻範囲（00:00〜23:59）としては無効値扱いにする。
- **出勤時刻を入力しない場合**: 退勤限界時刻のみを表示し、実労働可能時間・朝の保育園出発目安は表示しない（エラーにはしない。この2項目は出勤時刻があって初めて意味を持つ付加情報のため）。
- **朝の保育園出発目安がマイナスになる場合**: メインの計算結果（退勤限界時刻・実労働可能時間）は表示しつつ、この項目だけ「算出できません」と表示する（部分的な情報不足でツール全体をエラーにしない）。

## YMYL配慮事項

- 本ツールはYMYL低区分（医療・お金・法律に関わる判断を含まない、純粋な時刻演算）。
- 「退勤限界時刻」「実労働可能時間」はいずれも入力値から機械的に導いた目安であり、実際の勤務時間・残業の可否は就業規則・上長の指示に従う旨を常時表示する。
- 残業が発生しそうな場合や、お迎えに間に合わない見込みの場合は、延長保育の利用可否を含めて事前に職場・保育園へ相談・確認するよう促す注記を表示する（断定的な指導や、残業を避けるべきという価値判断は行わない）。
- 出典・準拠年度は「なし（時刻演算のみ）」であることを正直に表示する。本仕様書自体をロジックの参照先として registry.json の sources に記載する（制度データではなく計算ロジックの説明であることを明記）。
