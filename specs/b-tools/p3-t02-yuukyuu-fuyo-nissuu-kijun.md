# 有給残・取得計画（P3-T02）

## 概要

雇い入れ日・勤務形態（フルタイム相当／週の所定労働日数が少ないパートタイム等）・直近の出勤率から、労働基準法第39条が定める年次有給休暇の発生要件・付与日数・付与予定日（基準日）を計算し、雇い入れ日を起点とした「取得計画」（6か月・1年6か月・2年6か月…6年6か月以上の7区分すべての基準日と付与日数の一覧）を示すツール。

- docs/02_ツール一覧.md **#84**（カテゴリ「有給残・取得計画」、優先度 中、難易度B、YMYL 低）。BACKLOG.md queueId `P3-T02`。
- 依存データ: `data/seido/yuukyuu-fuyo-nissuu-kijun.json`（労働基準法第39条・労働基準法施行規則第24条の3。既存・✔）。
- YMYL区分: **低〜中**。付与日数・時季指定義務は法定の最低基準であり、就業規則がこれを上回ることがある旨、また出勤実績（利用実績）までは追跡しないことを明示する。

★本ツールの前提★: 本ツールは「制度上、いつ・何日付与されるか」という取得計画（スケジュール）を示すものであり、実際に何日取得済み・何日残っているかという利用実績は入力を受け取らないため出力しない（架空の「残日数」を捏造しない）。

## 入力仕様

| フィールド名 | 型 | 必須/任意 | バリデーションルール |
|---|---|---|---|
| 雇い入れ日（hireDate） | 日付（YYYY-MM-DD） | 必須 | 1950-01-01〜2100-12-31の範囲の実在の暦日であること。範囲外・不正な形式は入力エラー |
| 勤務形態（employmentType） | 選択式（5択） | 必須 | `standard`（フルタイム相当。週所定労働時間30時間以上、または週所定労働日数5日以上）／`days4`／`days3`／`days2`／`days1`（週4〜1日勤務。いずれも週所定労働時間30時間未満が前提） |
| 直近の基準期間の出勤率（attendanceRateOk） | 選択式（2択・自己申告） | 必須（初期値: 8割以上） | 「8割以上出勤した」／「8割未満だった」の二択。全労働日に対する出勤率で判定する（労働基準法第39条第1項・第2項ただし書） |

「算定基準日時点で継続勤務した年数」は利用者に直接入力させず、雇い入れ日と今日の日付（システム時刻）から自動的に算出する。これにより、区分の当てはめ間違い（例: 満年数と経過月数の混同）を防ぐ。

## ロジック仕様

### 1. 基準日スケジュールの算出（雇い入れ日からの日付演算）

労働基準法第39条第2項・同法施行規則第24条の3が定める継続勤務年数の7区分（6か月・1年6か月・2年6か月・3年6か月・4年6か月・5年6か月・6年6か月以上）は、いずれも「雇い入れの日から起算した継続勤務期間」で定義される。`data/seido/yuukyuu-fuyo-nissuu-kijun.json` の `data.standardGrantTable.rows[].serviceMonths`（6, 18, 30, 42, 54, 66, 78）は、この7区分を「雇い入れ日からの経過月数」として構造化済みの値である。

```
basisDate[i] = addMonths(hireDate, serviceMonths[i])   （i = 0..6）
```

`addMonths` は応当日方式・月末クランプ（例: 8/31の6か月後は2月末日）で計算する暦月演算であり、うるう年の2/29は非うるう年では2/28にクランプする。

6年6か月（`i=6`）以降は、以後12か月ごとに同じ区分（付与日数・年5日義務の対象可否とも据え置き）が繰り返される。本ツールはこれを7個より先まで基準日を無限に列挙するのではなく、「6年6か月の基準日から12か月刻みで何回分経過したか」（`extraYearsBeyondCap`）を数える方式で表現する。

### 2. 発生要件の判定

発生要件は次の2つ（第39条第1項）。

1. 雇い入れの日から起算して継続勤務6か月（`data.eligibility.continuousServiceMonths.value`）
2. その期間の全労働日の8割以上出勤（`data.eligibility.attendanceRate.value` = 0.8）

要件①は `today >= basisDate[0]` で判定する（`today < basisDate[0]` なら「まだ発生していない」とし、`daysUntilFirstEligible` として残り日数を示す）。要件②は、今回は自己申告の二択入力（`attendanceRateOk`）で判定し、`false` の場合は「直近の基準日での新たな付与はない」ことを示す（`currentGrantWithheld = true`）。★継続勤務年数はリセットされない★（`data.eligibility.attendanceRate.note`）ため、次の基準期間に8割以上出勤すれば、その時点の継続勤務年数に応じた日数が通常どおり付与される旨を明示する。

### 3. 付与日数の決定（勤務形態による分岐）

- `employmentType = "standard"`（フルタイム相当。週所定労働時間30時間以上、または週所定労働日数5日以上）→ `data.standardGrantTable.rows[i].grantDays` をそのまま使う（10/11/12/14/16/18/20日）。
- `employmentType` が `days4`／`days3`／`days2`／`days1`（週所定労働時間30時間未満の労働者）→ `data.proportionalGrant.table.rows` のうち `weeklyWorkDays` が一致する行の `grantDays[i]`（7区分の配列）を使う。

### 4. 年5日の時季指定義務の対象判定

- 標準（フルタイム）は7区分すべてで `subjectToMandatory5Days: true`（6か月時点で既に10日のため）。
- 比例付与は行ごとの `subjectToMandatory5DaysFrom`（例: 週4日は「3年6か月」から、週2日・週1日は `null` ＝対象外）を `serviceLabels` 配列内の位置に変換し、`区分のindex >= その位置` で判定する。

### 5. 直近・次の基準日の提示（取得計画）

`today` から見て「直近に到達した基準日」（`current`）と「次に到来する基準日」（`next`）を求め、それぞれの付与日数・年5日義務の対象可否を表示する。加えて、7区分すべての基準日・付与日数・年5日義務の対象可否を一覧表（取得計画）として常に表示する。

## エッジケース・注意事項

- **6か月経過前（発生要件①未達）**: エラーにはせず、「あと◯日で発生要件を満たす」旨と、最初の付与予定日・日数を表示する。
- **基準日ちょうど（境界値）**: `today == basisDate[i]` はその区分に「到達済み」として扱う（`isReached = true`）。
- **出勤率8割未満**: 到達済みの区分でも新たな付与は行わない（`currentGrantWithheld = true`）が、継続勤務年数はリセットされない旨を明示する。
- **6年6か月を超えて長期間勤務**: 付与日数（標準20日、比例付与は各表の最終列の値）は据え置かれたまま、12か月ごとに同じ区分の付与が繰り返される（`extraYearsBeyondCap` で管理し、基準日の一覧を無限に増やさない）。
- **雇い入れ日が今日より未来**: エラーにはせず、入社前のプレビュー（発生要件未達・基準日はすべて未来）として計画を表示する。
- **雇い入れ日の範囲外（1950年より前・2100年より後）**: 入力エラーとして結果を表示しない。
- **うるう年をまたぐ月演算**: `addMonths` は月末クランプ方式（例: 2024-02-29の12か月後は非うるう年の2025-02-28）で計算し、特別な補正コードは持たない（クランプで自然に処理される）。
- **週2日・週1日勤務**: 6年6か月以上でも付与日数が10日に届かないため、年5日の時季指定義務の対象には一度もならない。

## ★データの限界★

- 出勤率は自己申告の二択入力であり、実際の出勤簿・欠勤日数から自動計算していない。年の途中で入社した場合の按分や、育児休業・産前産後休業等の「出勤したものとみなす」期間（`data.eligibility.deemedAttendanceDays`）の反映は利用者自身の申告に委ねている。
- 週所定労働日数が曜日ベースではなくシフト制・年間所定労働日数（`data.proportionalGrant.annualWorkDaysThreshold` = 216日）で定められている労働者向けの按分は、本ツールでは扱っていない（週所定労働日数4〜1日の選択のみに対応。年間所定労働日数ベースの判定が必要な方は、施行規則第24条の3第3項の表を直接ご確認いただく必要がある）。
- 実際に何日取得済み・何日残っているか（利用実績）は入力を受け付けていないため出力しない。繰越し（時効2年）を踏まえた正確な残日数の管理は、勤務先の給与明細・就業規則の記載を確認する必要がある。
- 就業規則や労働協約により、本ツールが示す法定の最低基準を上回る日数・条件を定めている会社もある（法定を下回る定めは労働基準法第13条により無効）。実際の付与日数・基準日は、まず就業規則を確認することを案内する。

## YMYL配慮事項

- 「これは法定の最低基準であり、実際の付与は就業規則を確認してください」という前提を、根拠・計算式セクションおよびツール本文の両方で明示する。
- 出勤率8割未満の場合も、不安を煽る表現を避け、「継続勤務年数はリセットされない」という事実を併記する。
- 実際の利用実績（残日数）を追跡しない設計であることを明示し、架空の数値を提示しない。
- 準拠年度（2026年度）・出典（e-Gov法令検索・厚生労働省）・最終確認日（2026-07-17）を画面内に表示する。
