# 冷凍保存期間検索

## 概要

食材名（または食材カテゴリ）を入力・選択すると、その食材を家庭用冷凍庫（目安 -18℃以下）で保存できるおおよその期間と、保存前の下処理のコツ・解凍のコツを一瞬で表示する検索ツール。

- **対象**: [docs/02_ツール一覧](../../docs/02_ツール一覧.md) #37「冷凍保存期間検索（食材150種、解凍のコツ付き）」。カテゴリ3「家事・料理」
- **優先度**: S（Phase 1 MVP）／**需要**: 中／**難易度**: A／**YMYL**: 低
- **対象ペルソナ**: ペルソナ1 佐藤美咲（献立を考えるのが苦痛。まとめ買い・作り置きの冷凍保存が前提）、ペルソナ3 山本遥（一人暮らしの食材使い切り）
- **提供価値**: 「これ、あと何日冷凍しておいて平気？」を検索1回・登録不要で即答する。ロジック自体は単純な検索＋固定データ表示だが、[03_コンテンツ戦略](../../docs/03_コンテンツ戦略.md) §5.1 のプログラマティックSEO主力軸として、食材ごとに個別URL（例: `/tools/reito-hozon/hourensou`）を展開する前提の設計とする
- **将来拡張**: 本仕様のデータ表は代表20〜30種のサンプル。docs/02 の計画どおり最終的に食材150種まで拡張する（データのSSOTは `data/tables/reito-hozon.json`。★2026-07-17 旧対照表 `data/tables/reitou-hozon.json` は削除し一本化した（Q3-13 残項目の二重管理解消）★）
- **出典**: 消費者庁／農林水産省の食品保存・食品ロス削減に関する一般案内。家庭用冷凍庫の目安温度は `data/tables/reito-hozon.json` の `kateiReitoukoMeyasuOndoDo: -18`（-18℃以下）と整合させる（★旧ファイルは符号曖昧な `18` を持っていた。負数で持つこと★）

## 入力仕様

| フィールド名 | 型 | 必須/任意 | バリデーション |
|---|---|---|---|
| `query` | 文字列（フリーテキスト） | 任意 | 全角/半角どちらも許容。前後空白トリム。1〜30文字。インクリメンタルサーチ（かな入力での部分一致・ひらがな⇄カタカナ正規化） |
| `food_id` | 文字列（食材マスタのスラッグ、例: `hourensou`） | 任意（`query` と排他的だが両方指定時は `food_id` 優先） | データ表の `id` に存在するもののみ許可。存在しない場合は「該当食材が見つかりません」を表示し #37 のカテゴリ一覧へ誘導 |
| `category` | 選択式（`野菜`/`肉`/`魚介`/`ご飯・主食`/`卵・乳製品`/`果物`/`調理済み・作り置き`のいずれか、または未選択＝全件） | 任意（絞り込みフィルタ） | データ表の `category` 列挙値のみ |
| `state` | 選択式（`生`/`加熱後`/`下処理後`のいずれか、未選択＝食材ごとのデフォルト） | 任意 | 食材によって選択肢が変わる（例: 生肉は `生` のみ、ほうれん草は `下処理後(下茹で)` のみ） |

- URLクエリパラメータでの直接アクセス（`?food_id=hourensou`）をサポートし、プログラマティック展開ページ（食材別個別URL）から結果画面へ直接遷移できるようにする
- 検索結果0件の場合は「まだ登録がありません。今後追加予定です」を表示し、フィードバックボタン（[03_コンテンツ戦略](../../docs/03_コンテンツ戦略.md) §6 の1クリックフィードバック）へのリンクを出す

## ロジック仕様

計算ロジックは存在しない（固定データの検索・表示のみ）。疑似コードは以下。

```
function search(query, food_id, category, state):
    if food_id is not null:
        item = DATA.find(d => d.id == food_id)
        if item is null:
            return NOT_FOUND
        return renderDetail(item, state ?? item.default_state)

    results = DATA
    if category is not null:
        results = results.filter(d => d.category == category)
    if query is not null and query != "":
        normalized = normalize(query)  # ひらがな化・全角半角統一・空白除去
        results = results.filter(d =>
            normalize(d.name).includes(normalized) OR
            d.aliases.any(a => normalize(a).includes(normalized))
        )

    if results.length == 1:
        return renderDetail(results[0], state ?? results[0].default_state)
    else if results.length == 0:
        return NOT_FOUND
    else:
        return renderList(results)  # 候補一覧（食材名・カテゴリ・保存期間目安を1行ずつ）

function renderDetail(item, state):
    period = item.storage_period_by_state[state]  # 例: { min_weeks: 4, max_weeks: 4, unit: "週" }
    return {
        name: item.name,
        category: item.category,
        state: state,
        period_text: formatPeriod(period),      # 例:「約1ヶ月（4週間）」
        prep_tips: item.prep_tips,               # 保存前の下処理のコツ
        thaw_tips: item.thaw_tips,                # 解凍のコツ
        caution: item.caution,                    # 品質劣化・再冷凍不可などの注意
        source: "消費者庁 / 農林水産省（食品保存の一般案内）",
        base_temperature: "-18℃以下（家庭用冷凍庫の目安）"
    }

function formatPeriod(period):
    if period.min_weeks == period.max_weeks:
        return "約" + weeksToReadable(period.min_weeks)  # 4週→「1ヶ月」、2週→「2週間」
    else:
        return weeksToReadable(period.min_weeks) + "〜" + weeksToReadable(period.max_weeks)
```

- 週数→表示文字列変換ルール: 4週=「1ヶ月」、2週=「2週間」、1週=「1週間」、8週=「2ヶ月」など、4週を1ヶ月単位に丸めて表示する（データ表の並び順と可読性を優先。データ本体は週単位で保持し、表示層でのみ変換する）
- 「生」「加熱後」「下処理後」で保存期間が異なる食材は、デフォルト表示状態（`default_state`）を食材ごとに1つ定義し、他の状態はタブ切り替えで表示する

## エッジケース・注意事項

- **生と加熱後で期間が異なる食材**: ご飯・パスタ・うどん等は「加熱後」の状態でのみ冷凍を想定。生米の長期保存（常温）は本ツールのスコープ外（別の「お米の保存」記事に誘導）
- **下処理の有無で結果が変わる食材**: ほうれん草・小松菜・ブロッコリー等の葉物・アブラナ科野菜は下茹で（ブランチング）が前提。生のまま冷凍した場合は品質劣化が早いため、本表は下処理後の目安のみを掲載し、生のままでの保存は非推奨である旨を明記する
- **無洗米・玄米等の主食保存**: 米そのものの保存は#39（炊飯の水の量・合数⇄g換算）と役割分担し、本ツールは「炊いたご飯」の冷凍のみを扱う
- **再冷凍**: 一度解凍した食材の再冷凍は品質劣化・衛生面のリスクが高まるため、全食材共通の注意書きとして「解凍したものの再冷凍は避けてください」を出力欄末尾に固定表示する
- **卵の殻のままの冷凍**: 破裂の危険があるため不可。必ず割りほぐしてからという前提を明記
- **検索の曖昧一致**: 「ほうれんそう」「ホウレンソウ」「ほうれん草」等の表記ゆれを吸収する正規化処理が必須（ロジック仕様参照）
- **未登録食材**: 150種への拡張までは多くの食材が未登録。0件時は「今後追加予定」である旨を明記し、離脱時のフラストレーションを最小化する

## YMYL配慮事項

- YMYL区分は**低**（[docs/02_ツール一覧](../../docs/02_ツール一覧.md) 判定基準：低=生活情報）だが、食品衛生に関わる目安値であるため、以下を必須表示とする
  - 「保存期間はあくまで品質が保てる目安であり、安全性を保証するものではありません。保存状態（冷凍庫の開閉頻度・詰め込みすぎ等）により品質劣化は早まることがあります」の固定注記
  - 「異臭・変色・霜が多く付着している等、異常を感じた場合は食べずに廃棄してください」の固定注記
  - 医療・食中毒リスクの断定的な安全宣言（「絶対に安全」等）は行わない。[03_コンテンツ戦略](../../docs/03_コンテンツ戦略.md) §4.1 の「症状・治療・薬は扱わない」方針に従い、食中毒の症状・対処には言及しない
- 出典は消費者庁・農林水産省の一般案内に限定し、個人ブログ・SNS由来の数値は採用しない
