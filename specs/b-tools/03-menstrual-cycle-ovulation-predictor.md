# 生理周期・排卵日予測

## 概要

最終月経開始日と平均月経周期日数を入力すると、次回（および次々回）の月経開始予定日と、それに基づく排卵予測日・妊娠しやすい期間の目安を日付演算のみで算出するツール。

- 対象ペルソナ: ペルソナ3（人に聞きにくいことを中立情報で知りたい）、ペルソナ4（妊活中）、ペルソナ1・2の体調管理にも利用
- 分類: [docs/02_ツール一覧](../../docs/02_ツール一覧.md) カテゴリ1 #3「生理周期・排卵日予測（日付演算のみ）」。優先度 **S**（Phase 1 MVP）。「医療判断は含めない設計」と明記された領域
- YMYL区分: **中**（[docs/03_コンテンツ戦略](../../docs/03_コンテンツ戦略.md) §4.1「妊娠・出産の週数/日付計算」に準ずる扱い）。診断・避妊/妊活の成否判定は一切行わない
- カテゴリ5「健康・美容」#67「PMS・体調予測カレンダー」と周期計算ロジックを部品共有する（[docs/02_ツール一覧](../../docs/02_ツール一覧.md) 「#3と部品共通」の注記）

## 入力仕様

| フィールド名 | 型 | 必須/任意 | 説明・バリデーション |
|---|---|---|---|
| `lmp`（最終月経開始日） | 日付（YYYY-MM-DD） | 必須 | 当日以前の日付のみ有効。基準日より90日以上前はエラー（下記エッジケース参照） |
| `cycleLength`（平均月経周期日数） | 整数（日） | 任意（デフォルト28） | 20〜45の範囲でのみ有効。範囲外はバリデーションエラー |
| `periodLength`（月経が続く日数） | 整数（日） | 任意（デフォルト5、表示のみに使用） | 2〜10の範囲でのみ有効 |
| `baseDate`（基準日） | 日付（YYYY-MM-DD） | 任意（デフォルト＝実行時の当日） | 「排卵日を過ぎているか」等の経過表示に使用。`lmp` より前はエラー |

## ロジック仕様

黄体期（排卵から次回月経開始までの期間）は個人差が小さく、平均して約14日でほぼ一定であるという一般的な知見に基づき、次回月経開始予定日から逆算して排卵日を推定する。妊娠の成否判定・診断は一切行わない。

### 1. 次回月経開始予定日

```
nextPeriodDate = LMP + cycleLength日
```

### 2. 次々回（2周期先）の月経開始予定日

```
next2PeriodDate = LMP + (cycleLength × 2)日
```

- 表示上は `calendarCount`（デフォルト3、1〜6の範囲で調整可）に応じて `LMP + cycleLength × n` を n=1,2,3... と繰り返し算出してカレンダー表示してもよい

### 3. 排卵予測日（黄体期14日固定モデル）

```
ovulationDate = nextPeriodDate − 14日
```

- 黄体期の長さには個人差があり（一般に10〜16日程度とされる）、本ツールは「14日」を固定値として使う簡易予測（俗に言う「オギノ式」的な周期からの逆算）であることを明示する

### 4. 妊娠しやすい期間（目安）の算出

```
fertileWindowStart = ovulationDate − 5日   // 精子の生存期間（一般に3〜5日程度とされる）を考慮した目安
fertileWindowEnd   = ovulationDate + 1日   // 排卵後の卵子の生存期間（一般に24時間程度とされる）を考慮した目安
```

### 5. 経過状況の判定（baseDate との比較、表示補助のみ）

```
if baseDate < ovulationDate:
    ovulationStatus = "これからです（予測）"
elif baseDate <= fertileWindowEnd:
    ovulationStatus = "妊娠しやすい期間の目安に入っています"
else:
    ovulationStatus = "この周期の排卵日は過ぎています"
```

| 黄体期の標準日数 | 14日（固定値として使用） | 黄体期は個人差が小さく約14日でほぼ一定とされる一般的な知見。産婦人科・家族計画関連の解説で広く共有される目安値（日本産科婦人科学会・日本家族計画協会等の一般向け解説に基づく） |

## エッジケース・注意事項

| ケース | 扱い |
|---|---|
| `baseDate` < `lmp` | 入力エラー。「基準日は最終月経開始日より後の日付を指定してください」 |
| `lmp` が `baseDate` から90日以上前 | 一般的な周期予測の実用範囲を超えるため「入力範囲外です。最新の月経開始日を入力してください」と表示し計算をブロック（3周期分程度先までの予測が実用上の目安のため） |
| `cycleLength` が20〜45の範囲外 | 入力エラーとして計算をブロック |
| `periodLength` が2〜10の範囲外 | 入力エラーとして計算をブロック（計算結果自体には影響しないが、カレンダー表示のバリデーションとして必須） |
| 月末日をまたぐ場合（例: LMP=1月31日、周期28日） | 暦日数の実加算で計算するため、2月は28日または29日として自動的に処理される（例: 2026年1月31日+28日=2026年2月28日） |
| うるう年をまたぐ場合 | 暦日数の実加算のため追加補正は不要（例: 2028年2月1日+28日=2028年2月29日、2028年はうるう年） |
| 年をまたぐ場合 | 年ごとの補正は不要。単純な日数加算で自動的に年をまたぐ |
| 周期が不規則な人（`cycleLength` が月ごとに大きく変動する） | 本ツールは直近の1周期分の平均値のみを使う単純予測であることを明示し、複数周期の記録に基づく予測ではない旨を注記する（記録機能はPhase 1スコープ外） |
| 避妊目的での利用 | 本ツールは避妊法（オギノ式等）としての精度を保証しない旨を明示し、避妊目的での使用を推奨しない注記を添える |
| 妊娠中・閉経後などそもそも周期がない場合 | 想定利用者外である旨を注記するのみで、入力自体はブロックしない（判定不能なため個別のエラー処理はしない） |

## YMYL配慮事項

[docs/03_コンテンツ戦略](../../docs/03_コンテンツ戦略.md) §4.1の方針に従い、妊娠・出産に関わる日付演算として中リスク区分の配慮を行う。

- 本ツールが行うのは**入力された周期日数に基づく単純な日付演算のみ**であり、実際の排卵・妊娠の有無を検査・診断するものではない。基礎体温や排卵検査薬による確認に代わるものではないことを明示する
- 結果画面に以下の固定文言を常設表示する（変更不可の定型文としてテンプレートに組み込む）:
  > 「この予測は、入力された周期日数をもとにした計算上の目安です。実際の排卵日や妊娠しやすい時期は、ストレスや体調によって毎周期ずれることがあり、この計算だけで正確に特定することはできません。より正確に把握したい場合は、基礎体温の記録や排卵日検査薬の利用、婦人科への相談をおすすめします。避妊の目的でこの予測のみに頼ることはお控えください。妊活・体調について気になることがある場合は、必ず医師にご相談ください。」
- 出典（日本産科婦人科学会・日本家族計画協会等）と最終更新日をページ下部に常時表示
- 計算式（黄体期14日固定モデルであること）をページ内に明示し、ブラックボックス化しない
- 排卵日検査薬・妊活サプリ等の特定商品への誘導リンクは掲載しない（[docs/06_紹介ポリシー](../../docs/06_紹介ポリシー.md) の条件に該当しない限り掲載しない方針を厳守）
