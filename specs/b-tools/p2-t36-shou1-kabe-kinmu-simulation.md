# 小1の壁 勤務シミュレーション（学童時間×勤務時間）（P2-T36）

## 概要

保育園時代と違い、学童保育は閉所時刻が早く、延長がない施設も多い（いわゆる「小1の壁」）。本ツールは、ユーザー自身が入力した「学童の閉所時刻」「学童から自宅までの移動時間」「退勤時刻」「職場から自宅までの移動時間」「退勤後の準備時間（バッファ）」から、実際に何時から何時まで子どもが一人になる時間が生じるか、また子どもを一人にしないためには何時までに退勤する必要があるか（＝勤務時間をどれだけ調整する必要があるか）をシミュレーションする。

- docs/02_ツール一覧.md **#83**（カテゴリ6: 仕事・キャリア、優先度 中、難易度B）。BACKLOG.md queueId: `P2-T36`。
- 依存データ: `data/seido/gakudou-hoiku-kijun.json`（放課後児童健全育成事業の設備及び運営に関する基準・こども家庭庁実施状況調査）。ただし本ツールの中心である時刻の加減算（「一人になる時間」の算出）そのものはこの制度データに依存しない。制度データから取得するのは「対象学年（小1〜小6）」「支援の単位の定員（おおむね40人）」「学年別待機児童数（全国）」という、判定結果とは独立した**参考情報**のみである。
- P2-T35（保育園お迎え逆算 勤務可能時間計算）の学童版だが、以下の点が異なる。
  1. **開所・閉所時刻をデータから取得できない**: 学童保育の開所時間は国の省令（放課後児童健全育成事業の設備及び運営に関する基準）第18条で「休業日は1日8時間以上、平日は1日3時間以上を原則として、当該事業所ごとに定める」とされており、かつこの基準自体が市町村への「参酌基準」（従うべき基準ではなく、参考にすべき基準）に過ぎない（`data/seido/gakudou-hoiku-kijun.json` の `data.overview.standardsNature`）。したがって閉所時刻・お迎え締切時刻に全国一律の数値は存在しない。★本ツールは架空の全国一律値を一切ハードコードせず、学童の閉所時刻をユーザー自身に入力してもらう設計とする★。
  2. **「お迎え」ではなく「子どもの一人歩き・留守番」を扱う**: 保育園版（P2-T35）は保護者が保育園まで迎えに行く前提だったが、小学生は学童から一人で（または集団で）帰宅できることが多い。本ツールは「子どもが学童から自宅に着く時刻」と「保護者が自宅に着く時刻」の差分から、子どもが一人になる時間を算出する。
  3. **対象学年・待機児童の現実を参考情報として添える**: 制度上の対象学年（小1〜小6）、支援の単位の定員（おおむね40人）、学年別待機児童数（全国。小4が最多）を、判定結果とは独立した参考情報として表示する。
- YMYL区分: 低〜中。子どもが一人になる時間が生じる可能性を煽らず、事実として淡々と示す（docs/09の口調厳守）。断定的な「危険です」「かわいそうです」等の評価は行わず、結果を機械的な目安として提示したうえで、留守番の可否や安全対策の検討・相談窓口の案内にとどめる。
- 日をまたぐ計算（深夜勤務等）は対象外。入力・計算はすべて同一日内の時刻（0:00〜23:59）として扱う。

## ロジック仕様（時刻演算）

すべての時刻は `HH:mm` 文字列を「0時0分からの経過分数（0〜1439の整数）」に変換してから計算する（`parseHHMM`）。日をまたぐ計算は行わないため、子ども・親いずれかの到着時刻が1440分（24:00）以上になる場合はエラーとして扱う。

### 1. 子どもの帰宅時刻・親の帰宅時刻

```
gakudouHeishoMin = parseHHMM(gakudouHeishoJikoku)
kodomoKitakuMin = gakudouHeishoMin + gakudouKaraJitakuIdouMin
if kodomoKitakuMin >= 1440:
    → 入力エラー（「学童の閉所時刻に対して自宅までの移動時間が長すぎます。日をまたぐ計算はできません。」）

taikinMin = parseHHMM(taikinJikoku)
oyaKitakuMin = taikinMin + bufferMin + shokubaKaraJitakuIdouMin
if oyaKitakuMin >= 1440:
    → 入力エラー（「退勤時刻に対して移動時間・準備時間が長すぎます。日をまたぐ計算はできません。」）
```

### 2. 子どもが一人になる時間

```
hitoriJikanMin = max(0, oyaKitakuMin - kodomoKitakuMin)
```

親の帰宅目安が子どもの帰宅目安より後になる場合、その差分が「子どもが一人になる時間」。親の帰宅が子どもの帰宅と同時またはそれより早い場合は0分（エラーにはしない。望ましい状態としてそのまま0分を表示する）。★退勤時刻が学童の閉所時刻よりも遅い（＝現実の「小1の壁」でよくある状況）場合も、エラーにはせず、そのまま大きな一人時間の値を機械的に算出して表示する★。これは異常な入力ではなく、本ツールが可視化すべき本来のユースケースである。

### 3. 子どもを一人にしないための退勤限界時刻（参考）

```
taikinGenkaiMinRaw = kodomoKitakuMin - bufferMin - shokubaKaraJitakuIdouMin
if taikinGenkaiMinRaw >= 0:
    taikinGenkai = { minutes: taikinGenkaiMinRaw, label: formatHHMM(taikinGenkaiMinRaw), possible: true }
else:
    taikinGenkai = { minutes: null, label: "算出できません（学童の閉所時刻までに間に合う退勤時刻がありません）", possible: false }
```

この値は「退勤時刻をどれだけ調整する必要があるか」の裏返しであり、数式上 `taikinJikoku（実際の退勤分）− taikinGenkaiMinRaw = hitoriJikanMin` の関係が常に成り立つ（一人になる時間ぶんだけ退勤を早める必要がある、という関係）。taikinGenkaiMinRaw が負になる場合（移動時間・バッファが学童の閉所時刻に対して長すぎる場合）は「算出できません」と表示し、メインの出力（hitoriJikan）はブロックしない。

### 4. 実労働可能時間（出勤時刻が入力され、かつ退勤限界時刻が算出できる場合のみ）

```
if shukkinJikoku が入力されている and taikinGenkai.possible:
    shukkinMin = parseHHMM(shukkinJikoku)
    workableMin = taikinGenkai.minutes - shukkinMin
    if workableMin >= 0:
        workable = { minutes: workableMin, label: "◯時間◯分" }
    else:
        workable = null（算出不可。エラーにはしない）
else:
    workable = null
```

「子どもを一人にしない場合、その日何時間働けるか」の目安。出勤時刻の形式が不正な場合のみ入力エラーとする。退勤限界時刻自体が算出できない場合や、出勤時刻が退勤限界時刻より後になる場合は、メインの一人時間の計算をブロックせず、この項目だけ非表示（null）にする。

### 5. 時刻・分数フォーマット関数

```
parseHHMM(s): "HH:mm" 形式（時00〜23・分00〜59の2桁ゼロ埋め）に一致しなければ null を返す。一致すれば 時*60+分 を返す。
formatHHMM(min): 0〜1439の整数を "HH:mm" のゼロ埋め文字列に変換する。
formatDuration(min): 0以上の整数を "◯時間◯分" 形式に変換する（0時間なら "◯分" のみ、0分なら "◯時間" のみ）。
```

### 6. 参考情報（判定結果とは独立）

`data/seido/gakudou-hoiku-kijun.json` から以下を取得し、計算結果とは無関係の参考情報として画面に併記する。

### ★データの限界: 開所・閉所時刻をユーザー入力とする理由★

`data/seido/gakudou-hoiku-kijun.json` の `data.overview.standardsNature`・`data.standards.openingHours` が明記するとおり、学童保育の開所時間に関する国の基準は次の2点で「全国一律の実際の時刻」を導出できない。

1. **基準自体が「参酌すべき基準」に過ぎない**（令和2年4月1日の第9次地方分権一括法により、職員配置基準も含め全ての基準が「従うべき基準」から「参酌すべき基準」に変更済み）。市町村・事業所は国の基準と異なる運用を適法に行える。
2. **国の基準が定めているのは「時刻」ではなく「時間数」の下限のみ**（休業日は1日8時間以上、平日は1日3時間以上を原則として「当該事業所ごとに定める」）。開所時刻・閉所時刻（何時から何時まで）そのものは省令に数値として存在せず、事業所の運営規程で個別に定められる。

このため、本ツールが仮に「学童は平日◯時まで」のような全国一律値をハードコードすると、実態と大きく異なる誤った目安を提示することになり、YMYL上のリスクが生じる。よって本ツールは、学童の閉所時刻（お迎え・帰宅の締切時刻）をユーザー自身の入力必須項目とし、画面上にも「学童の開所・閉所時刻は施設や市区町村によって異なり全国一律の基準はない」旨を常時表示する。制度データから取得できる対象学年・定員・待機児童統計は、あくまで判定結果と独立した参考情報として扱い、計算そのものには一切使用しない。

## エッジケース

- **子どもが一人になる時間が0分**: 親の帰宅目安が子どもの帰宅目安と同時またはそれより早い場合。エラーにはせず「0分」を表示する。
- **移動時間がすべて0分**: 学童・職場がいずれも自宅に隣接している等の場合。バリデーションはそのまま通過する。
- **退勤時刻が学童の閉所時刻よりも遅い（現実によくある「小1の壁」そのもの）**: エラーにはせず、大きな一人時間の値をそのまま算出・表示する（本ツールが可視化すべき本来のユースケース）。
- **退勤限界時刻が算出できない（移動時間・バッファが長すぎて0時より前になる）**: 入力エラーにはせず、「算出できません」の文言を表示しつつ、メインの一人時間の計算・表示は継続する。
- **出勤時刻が退勤限界時刻より後（実労働可能時間が負になる）**: エラーにはせず、実労働可能時間の項目のみ非表示にする。
- **子ども・親いずれかの到着時刻が24:00以上になる（日をまたぐ）**: 入力エラー。「日をまたぐ計算はできません」という趣旨のメッセージを表示する。
- **不正な時刻文字列**（`25:00`、`9:5`、`abc`、空文字、`24:00`等）: いずれも入力エラー。
- **移動時間・バッファが負数、またはNaN**: 入力エラー。
- **学年を選択しない場合**: 学年別の待機児童数は表示せず、対象学年・定員・全国合計の待機児童数のみを参考情報として表示する。

## YMYL配慮事項

- 本ツールはYMYL低〜中区分。子どもが一人になる時間が生じる可能性を、断定的な評価（「危険です」「かわいそうです」等）を付けずに事実として淡々と示す（docs/09の口調厳守）。
- 結果画面には常時、以下の趣旨のCalloutを表示する: 「学童の開所時間は施設ごとに異なるため必ず自分の学童で確認してください」「この結果は入力値から機械的に導いた目安であり、実際の勤務調整は就業規則・上長の指示に従ってください」「子どもが一人になる時間が生じる場合は、留守番の可否・安全対策を家庭の状況に応じて検討し、必要に応じて学童・学校・自治体の窓口に相談してください」。
- 対象学年・支援の単位定員・待機児童統計は、判定結果とは独立した参考情報であることを明記し、あたかも「あなたの学童の状況」であるかのように誤解されないようにする。
- 学年別待機児童数のうち小学4年生が最多である事実は淡々と紹介し、「高学年は入れない」という不安を煽る書き方は避ける。
