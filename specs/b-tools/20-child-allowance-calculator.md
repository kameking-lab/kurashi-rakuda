# 児童手当計算（2026年度・所得制限撤廃後）

## 概要

世帯の対象児童（0歳〜高校生年代）の生年月日と出生順位を入力すると、2024年10月改正後（所得制限撤廃・高校生年代までの支給延長・多子加算の判定方法変更）の制度に基づき、児童ごとの月額および世帯合計の月額・年額・1回あたりの支給額を計算するツール。

- 対象ペルソナ: ペルソナ1（佐藤美咲）・ペルソナ2（田中絵里、「正確な数字を最速で出す」ニーズ）を主対象。
- docs/02_ツール一覧.md **#20**（カテゴリ2: 子育て、優先度 **S**、難易度 B）。`factory/data/seido/jidou-teate.json` の制度データを**移植**し、年度更新は本サイト側で保守する方針（出典: [02_ツール一覧](../../docs/02_ツール一覧.md) 該当行）。
- YMYL区分: **中**（お金の制度計算。金額の食い違いは許容されないため、下記の金額はすべて `factory/data/seido/jidou-teate.json` の値と完全一致させる）。

## 入力仕様

| フィールド名 | 型 | 必須/任意 | バリデーションルール |
|---|---|---|---|
| 基準日（baseDate） | 日付（YYYY-MM-DD） | 任意（未入力時は今日の日付） | 妥当な日付形式であること |
| 対象児童リスト（children） | 配列（1〜6人程度を想定、上限は画面側のUIで規定） | 必須（1人以上） | 各要素は下記の子ども単位フィールドを持つ |
| （子ども単位）生年月日（birthDate） | 日付（YYYY-MM-DD） | 必須 | 基準日以前の日付であること |
| （子ども単位）出生順位（birthOrder） | 整数 | 必須 | 1以上の整数。「大学生年代（22歳の年度末）までの経済的に扶養している子どもを年長者から数えた順位」を利用者自身に入力してもらう（ツール側では自動判定しない。入力欄のヘルプ文言でこの定義を明記する） |

補足: 所得を尋ねる入力欄は設けない。2024年10月の所得制限撤廃（`income_limit.abolished: true`）により、本ツールでは所得情報を一切収集しない設計とする。

## ロジック仕様

`factory/data/seido/jidou-teate.json` の値をそのまま参照する（下記データ表に転記）。ブラックボックス化を避けるため、判定ロジックと参照した制度データを結果画面に明示する。

### 1. 年齢・区分の判定（子どもごと）

```
年齢 = floor(基準日 と 生年月日 の満年齢差)   # 誕生日を迎えたかどうかで判定する通常の満年齢計算

# 高校生年代の年度末カットオフ（18歳に達した日以後の最初の3月31日）
eighteenthBirthday = 生年月日 に18年を加算した日付
if eighteenthBirthday.month >= 4:
    koukouCutoff = Date(eighteenthBirthday.year + 1, 3, 31)
else:  # 1〜3月生まれ
    koukouCutoff = Date(eighteenthBirthday.year, 3, 31)

if 基準日 > koukouCutoff:
    区分 = "対象外"（高校生年代を超過）
elif 年齢 < age_boundary.jidou_kara_sai (= 3):
    区分 = "0〜2歳"
else:
    区分 = "3歳〜高校生年代"
```

### 2. 月額の決定（子どもごと）

```
if 区分 == "対象外":
    月額 = 0
elif 区分 == "0〜2歳":
    月額 = monthly_amount.0_to_2   # 15,000円。出生順位に関わらず一律（json仕様どおり、0〜2歳区分に多子加算は適用しない）
elif 区分 == "3歳〜高校生年代":
    if 出生順位 >= 3:
        月額 = monthly_amount.3_to_high_school_third_or_later   # 30,000円
    else:
        月額 = monthly_amount.3_to_high_school   # 10,000円
```

### 3. 世帯合計・年額・1回あたり支給額

```
世帯合計月額 = Σ(各子どもの月額)
世帯合計年額 = 世帯合計月額 × 12
1回あたり支給額 = 世帯合計月額 × (12 / payment.times_per_year)   # times_per_year=6のため ×2
```

### 4. 所得制限の扱い

```
income_limit.abolished == true のため、所得に関する判定は一切行わない。
結果画面には「所得制限は2024年10月に撤廃されているため、世帯所得にかかわらず全額支給されます」という固定文言を表示する。
```

## エッジケース・注意事項

- **0〜2歳区分の子どもの出生順位が3以上の場合**: json仕様上、0〜2歳区分には多子加算が適用されないため、出生順位に関わらず月額は15,000円で一律になる。この挙動は仕様であり不具合ではない旨をテストケースで明示する。
- **高校生年代の年度末カットオフの境界**: 18歳誕生日が1〜3月かどうかで年度末の年が変わるため、誕生日の月によって挙動を分岐させる（上記ロジック仕様参照）。境界日そのもの（3月31日 vs 4月1日）で対象・対象外が切り替わることをテストケースで明示する。
- **高校生年代を超えた子ども（19歳以上等）**: 支給対象外・月額0円として扱う。ただし、その子を出生順位のカウント（下の子の「第3子」等の判定）に含めるかどうかは、利用者が入力する出生順位の定義に委ねる（ツール側では自動計算しない設計のため、入力ヘルプで「大学生年代までの子を含めて数える」という一般的な考え方を案内する）。
- **対象児童が0人**: 「対象のお子さまがいないため、世帯合計は0円です」と表示する。
- **生年月日が基準日より未来**: 入力エラー（「生年月日は基準日より前の日付にしてください」）。
- **出生順位が0以下・小数**: 入力エラー（「出生順位は1以上の整数で入力してください」）。
- **双子・多胎児**: 同一の生年月日で複数人分を入力する。出生順位はそれぞれ利用者の判断で異なる値（例: 第2子・第3子）を入力する。
- **制度改定への追随**: 金額・年齢境界・支給回数・所得制限の有無はすべて `factory/data/seido/jidou-teate.json` を単一の情報源（Single Source of Truth）とする。本仕様書に金額を直接ハードコードで説明しているのは可読性のためであり、実装上は必ず同JSONを参照し、JSON側が更新された場合はこの仕様書の記載も追随して更新する。

## YMYL配慮事項

- お金に関わる制度計算のため、金額の算出根拠（区分・出生順位・年度末カットオフ）を必ず画面内に明示し、ブラックボックス化しない（docs/03_コンテンツ戦略.md §4.1の原則）。
- 固定文言案（結果画面下部・常時表示）: 「この結果は2026年度時点の制度（こども家庭庁「児童手当制度のご案内」、2024年10月改正後）に基づく目安です。実際の支給額・支給時期は、お住まいの自治体の窓口・通知でご確認ください。」
- 出生順位のカウント方法について誤解を招かないよう、固定文言案: 「『第◯子』の数え方は、高校生年代を超えたお子さまも含めて年長者から数えるのが一般的です。詳しい判定はお住まいの自治体にご確認ください。」
- 最終更新日／準拠年度／出典の3点セット（`last_verified: 2026-07-17`／`revision_year: 2026`／こども家庭庁）を必ず表示する。次回確認期限（`next_check_due: 2026-10-01`）を過ぎたコンテンツは記事工場のCI忠実性ゲートで更新確認の対象とする。
