# 洗濯表示検索

## 概要

衣類タグの洗濯表示記号（新JIS記号）から、その記号の意味・取り扱い方法を検索できるツール。記号の見た目（形状・数字・線の本数）やキーワード（例:「手洗い」「アイロン不可」「漂白剤」）から該当する記号を絞り込み、意味と具体的な取り扱い方を平易文で表示する。

- **対象**: [docs/02_ツール一覧](../../docs/02_ツール一覧.md) #41「洗濯表示検索（新JIS記号、写真対応は将来）」。カテゴリ3「家事・料理」
- **優先度**: S（Phase 1 MVP）／**需要**: 高／**難易度**: B／**YMYL**: 低
- **対象ペルソナ**: ペルソナ1 佐藤美咲（子ども服・デリケートな衣類の洗濯で失敗したくない）、ペルソナ2 田中絵里（時間がない中で素早く正しい取り扱いを知りたい）
- **提供価値**: 「このマーク、見たことあるけど意味が分からない」を検索1回・登録不要で即答する。将来的にカメラで記号を撮影して自動判定する機能（画像認識）を検討するが、**Phase 1では写真対応はスコープ外**とし、テキスト検索・カテゴリ一覧からの選択のみを実装する
- **準拠規格**: 家庭用品品質表示法に基づく繊維製品品質表示規程が引用する JIS L 0001。**令和6年8月20日改正版（JIS L 0001:2024）**に準拠する。国際規格 ISO 3758 の令和5年12月改正との整合のための改正であり、⑴洗濯記号の新規追加（記号111・511）、⑵洗濯記号の意味の変更（記号530・520・510・620・621・610・611）、⑶記号のデザインの微修正（記号110・100・200・500）の3点が変更された。経過措置（従前の表示での販売可）は令和7年8月19日で終了している
- **★記号の総数を断定しないこと★**: 消費者庁の現行解説ページには**記号の総数の明記がない**ため、仕様書・UI・記事のいずれでも「全◯種類」と書かない。記号は消費者庁ページの**記号番号**（190・111・511 等）で一意に識別する。従前この仕様書が記していた「全41記号」は、平成27年3月31日の消費者庁公表文が旧JIS L 0217の22種類との比較で述べた**2014年版JISについての数値**であり、記号111・511が追加された現行版の数値ではない（D2・2026-07-17）
- **データのSSOT**: [`data/tables/sentaku-hyoji.json`](../../data/tables/sentaku-hyoji.json)。記号の追加・修正は同ファイルのみを更新する。★従前は `data/tables/sentaku-hyouji.json` との2ファイルに分かれていたが、D2 で後者を廃止し統合した★
- **出典**: 消費者庁「洗濯表示（令和6年8月20日以降）」（家庭用品品質表示法・製品別品質表示の手引き）<https://www.caa.go.jp/policies/policy/representation/household_goods/guide/wash_02.html>（データSSOTの `source.url` と同一）

## 入力仕様

| フィールド名 | 型 | 必須/任意 | バリデーション |
|---|---|---|---|
| `query` | 文字列（フリーテキスト） | 任意 | 1〜30文字。「手洗い」「40度」「アイロン不可」「ドライクリーニング」等、記号の意味・見た目の特徴語での検索を許容 |
| `category` | 選択式（`洗濯`/`漂白`/`乾燥`/`アイロン`/`クリーニング`のいずれか、未選択＝全件） | 任意（絞り込みフィルタ） | データ表の `category` 列挙値のみ |
| `symbol_id` | 文字列（記号マスタの`id`） | 任意（`query`と排他的だが両方指定時は`symbol_id`優先） | データ表の`id`に存在するもののみ許可 |

- 記号の「見た目」からの絞り込みを助けるため、カテゴリ選択画面では各カテゴリの記号の基本形（桶／三角／四角／アイロン／丸）をアイコンとテキストで併記した一覧をまず表示し、そこから該当しそうな記号を選ばせるUXとする（フリーテキスト検索が苦手なユーザーへの導線）
- **写真アップロードによる記号自動判定は本仕様のスコープ外**（[00_企画書](../../docs/00_企画書.md)・docs/02 の「写真対応は将来」に準拠。将来のPhaseで別仕様として検討する）

## ロジック仕様

計算ロジックは存在しない（固定データの検索・表示のみ）。疑似コードは以下。

```
function search(query, category, symbol_id):
    if symbol_id is not null:
        item = DATA.find(d => d.id == symbol_id)
        if item is null:
            return NOT_FOUND
        return renderDetail(item)

    results = DATA
    if category is not null:
        results = results.filter(d => d.category == category)
    if query is not null and query != "":
        normalized = normalize(query)  # ひらがな化・全角半角統一・空白除去
        results = results.filter(d =>
            normalize(d.meaning).includes(normalized) OR
            d.keywords.any(k => normalize(k).includes(normalized))
        )

    if results.length == 1:
        return renderDetail(results[0])
    else if results.length == 0:
        return NOT_FOUND
    else:
        return renderList(results)  # 候補一覧（記号名・カテゴリ・意味の要約）

function renderDetail(item):
    return {
        category: item.category,
        symbol_shape: item.symbol_shape,     # 見た目の説明文（例:「洗濯桶の中に『40』、下に線1本」）
        meaning: item.meaning,               # 正式な意味
        handling_tip: item.handling_tip,     # 実際の取り扱い方の平易な言い換え
        related_symbols: item.related_symbols, # 見間違えやすい・対になる記号への内部リンク
        source: "消費者庁（家庭用品品質表示法・JIS L 0001:2024）",   # データSSOTの source.label ＋ jisEdition から生成
        effective_date: "令和6年8月20日以降"                        # データSSOTの effectiveDateLabel
    }
```

- 「洗濯桶に40、下線1本」のように、記号は「基本形（桶/三角/四角/アイロン/丸）」＋「数字（温度上限）」＋「線の本数（処理の強さ）」の組み合わせで構成されるため、データ表の `symbol_shape` は視覚的特徴を文章化して保持する（実際のSVGアイコン表示は別途フロントエンド実装で対応し、本仕様ではデータ構造とテキスト説明のみを定義する）

## データ表

★データは仕様書に持たない★ 記号（記号番号・カテゴリ・見た目・意味・お手入れ方法・キーワード）と関連記号の対応は、
**単一の情報源(SSOT)** である [`data/tables/sentaku-hyoji.json`](../../data/tables/sentaku-hyoji.json) が保持する。
仕様書に記号表を再掲すると、改正時に二重管理となって片方が陳腐化するため（D2 の原因がこれ）、ここでは構造のみを定義する。

| フィールド | 内容 |
|---|---|
| `id` | 内部ID（`sen_40_yowai` 等）。UIの遷移・関連記号の参照に使う |
| `number` | ★消費者庁の解説ページ『各記号の詳細』の記号番号★（190・111・511 等）。一次情報との対応はこの番号で取る。分岐が必要な場合は `id` ではなく `number` を使う |
| `category` | `洗濯` / `漂白` / `乾燥` / `アイロン` / `クリーニング` の5カテゴリ（データSSOTの `categories`） |
| `symbolShape` | 見た目の説明文（例:「洗濯桶の中に『40』、下に線1本」） |
| `meaning` | 記号の意味。消費者庁の解説ページの記述に基づく要約（★JIS規格票本文の転載はしない★） |
| `handlingTip` | 実際の取り扱い方の平易な言い換え |
| `keywords` | 検索用のキーワード。表記ゆれ（「ウエット」/「ウェット」）を含む |

収録範囲は、消費者庁の解説ページ「(2)各記号の詳細」の表1〜表7（洗濯処理・漂白処理・タンブル乾燥・自然乾燥・
アイロン仕上げ処理・ドライクリーニング・ウエットクリーニング）に掲載された記号。

基本記号は5種類（洗濯処理＝桶／漂白処理＝三角形／乾燥処理＝正方形／アイロン仕上げ処理＝アイロン形／
商業クリーニング処理＝円）。付加記号は、処理温度（桶の中の数字、乾燥・アイロンのドット数。★ドット数が増えると
処理温度がより高い★）、弱い処理を表す下線1本、非常に弱い処理を表す下線2本、禁止を表す重ね書きの「×」。

## エッジケース・注意事項

- **★改正前の表示との混同（2世代ある）★**: ①令和6年8月19日以前に表示された衣類には**改正前の記号**が付いていることがある。見た目が同じでも意味が違う記号があるため特に危険（例: アイロンの点3つは改正前200℃・改正後210℃／点1つは改正前「110℃・スチームなし」・改正後「120℃」で、スチームなしは新記号511に分離された）。②平成28年11月30日以前の衣類には、体系の異なる**旧JIS（JIS L 0217、22記号）**が付いていることがある。本ツールは**令和6年8月20日以降の記号のみに対応**し、データSSOTの `oldJisNote` を検索結果に固定表示する。改正前の記号・旧JISへの対応は将来検討事項としてスコープ外
- **記号の組み合わせ（タグ1枚に複数記号）**: 実際の衣類タグには洗濯・漂白・乾燥・アイロン・クリーニングの5記号が並んで表示されるのが一般的。本ツールは記号1つずつの意味を検索する設計とし、タグ全体の一括読み取り（写真対応）は将来機能とする
- **デリケート素材の扱い**: 「非常に弱い」処理指定やドライクリーニングのみ可の表示がある衣類（シルク・ウール・スーツ等）は、家庭洗濯を試みると型崩れ・縮み・色落ちのリスクが高いため、「表示に忠実に、専門のクリーニング店へ相談することを推奨します」という注記を該当記号（カテゴリ=クリーニング、および手洗い記号＝**記号番号110・111**）の詳細表示に付与する。★令和6年改正で手洗いは110（40℃）・111（30℃）の2記号になったため、`id` の直書き（`sen_tearai`）ではなく記号番号で判定する★
- **記号が読み取れない・タグが切れている場合**: データ検索の対象外となるため、「表示が読み取れない場合は、無理に家庭で洗濯せず、クリーニング店に相談することをおすすめします」という一般的な注意喚起を用意する
- **温度表記の誤読**: 「40」と「140」など、桁数を見誤りやすい表示は見間違えやすい記号として `related_symbols` にまとめ、並べて比較できるようにする
- **0件時**: データSSOTの `notFoundNote`（「該当する記号が見つかりませんでした。カテゴリから探すか、消費者庁の出典ページでご確認ください。」）を表示し、フィードバック導線（[03_コンテンツ戦略](../../docs/03_コンテンツ戦略.md) §6）へ誘導する

## YMYL配慮事項

- YMYL区分は**低**（生活情報）だが、誤った取り扱いにより衣類の損傷（縮み・色落ち・型崩れ）という実害につながりうるため、以下を必須表示とする
  - 「記号の意味は一般的な目安です。実際の取り扱いは、衣類ごとのタグ表示を優先してください」の固定注記
  - デリケート素材・クリーニング専用表示については「不安な場合は自己判断で洗わず、クリーニング店に相談してください」という安全側の案内を明記し、断定的に「家庭で洗っても大丈夫」という表現は避ける
- 出典は消費者庁の公式案内（家庭用品品質表示法・JIS L 0001:2024）に限定し、準拠年度（令和6年8月20日以降）を全記号詳細ページに固定表示する（[03_コンテンツ戦略](../../docs/03_コンテンツ戦略.md) §4.2「全ツールに最終更新日／準拠年度／出典の3点セットを表示」に整合）
