# 介護サービス種類 かんたん逆引き（P2-T40）

## 概要

「今、困っていること」（例: 一人で入浴するのが不安、日中一人にするのが心配、少しの間だけ預けたい、自宅を暮らしやすく改修したい等）を複数選択すると、それに対応する介護保険サービスの種類（訪問介護・通所介護・短期入所生活介護・福祉用具貸与・住宅改修等）を、名前と概要つきで逆引き表示するツール。

- docs/02 **#101**（カテゴリ7: 介護、優先度 中、難易度 B、優先度A）。BACKLOG.md queueId `P2-T40`。
- 依存データ: `data/seido/kaigo-hoken.json`（既存。ただし後述のとおり限定的な引用にとどまる）。
- カテゴリ: `care`。
- YMYL区分: **中**。サービス選択の参考情報にとどめ、「実際の利用にはケアマネジャー等への相談が必要」である旨を常に明示する。

## ★データの限界（着手前調査の結論。正直な記載）★

`data/seido/kaigo-hoken.json` を精査した結果、このファイルは `futanWariai`（利用者負担割合）・`kubunShikyuGendo`（区分支給限度基準額）・`tanka`（1単位の単価）・`kougakuKaigoServiceHi`（高額介護サービス費）・`hojokyufu`（補足給付）・`yokaigoNintei`（要介護認定の区分）・`gassan`（高額医療・高額介護合算療養費）という**費用計算系のキー構成**であり、「訪問介護」「通所介護（デイサービス）」「短期入所生活介護（ショートステイ）」等のサービス種類名を体系立てて一覧化したカタログは**含まれていない**。

具体的に確認した箇所:

- `tanka.serviceTypeRule`：「上乗せ割合はサービスの人件費割合区分（70%／55%／45%）により異なる」という**区分の存在**には触れているが、個々のサービス名の列挙・定義はない（`verify.skip=true`、告示別表の構造説明のため単一文字列照合不能と明記）。
- `kubunShikyuGendo.description`：「これを超えた分は全額自己負担」という限度額の説明のみで、限度額の対象となる個別サービス名の列挙はない。
- その他のキー（`futanWariai`・`kougakuKaigoServiceHi`・`hojokyufu`・`yokaigoNintei`・`gassan`）にもサービス種類のカタログはない。

したがって本ツールは、次の2層構造で正直に情報源を分離する。

1. **サービス種類の名称・分類・概要**（訪問介護、通所介護、短期入所生活介護、福祉用具貸与、住宅改修等の名前と説明）は、**介護保険法（平成九年法律第百二十三号）第8条・第8条の2・第45条が定める類型に基づく一般的な整理**であり、`data/seido/kaigo-hoken.json` の数値データそのものではない。2026-07-18に e-Gov法令検索の法令API（`?response_format=xml`）で介護保険法第8条第2項〜第13項・第24項・第26項〜第29項の条文を取得し、サービス名・項番号を機械照合済み（`KaigoServiceGyakuHiki.calc.ts` 冒頭コメント参照）。住宅改修（第45条）は条項の存在は確認したが、支給限度額など具体的な金額は本セッションで条文全文を取得できず未照合のため、**金額を書かず「限度額がある」という定性的な説明にとどめている**（捏造回避）。
2. **要介護度の区分**（`yokaigoNintei.levels`）と**区分支給限度基準額**（`kubunShikyuGendo.levels`）の2点のみを、`data/seido/kaigo-hoken.json` からの実引用として画面下部に参考情報として併記する。これは「サービスを使うには要介護認定が必要」「利用できる量には要介護度ごとの上限がある」という文脈情報であり、逆引き結果（サービス名の一覧）とは独立した情報として表示する。

## 入力仕様

| フィールド名 | 型 | 必須/任意 | バリデーションルール |
|---|---|---|---|
| 困りごと（worries） | チェックボックスの複数選択（`WORRIES` の `id` の配列） | 任意（複数選択可・0件でも可） | 未選択時は結果を表示せず、選択を促す案内のみ表示する |

## ロジック仕様

### 1. サービスカタログ（`SERVICE_CATALOG`）

17件のサービス種類を、7つの分類（訪問系／通所系／短期入所系／福祉用具・住宅改修系／特定施設／居宅介護支援／施設サービス）に整理した定数配列。各要素は `key`・`name`・`category`・`overview`・`legalBasis`（根拠条項）を持つ。

### 2. 困りごと対応表（`WORRIES`）

10件の「困りごと」を定義し、各困りごとに該当する `SERVICE_CATALOG` の `key` を1件以上紐付ける。

### 3. 逆引き関数（`getServicesForWorries`）

```
getServicesForWorries(選択された困りごとのid配列):
  未知のidは無視する（例外を投げない）
  各困りごとのserviceKeysを集約し、同じサービスが複数の困りごとに該当する場合は
  1件にまとめて matchedWorryLabels（該当理由になった困りごとのラベル一覧）を持たせる
  表示順は SERVICE_CATALOG の定義順（＝分類順）を維持する
  未選択（空配列）は空配列を返す
```

### 4. カテゴリ別グループ化（`groupServicesByCategory`）

逆引き結果を7分類ごとに整理し、UI側でカテゴリ見出しつきで表示する。

## エッジケース・注意事項

- **未選択**: 困りごとを1件も選んでいない場合は結果を表示せず、選択を促す案内のみ表示する。
- **未知のid**: `getServicesForWorries` に `WORRIES` に存在しないidを渡しても例外を投げず、単に無視する（防御的実装）。
- **重複選択・べき等性**: 同じ困りごとのidを複数回含む配列を渡しても、結果は1回だけ渡した場合と同じになる。
- **カテゴリの網羅性**: すべての困りごとを選択すると `SERVICE_CATALOG` の全17件が過不足なく該当する（テスト#14で機械検証）。
- **サービス名の断定回避**: 「このサービスを使えば必ず解決する」という断定はせず、あくまで一般的な対応の目安である旨と、実際の利用にはケアマネジャー等への相談が必要である旨を常時表示する。
- **要支援者向けサービスの実施主体差**: 訪問介護・通所介護は、要支援1・2の方については市町村の地域支援事業（介護予防・日常生活支援総合事業）として提供され、内容・料金が市町村ごとに異なる場合がある。本ツールはこの制度上の分岐を個別に判定せず、「実際の利用可否・内容はケアマネジャー・地域包括支援センターに確認」という一般的な注意喚起に含めて対応する（断定的な判定はしない）。

## YMYL配慮事項

- サービス種類の名称・分類が `data/seido/kaigo-hoken.json` の数値データそのものであるかのような誤解を避けるため、ツール画面・出典表示の両方で「一般的な整理であり、介護保険法第8条等に基づく」旨を明記する。
- 「実際の利用にはケアマネジャー（居宅介護支援事業所）またはお住まいの市区町村・地域包括支援センターへの相談が必要」である旨を、結果表示のたびに（tone="caution" の Callout で）明示する。
- 要介護度の「状態像」（法令・告示に根拠がない）を一切説明しない（`kaigo-hoken.json` の `noStateDefinition` の鉄則を継承）。
- 住宅改修の支給限度額など、本セッションで一次資料を機械照合できなかった数値は書かない（金額の断定を避ける）。
- 紹介送客（特定の事業者・施設への誘導）は一切行わない。
