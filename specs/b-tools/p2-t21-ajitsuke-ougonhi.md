# 味付け黄金比計算（人数・分量スケール）（P2-T21）

## 概要

和食の定番料理8品目（万能だれ〈照り焼き用〉・煮魚・煮物・めんつゆ・万能合わせだれ・すき焼きの割り下・天つゆ・酢の物〈三杯酢〉）について、調味料の配合比率（いわゆる「黄金比」）を、人数または基準となる調味料の分量（ml直接指定）でスケールし、各調味料の使用量を大さじ・小さじ・mlで表示するツール。

- docs/02_ツール一覧.md **#42**（カテゴリ: 家事・料理、難易度B）。BACKLOG.md P2-T21。
- データ: `data/tables/ajitsuke-ougonhi.json` を単一の情報源（SSOT）とする。比率・出典確度・1人あたりの目安量はすべて同JSONを参照し、本ツール・本仕様書に直接ハードコードしない（可読性のため転記している数値は「データ表・出典」節を参照）。
- YMYL区分: 低（味付けの好みの領域。健康・金額に関わる判断は行わない）。ただし出典確度の誠実な区別（後述）は必須。

## 入力仕様

| フィールド名 | 型 | 必須/任意 | バリデーションルール |
|---|---|---|---|
| 料理の種類（dishKey） | 列挙値 | 必須 | `data.dishes[].key` のいずれか（8種類） |
| 人数（servings） | 数値 | 条件付き必須 | `directMl` が未入力の場合に使用。0より大きい数値であること（0・負値・非数はエラー） |
| 基準となる調味料の分量（directMl） | 数値（ml） | 任意 | 入力された場合、`servings` の代わりにこちらを使う。0より大きい数値であること（0・負値・非数はエラー） |

基準となる調味料（`base_key`）は料理ごとに異なる（例: 煮物は「だし」、照り焼きのたれは「しょうゆ」）。UIは選択中の料理の `base_key` に対応する日本語ラベルを入力欄のヒントに表示する。

## ロジック仕様

### 1. 基準量(ml)の決定

```
if directMl が入力されている:
    baseMl = directMl
else:
    baseMl = dish.default_ml_per_nin * servings   # 1人あたりの目安量 × 人数
```

`default_ml_per_nin` は比率そのものの出典とは別に、本ツールが「人数」入力をわかりやすくするために設定したUX用の初期値（出典なし）である。ユーザーが `directMl` を入力すればこの初期値は使われない。

### 2. 各調味料の量へのスケール計算

```
basePart = dish.parts.find(p => p.key == dish.base_key).part
scaleFactor = baseMl / basePart

for each part in dish.parts:
    ml = part.part * scaleFactor
    tbsp = ml / 15    # 大さじ1杯=15ml
    tsp  = ml / 5     # 小さじ1杯=5ml
```

例: 煮物（だし:しょうゆ:みりん = 8:1:1）で「だし400ml」を基準にすると、scaleFactor = 400/8 = 50。しょうゆ = 1×50 = 50ml（大さじ3.3杯）、みりん = 50ml（大さじ3.3杯）。

### 3. 表示単位の丸め

`formatAmount(ml)` は、15ml以上なら大さじ表記、5ml以上15ml未満なら小さじ表記、5ml未満はml表記を返す（値は小数第1位までの四捨五入）。詳細画面では常に ml・大さじ・小さじの3通りを併記する。

## データ表・出典

`data/tables/ajitsuke-ougonhi.json` より転記。

### 出典が確認できた5品目（confirmed=true）

農林水産省「伝統調味料は『黄金比』で手間いらず」（Let's！和ごはんプロジェクト、大臣官房新事業・食品産業部 外食・食文化課食文化室）
https://www.maff.go.jp/j/keikaku/syokubunka/culture/wagohan/articles/2202/spe6_01.html （`last_verified: 2026-07-17`）

| 料理 | 比率（記事原文） | base_key |
|---|---|---|
| 万能だれ（照り焼き用） | しょうゆ:酒:みりん = 1:1:1 | shoyu |
| 煮魚 | 酒:しょうゆ:みりん = 6:1:1 | sake |
| 煮物 | だし:しょうゆ:みりん = 8:1:1 | dashi |
| めんつゆ | だし:しょうゆ:みりん = 12:1:1 | dashi |
| 万能合わせだれ | みそ:酒:みりん:すりごま:水 = 1:1:1:1:1（大さじ1ずつ） | miso |

`factory/config/source-allowlist.json` に `maff.go.jp`（農林水産省）が「食品の保存・調理に関する一次情報」として許可済みであることを確認した上で採用している。

### 出典が確認できなかった3品目（confirmed=false）

すき焼きの割り下・天つゆ・酢の物（三杯酢）は、上記の農林水産省記事に比率の記載がなく、`source-allowlist.json` の許可ドメイン内に比率を裏付ける一次情報を見つけられなかった。これらは複数の一般レシピサイト（クックパッド・macaroni等。いずれも許可ドメイン外のため出典として明記しない）で比較的よく紹介される比率を「一般的な目安（出典なし）」として採用した。

| 料理 | 採用した目安比率 | base_key | 備考 |
|---|---|---|---|
| すき焼きの割り下 | しょうゆ:みりん:酒:砂糖 = 1:1:1:1 | shoyu | 関東風・関西風など地域や家庭による違いが大きい |
| 天つゆ | だし:しょうゆ:みりん = 4:1:1 | dashi | 「5:1:1」等、他の比率で紹介される例もある |
| 酢の物（三杯酢） | 酢:砂糖:しょうゆ = 3:2:1 | su | 「1:1:1」等、他の比率で紹介される例もある |

これらの数値を農林水産省など公的機関の見解であるかのように断定的に表示しないこと。UI（`components/tools/impl/AjitsukeOugonhi.tsx`）は `dish.confirmed === false` の場合に必ず注意書き（Callout）を表示する。

### 単位定義

大さじ1杯=15ml、小さじ1杯=5ml（`data.tan_i`。一般的な計量スプーンの規格）。

## エッジケース

- **分量（人数 または 直接指定ml）が0**: 入力エラー（「分量は0より大きい数値を入力してください」等）。比率計算はゼロ除算・無意味な結果を避けるため実行しない。
- **分量が負の値**: 入力エラー。
- **分量が非数（空欄・文字列など）**: 入力エラー。
- **人数が極端に大きい値（例: 100人分）**: エラーにはせず、そのままスケールして計算する（大人数の作り置き・イベント用途を想定）。表示桁数はfmt()の丸め処理のみ。
- **料理を切り替えた直後**: 直接指定ml欄をクリアし、人数モードに戻す（前の料理の基準量が残って誤解を招かないようにする）。
- **未知の dishKey が渡された場合**: 入力エラーとして扱う（通常UIからは発生しないが、関数の防御的実装として用意）。
- **confirmed=false の料理を選んだ場合**: 常に注意書き（Callout）を表示し、断定的な表現を避ける。

## YMYL配慮事項

- 味付けの好みは個人差・地域差・家庭差が大きいテーマであり、健康・金額のような厳格な一次情報は必須としないが、確認できた情報とできなかった情報を区別して表示する。
- confirmed=true の5品目についても「農林水産省の記事で紹介されている目安」であることを明記し、「これが正解・唯一の比率である」という誇張した権威づけをしない。
- confirmed=false の3品目は、常時Calloutで「一般的に紹介されている目安（出典なし）」であることを明示する。
- 画面下部に常時、味の好み・地域・家庭による違いがある旨の注意書きを表示する（アレルギー等の健康情報は扱わない）。
