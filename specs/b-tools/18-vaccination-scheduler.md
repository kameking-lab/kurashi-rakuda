# 予防接種スケジューラー（2026年度定期接種、生年月日連動）

## 概要

生年月日（と性別）を入力すると、予防接種法に基づく定期接種（A類疾病）の標準的なスケジュールを、対象ワクチンごとに「対象時期・回数・現在の状態（対象前／対象期間内／対象期間終了）」として一覧表示するツール。

- **対象ペルソナ**: ペルソナ1（佐藤美咲）を中心に、乳幼児〜中高生の子を持つすべての保護者
- **docs/02 の該当番号**: #18（カテゴリ2「子育て」）。需要=高／難易度=S／YMYL=**高**／優先度=S。Phase 2 でPushリマインダー接続予定
- **YMYL区分と設計方針**: 法定接種スケジュールという公的制度データを扱うため、厚生労働省の定期接種制度に忠実であることを最優先する。一方で「受けるべきかどうか」「受けても安全か」という**医学的判断は一切行わない**。本ツールは「制度上、標準的にいつ・何回、どのワクチンが定期接種の対象になるか」という**スケジュール表示に徹する**（docs/03 §4.1「症状・治療・薬は扱わない、医療判断系は作らない」の原則を最も厳格に適用する対象）
- **スコープ外（明示的に扱わない）**: 接種の可否判断、副反応の説明・対応、体調不良時に受けてよいかの判断、任意接種（インフルエンザ・おたふくかぜ等、定期接種でないもの）、ワクチンの銘柄選択（例: どのHib製剤を選ぶか）、同時接種の可否

## 入力仕様

| フィールド名 | 型 | 必須/任意 | バリデーションルール |
|---|---|---|---|
| `birth_date`（生年月日） | 日付（YYYY-MM-DD） | 必須 | 本日以前の日付。本日から起算して0〜19歳（19歳の誕生日前日まで）の範囲。範囲外は「本ツールが対象とする0〜18歳の範囲外です」と表示し、結果は出さない |
| `sex`（性別） | enum: `female` / `male` / `unspecified` | 必須（デフォルト `unspecified`） | HPVワクチン欄の表示切り替えにのみ使用。`male` の場合はHPV欄を「定期接種の対象は女性のみです」と表示。`unspecified` の場合はHPV欄を「対象年齢の方は表示可能です」の中立表示にする |
| `today_override`（基準日、開発・テスト用） | 日付 | 任意（通常は実行日を使用） | テスト・確認用途のみ。本番UIには表示しない |

**設計上の注記**: 本ツールは「過去に実際に接種したワクチン」の入力・記録は Phase 1 では受け付けない。すべてのワクチンについて「制度上の標準スケジュールに対して、生年月日からみて現在どの時期に当たるか」を一律に計算して表示する（実際の接種状況の管理・記録機能はPhase 2のリマインダー機能で検討）。

## ロジック仕様

### 年齢・月齢の算出

```
function calcAge(birth_date, today):
    age_years  = 満年齢（誕生日を迎えているかで判定）
    age_months = (today.year - birth_date.year) * 12 + (today.month - birth_date.month)
                 - (today.day < birth_date.day ? 1 : 0)
    return { age_years, age_months }
```

年齢は**実際の生年月日に基づく暦年齢**のみを用いる。定期接種の対象年齢は法令上「生後◯月に至るまで」「◯歳に至るまで」という実齢基準で定められているため、離乳食ツール（#19）と異なり、早産児であっても修正年齢は用いない（後述エッジケース参照）。

### ワクチンごとのステータス判定

各ワクチン `v` について、§データ表 の `start`（対象開始）・`end`（対象終了）・`standard_start`（標準的接種開始時期）・`doses`（回数と標準的間隔）を用いて判定する。

```
function judgeStatus(v, age_months, age_years, today, birth_date):
    if today < birth_date + v.start:
        return "対象期間前"   # まだ対象年齢に達していない

    if today >= birth_date + v.end:
        return "定期接種の標準的な対象期間を過ぎています"  # 制度上の対象年齢を超過

    return "対象期間内"  # 対象年齢の範囲内（推奨される標準的な時期かどうかは v.standard_start との比較で別途表示）

function nextDoseEstimate(v, birth_date):
    # 各回の目安日 = 出生日 + 標準的接種開始オフセット + (回数-1)ごとの標準的interval の積算
    dose_dates = []
    d = birth_date + v.standard_start
    dose_dates.append(d)
    for i in 2..v.dose_count:
        d = d + v.interval[i]
        dose_dates.append(d)
    return dose_dates  # 「制度上の目安日」として表示。実際の接種間隔は体調・ワクチンの種類により医療機関が個別に決定する
```

- `nextDoseEstimate()` は「標準的にはこの時期が目安」という表示専用の計算であり、実際に接種すべき日を確定するものではない。UI上は「◯回目の目安: 20XX年X月頃」という形式で幅を持たせて表示する
- 体調不良等により接種が遅れた場合の再計算ロジックは実装しない（「最短接種間隔」を保ったまま医療機関が個別に判断する事項のため）。遅れているケースは「まだ完了していない回があります。標準的な時期を過ぎている場合は医療機関にご相談ください」という定型メッセージに留める

### HPVワクチンの分岐

```
function judgeHPV(sex, age_years, age_months):
    if sex == "male":
        return "定期接種の対象は女性のみです（任意接種としての情報は医療機関にご確認ください）"
    if age_years系 が 小学6年相当(標準11歳)〜高校1年相当(標準16歳の学年末) の範囲外:
        return judgeStatus() の通常ロジックで 対象期間前 / 対象期間終了 を表示
    else:
        return "対象期間内"（+ キャッチアップ接種対象者への注記を年齢に応じて表示）
```

学年基準（4月1日時点の年齢で学年を判定する日本の学齢ルール）を用いるため、単純な満年齢の月日境界ではなく「年度（4月1日区切り）」で対象学年を判定する点に注意する。

## エッジケース・注意事項

- **境界値**: 「生後2ヶ月」「1歳に至るまで」等の対象年齢は誕生日・月齢当日を含むか含まないかで解釈が分かれやすいため、本ツールでは「◯ヶ月に達した日（誕生日相当日）を含む」形で統一する（例: 生後2ヶ月の対象開始は、出生日の2ヶ月後の応当日から）
- **早産児**: 定期接種の対象年齢は実齢（暦年齢）基準であり、離乳食ガイド（#19）のような修正月齢は用いない。ただし低出生体重児等では医師の判断で接種開始時期が調整される場合があるため、「低出生体重児・早産児の場合、接種開始時期について医師の指示に従ってください」と注記する
- **体調不良による延期**: 接種予定日に発熱等の体調不良があった場合の延期・スケジュール再調整は、最短接種間隔のルールの範囲内で医療機関が個別に判断する。本ツールは延期後の新しい目安日の自動再計算は行わず、「体調不良で延期した場合は、次回の接種時期について医療機関にご相談ください」という定型文のみ表示する
- **地域差・在庫差**: ワクチンの種類（メーカー）・在庫状況により接種間隔が異なる場合があるため、「実際の接種間隔は使用するワクチンの種類により異なります。予診票・医療機関の説明をご確認ください」と注記する
- **転居・自治体をまたぐ接種歴**: 自治体をまたいだ接種歴の通算・不足分の扱いは自治体窓口・医療機関への相談が必要である旨のみ案内し、判定は行わない
- **キャッチアップ接種等の特例**: 対象年齢に近い場合は「対象になる可能性があります」と幅を持たせ、具体的な制度終了日は断定しない（制度変更リスクが高いため）
- **同時接種**: 複数ワクチンを同日に接種できるかどうかは医療機関・保護者の判断事項であり、本ツールは同日接種の可否・組み合わせについて一切言及しない
- **19歳以上・生年月日超過**: 本ツールの対象範囲外として結果を表示しない（バリデーションエラー）

## YMYL配慮事項

docs/02 の YMYL 区分「高」。docs/03 §4.1 で「症状・治療・薬は扱わない」「医療判断系は作らない」と定める原則の中でも最も慎重な運用が必要な領域として設計する。

- **やらないことの明記**: 「受けるべきかどうか」「安全かどうか」「副反応のリスク」についての判断・助言は一切行わない。あくまで「制度上、標準的にいつ・何回が目安か」という**カレンダー表示**に徹する
- 固定文言案（結果画面の最上部および最下部に常時表示）:
  > 「本ツールは予防接種法に基づく定期接種の標準的なスケジュール（制度上の目安）を、生年月日から自動計算して表示するものです。接種を受けるかどうか、受けられる体調かどうかについての医学的な判断は行っていません。」
  >
  > 「実際の接種スケジュールは、お子さまの体調・体質・これまでの接種歴・使用ワクチンの種類などにより、医師が個別に判断します。母子健康手帳・医療機関から渡される予診票、お住まいの自治体の案内と本ツールの表示が異なる場合は、必ず母子健康手帳・自治体・医療機関の情報を優先してください。」
  >
  > 「接種を受けるかどうか、当日の体調で接種できるかどうかについては、必ずかかりつけ医・接種医療機関にご相談ください。」
- 準拠年度・出典を全画面に必須表示（docs/03 §4.2「最終更新日／準拠年度／出典の3点セット」を厳格適用）
- 数値（対象年齢・回数・間隔）はすべて §データ表 の出典に機械照合可能な形でリポジトリ管理し、出典と食い違う数値が生成された場合はビルドを失敗させる想定（docs/03 §2「CI忠実性ゲート」の対象データとする）
- 制度改定（キャッチアップ接種の対象期間、五種混合の経過措置終了時期など）が起きやすい項目は、断定的な終了日を表示せず「自治体・厚生労働省の最新情報をご確認ください」という誘導に留める
