## 概要

最終月経開始日・平均月経周期日数を入力すると（Q3-03「生理周期・排卵日予測」と同じ入力）、`SeiriShuki.calc.ts` の `calcSeiriShuki` をそのまま呼び出して次回月経開始予定日・排卵予測日・カレンダー（複数周期分の月経開始予定日）を算出し、そこから黄体期間（排卵後〜次回月経前）と、PMS（月経前症候群）の定義（日本産婦人科学会「月経前3〜10日の間続く…症状」）に基づく「症状が出やすいとされる期間の目安」を導出してカレンダー表示するツール。

**記録機能なし**: localStorage等への入力値・結果の保存は一切行わない。毎回その場で入力→即時計算のみ（他のツールと同様、送信・保存はしない設計を踏襲）。

**周期計算ロジックは独自実装しない**: 次回月経開始予定日・排卵予測日（黄体期14日固定モデル）・複数周期分のカレンダーは `components/tools/impl/SeiriShuki.calc.ts` の `calcSeiriShuki` 関数と `CYCLE_LENGTH_MIN/MAX`・`LUTEAL_PHASE_DAYS` 等の定数をそのまま呼び出す。本ツールが追加するのは、その結果（各周期の月経開始予定日）から黄体期間とPMS症状の目安期間を導出する薄い後処理レイヤーのみである。

データのSSOTは `data/tables/pms-yosoku-shikumi.json`（PMSの定義・有病率=日本産婦人科学会の定義を厚労省・厚労省事業サイトが引用したもの、黄体期の定義=独立行政法人日本スポーツ振興センター）。数値（PMS発症の目安日数3〜10日・有病率2〜10%）はすべて同JSONを参照し、本ツール・本仕様書に直接ハードコードしない。

## 入力仕様

Q3-03（生理周期・排卵日予測）と同一の入力を用いる（`SeiriShuki.calc.ts` の `SeiriShukiInput` をそのまま利用するため）。

| フィールド名 | 型 | 必須/任意 | バリデーションルール |
|---|---|---|---|
| `lmp`（最終月経開始日） | 日付（YYYY-MM-DD） | 必須 | 当日以前の日付のみ有効。基準日より90日以上前はエラー（`calcSeiriShuki` の `LMP_TOO_OLD` をそのまま使用） |
| `cycleLength`（平均月経周期日数） | 整数（日） | 任意（デフォルト28） | `CYCLE_LENGTH_MIN`（20）〜`CYCLE_LENGTH_MAX`（45）の範囲。範囲外は `CYCLE_LENGTH_OUT_OF_RANGE` |
| `baseDate`（基準日） | 日付（YYYY-MM-DD） | 任意（デフォルト＝実行時の当日） | `lmp` より前はエラー |
| `calendarCount`（カレンダー表示する周期数） | 整数 | 任意（デフォルト3、1〜6にクランプ） | `calcSeiriShuki` が内部でクランプするため本ツール側での追加バリデーションはしない |

本ツールのUIでは `periodLength`（月経が続く日数）は入力させない（PMS症状が出やすい時期の算出に使わないため）。`calcSeiriShuki` へはデフォルト値（5日）のまま渡す。

## ロジック仕様

### 1. 次回月経開始予定日・排卵予測日・複数周期カレンダー（SeiriShuki.calc.tsの再利用）

```
seiriResult = calcSeiriShuki({ lmp, cycleLength, baseDate, calendarCount })
```

`seiriResult.ok === false` の場合は、そのままエラー（`error`・`message`）を呼び出し元に返す（本ツール独自のエラー種別は追加しない）。`seiriResult.calendar`（`calcSeiriShuki` が算出する n=1,2,3...の月経開始予定日の配列）を、以降の黄体期間・PMS目安期間の算出の基点として使う。

### 2. 黄体期間の算出（各周期ごと）

`data/tables/pms-yosoku-shikumi.json` の `menstrualCycle.lutealPhase.implementationNote`（「排卵日を推定する場合は『次回月経開始予定日 − 約14日』とするのが定義に沿う」）に基づき、`calcSeiriShuki` が使う黄体期定数 `LUTEAL_PHASE_DAYS`（=14。SeiriShuki.calc.tsからそのままimport）を用いて、カレンダー配列の各月経開始予定日 `periodDate` から次のように算出する。

```
ovulationDate（黄体期の開始）  = periodDate − LUTEAL_PHASE_DAYS日
lutealPhaseEnd（黄体期の終了） = periodDate − 1日   // 月経開始前日まで
```

n=1周期目については `calcSeiriShuki` が返す `ovulationDate` と一致する（同じ定数・同じ起点日付から算出するため）。n=2以降は `calcSeiriShuki` が排卵日を返さないため、本ツールが同じ定数を各カレンダー日付に適用して補完する（周期計算そのものの再実装ではなく、既存定数の適用範囲を広げる後処理）。

### 3. PMS症状が出やすい時期の目安（黄体期後半）の算出

`data/tables/pms-yosoku-shikumi.json` の `pmsDefinition.implementationNote`（★最重要★「PMSの症状が出る期間は『次回月経開始予定日の3〜10日前』であり、月経開始とともに軽快・消失する」）に基づき、黄体期間のうち後半（月経開始前3〜10日）を「PMS症状が出やすいとされる時期の目安」として次のように算出する。

```
pmsWindowStart = periodDate − pmsDefinition.onsetBeforeMenstruationDaysTo日   // 10日前
pmsWindowEnd   = periodDate − pmsDefinition.onsetBeforeMenstruationDaysFrom日 // 3日前
```

`onsetBeforeMenstruationDaysFrom`（3）・`onsetBeforeMenstruationDaysTo`（10）は `pms-yosoku-shikumi.json` からそのまま読み、数値をハードコードしない。この期間は黄体期間（`ovulationDate`〜`lutealPhaseEnd`）の部分集合であり、黄体期間全体を「PMSが起こりうる期間」、その後半3〜10日を「症状が出やすいとされる期間の目安」として画面上で区別して表示する。

### 4. 日付演算の実装

`SeiriShuki.calc.ts` は日付シフト用の内部ヘルパー（`addDays`・`formatDate`等）を外部にexportしていないため、本ツールは `calcSeiriShuki` が返した検証済みのISO日付文字列（`YYYY-MM-DD`）に対して固定日数を加減するだけの小さな日付ヘルパーを自前で持つ（UTC正午固定で解釈し、月末・年またぎ・うるう年も暦日数の実加算で自動的に処理される点は `SeiriShuki.calc.ts` と同じ方式）。**これは周期予測ロジックの再実装ではなく、既に確定した日付に対する単純なオフセット計算である。**

## エッジケース

- **`lmp`・`cycleLength`・`baseDate` の入力エラー**: `calcSeiriShuki` のエラー（`INVALID_LMP`・`INVALID_BASE_DATE`・`BASE_BEFORE_LMP`・`LMP_TOO_OLD`・`CYCLE_LENGTH_OUT_OF_RANGE`）をそのまま呼び出し元に返し、本ツール独自の計算・表示は一切行わない（NaN表示・無言失敗を避ける）。
- **月末日をまたぐ場合（例: 月経開始予定日が月末付近）**: 暦日数の実加算で計算するため自動的に処理される（例: 次回月経開始予定日2026-02-28 → PMS目安期間は2026-02-18〜2026-02-25）。
- **うるう年をまたぐ場合**: 追加補正は不要（例: 次回月経開始予定日2028-02-29 → PMS目安期間は2028-02-19〜2028-02-26）。
- **年をまたぐ場合**: 単純な日数減算で自動的に年をまたぐ（例: 次回月経開始予定日2027-01-17 → PMS目安期間は2027-01-07〜2027-01-14）。
- **`calendarCount` を指定した場合（複数周期表示）**: n=2以降の各周期についても同じ`LUTEAL_PHASE_DAYS`・PMS目安日数を適用し、黄体期間・PMS目安期間を算出する。
- **周期が不規則な人**: 本ツールは直近1周期分の平均値のみを使う単純予測であり、複数周期の記録に基づく予測ではないことを注記する（記録機能はスコープ外）。
- **PMSがない人・症状に個人差がある人**: PMSの有病率は2〜10%であり、黄体期間・PMS目安期間を表示してもすべての人に症状が出るわけではないことを常時明記する。

## YMYL配慮事項（医療判断なしの明記）

- **診断・重症度判定は一切行わない**: 本ツールは「この時期は症状が出やすいと言われている」という一般情報（日付演算による目安の提示）にとどめ、実際に症状が出るかどうか・その重さ・PMSかどうかの診断は一切行わない。ユーザーの入力から症状の有無・程度を尋ねる項目も持たない。
- **「PMSがある人にとっての目安」であることの明記**: PMSの有病率（月経がある女性の2〜10%。`w-health.jp`出典）を結果画面に明記し、「黄体期間・PMS目安期間を表示しても、全員に症状が当てはまるわけではない」旨を常時表示する。
- **PMDDへの言及は呼称のみ**: 症状が重い場合はPMSの重症型である月経前不快気分障害（PMDD）の可能性がある旨を触れるが、診断基準は一次情報で確認できていないため記載しない（`pms-yosoku-shikumi.json` の `pmdd.diagnosticCriteria` は `null`）。
- **受診の目安は定性的な表現のみ**: 「月経前後の体調不良が続く場合は産婦人科にご相談ください」という定性的な文言にとどめ、スコアリングによる「受診すべき／不要」の判定は行わない（`consultationGuidance.numericThreshold` が一次情報になく `null` のため）。
- **社会的インパクトの数値（推定1300万人・63%等）は不採用**: 出典が民間調査であるため、記事・ツールいずれにも掲載しない（`socialImpact` の注記どおり）。
- **固定文言**: 結果画面に、次の趣旨のCalloutを常時表示する。「これは黄体期・PMSの定義に基づく日付計算上の目安であり、診断ではありません。PMSは月経がある女性の2〜10%に起こるとされ、症状の出方には個人差があります。月経前後の体調不良が続く場合や、日常生活に支障がある場合は、自己判断せず産婦人科にご相談ください。」
- **避妊・妊活目的での利用への注意**: 排卵予測日を扱うため、Q3-03と同様に避妊・妊活の判断に用いない旨を明記する。
