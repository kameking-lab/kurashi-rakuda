# 保活スケジュールメーカー（自治体申込期から逆算）

## 概要

入園希望年月を入力すると、認可保育所（保育所・認定こども園等）への入園に向けて「いつ・何をすべきか」を月単位で逆算して一覧表示するスケジュール生成ツール。

- **対象ペルソナ**: ペルソナ1（佐藤美咲・専業主婦、下の子の保育を検討）／ペルソナ2（田中絵里・時短勤務、2人目の保活を見据える）
- **docs/02 の該当番号**: #23（カテゴリ2「子育て」）。需要=中／難易度=A／YMYL=低／優先度=S
- **性格**: 計算ツールではなく「段取り・チェックリスト系」（docs/03 §3の3類型のうち「段取り型」に対応する非計算ツール）。保育料や入園可否の判定は行わない
- **kurashi-keisan との関係**: 重複なし。汎用の日付逆算ではなく「保活」という文脈に特化したマイルストーン設計が独自価値
- **季節性**: 一次申込みが集中する秋（10〜11月）に検索需要が伸びる季節ツール。前年の情報収集期（春〜夏）から通年で流入を狙う

## 入力仕様

| フィールド名 | 型 | 必須/任意 | バリデーションルール |
|---|---|---|---|
| `target_month`（入園希望年月） | 年月（YYYY-MM、月初1日で内部保持） | 必須 | 本日（実行日）が属する月から起算して +1ヶ月〜+18ヶ月の範囲。範囲外はエラーメッセージ「入園希望月は1〜18ヶ月先の範囲で指定してください」を表示 |
| `entry_type`（申込み区分） | enum: `april`（4月一斉入園）／`midyear`（年度途中・随時入園） | 必須（デフォルト `april`） | `target_month` が4月以外の場合は自動的に `midyear` を初期選択（ユーザーは変更可） |
| `municipality_deadline`（お住まいの自治体の公表締切日） | 日付（YYYY-MM-DD） | 任意 | 入力された場合、`target_month` より前の日付であること。未入力時は §データ表 の全国目安値で自動計算 |
| `facility_type`（施設種別） | enum: `ninka`（認可保育所・認定こども園）／`other`（認可外・企業主導型など） | 任意（デフォルト `ninka`） | `other` 選択時は「本ツールの逆算ロジックは主に認可施設を想定しています。認可外・企業主導型は施設ごとに申込方法が異なるため、各施設に直接ご確認ください」という注記を表示するのみで、計算ロジックは変えない（Phase 1 では認可施設のみ対応） |
| `has_sibling`（きょうだいが同一施設に在園中か） | boolean | 任意（デフォルト false） | true の場合、「きょうだい同時入所・優先利用調整の扱いは自治体ごとに異なります。詳細はお住まいの自治体にご確認ください」という注記のみ表示（優先順位の計算はスコープ外） |

## ロジック仕様

### 基本方針
「入園希望月（E）」を起点に、月単位のオフセットでマイルストーンを逆算する。日付が自治体ごとに大きく異なる項目（一次申込み締切等）は、`municipality_deadline` が入力されていればそれを正とし、未入力なら全国的に多い傾向値（目安）を用いる。すべての生成日付には「目安」ラベルを付与し、断定表現を避ける。

### 疑似コード

```
function generateSchedule(target_month E, entry_type, municipality_deadline?):
    E_date = E の1日 (例: 2027-04 → 2027-04-01)

    if entry_type == "april":
        # 4月一斉入園モデル（表A参照）
        if municipality_deadline is provided:
            D_primary = municipality_deadline          # 一次申込み締切（実測値）
        else:
            D_primary = E_date - 6ヶ月, その月の25日   # 目安: 10月下旬相当（表A row2）

        milestones = [
            { name: "情報収集・希望園リストアップ開始", date: E_date - 12ヶ月 },
            { name: "保育園見学の受付・見学開始",       date: E_date - 10ヶ月 },
            { name: "見学の目安終了",                   date: E_date - 7ヶ月 },
            { name: "申込書類の準備開始（就労証明書等）", date: D_primary - 1ヶ月 },
            { name: "一次申込み締切",                   date: D_primary },
            { name: "一次選考結果通知",                 date: D_primary + 2ヶ月, 上限 = E_date - 3ヶ月 },
            { name: "（落選時）二次申込み締切",          date: E_date - 3ヶ月, その月の15日 },
            { name: "（落選時）二次選考結果通知",        date: E_date - 2ヶ月, その月の15日 },
            { name: "入園前健診・面談・書類提出",        date: E_date - 1ヶ月 },
            { name: "入園",                              date: E_date },
        ]
    else: # "midyear" 随時入園モデル（表A参照）
        milestones = [
            { name: "情報収集・希望園リストアップ開始", date: E_date - 3ヶ月 },
            { name: "保育園見学",                       date: E_date - 2ヶ月 },
            { name: "空き状況の自治体窓口への確認・申込書類準備", date: E_date - 2ヶ月, その月の15日 },
            { name: "申込み締切（目安）",                date: E_date - 1ヶ月, その月の10日 },
            { name: "選考結果通知（目安）",               date: E_date - 1ヶ月, その月の25日 },
            { name: "入園前健診・面談",                   date: E_date - 2週間 },
            { name: "入園",                               date: E_date },
        ]

    # 過去日フィルタ: 実行日(today)より前のマイルストーンには
    # ステータス "既に目安時期を過ぎています" を付与して赤字表示（削除はしない）
    for m in milestones:
        m.status = (m.date < today) ? "過去" : "今後"

    # 直近アラート: E_date までの残り月数が3ヶ月未満、かつ entry_type == "april" の場合、
    # 「一次・二次申込みの標準的な時期を過ぎている可能性が高いため、
    #   お住まいの自治体の保育課へ直接、二次募集・随時募集の有無を問い合わせてください」
    # という警告を先頭に表示する
    if entry_type == "april" and (E_date - today) < 3ヶ月:
        show_warning("残り期間が短いため自治体へ直接確認してください")

    return milestones
```

### 日付の丸めルール
- 「◯ヶ月前」は暦月で計算する（例: 2027-04-01 の6ヶ月前 = 2026-10-01）。月末日を跨ぐ入力（例: 1月31日）は当月末に丸める
- 代表日（25日・15日・10日等）は §データ表 の「代表日」列を採用し、範囲表記（例: 10月下旬〜11月上旬）は結果画面ではその範囲もあわせて表示する

## 出力仕様

- ページ上部: 入力内容のサマリー（入園希望年月／申込み区分／自治体締切の有無）
- マイルストーンタイムライン（カード形式、時系列順）: 各カードに「マイルストーン名」「目安時期（年月または年月日、範囲表記があれば併記）」「やることの一言説明」「ステータス（今後／過去）」
- 直近アラート（該当時のみ、赤系の警告ボックスで最上部に表示）
- 固定注記ブロック（下部）: 「このスケジュールは全国の傾向から算出した目安です。締切日・選考基準はお住まいの市区町村により異なります。正式な情報は必ず自治体の保育課・公式サイトでご確認ください」
- チェックリストのCSV/印刷用ビュー（将来拡張、Phase 1 では画面表示のみで可）

## エッジケース・注意事項

- **境界値**: `target_month` が実行月の翌月（+1ヶ月）の場合、多くのマイルストーンが既に過去日になる。過去日は削除せず「過去」ラベルで表示し続けることで「間に合わなかった項目」を可視化する
- **地域差**: 一次申込み締切は自治体により9月〜12月まで幅がある。`municipality_deadline` 入力を強く推奨する導線（入力すると精度が上がる旨のヒント表示）を設ける
- **年度またぎの随時入園**: 1〜3月入園希望の場合、年度切り替え（4月）をまたぐ運用（新年度分の申込みに切り替わる等）がある自治体が存在するため、「1〜3月入園をご希望の場合は年度の切り替え時期の運用が自治体により異なります」と注記する
- **育休延長との関係**: 保育園に入れなかった場合の育休延長手続き（ハローワークへの証明書提出等）は本ツールのスコープ外。関連記事へのリンクのみ設置し、給付金の計算等は行わない
- **きょうだい同時申込み・転園**: 優先順位のロジックは自治体ごとに異なり複雑なため計算対象外。注記のみ
- **0歳児クラス vs 1歳児クラス**: 入園希望月時点の子の年齢によってクラス枠の空き状況（0歳児クラスは激戦・1歳児クラスはさらに激戦等）が大きく異なるが、本ツールは日程のみを扱い、当落可能性の予測は行わない
- **認可外・企業主導型保育**: 申込み方式が施設ごとに異なるため、本ツールの逆算ロジックの対象外（注記のみ表示）

## YMYL配慮事項

本ツールは docs/02 の YMYL 区分「低」（日付・段取り情報であり、お金・医療の判断を伴わない）。ただし「入園できるか否か」は家庭の生活設計に直結するため、断定・保証と受け取られない表現に配慮する。

- **医学的判断は行わない**旨の明記は不要（該当しないため）だが、「制度上の判断（当落・優先順位）」を行わない旨は明記する
- 固定文言案（結果画面下部に常時表示）:
  > 「このスケジュールは全国の傾向をもとに算出した目安であり、入園の可否・選考結果を保証するものではありません。締切日・必要書類・選考基準は市区町村ごとに異なります。必ずお住まいの自治体の保育担当窓口・公式サイトで最新情報をご確認ください。」
- 「二次募集」「随時募集」など自治体により実施有無が異なる制度については「実施していない自治体もあります」と併記する
