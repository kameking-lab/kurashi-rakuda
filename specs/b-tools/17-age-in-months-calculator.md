# 月齢計算（生後日数・週数・修正月齢対応）

## 概要

生年月日と基準日（通常は「今日」）から、赤ちゃんの生後日数・生後週数・月齢（暦月ベース）を計算するツール。出産予定日を入力すると、早産児向けの「修正月齢」も併せて計算する。

- 対象ペルソナ: ペルソナ1（佐藤美咲・4歳/1歳の子育て中）を主対象に、ペルソナ4（鈴木奈々・妊活〜プレママ、早産の可能性がある層）の修正月齢ニーズもカバーする。
- docs/02_ツール一覧.md **#17**（カテゴリ2: 子育て、優先度 **S**、難易度 B）。「SERP最薄の一丁目一番地」と位置付けられ、子育てカテゴリ全ツール（#31 誕生日カウント等）のハブとなる想定。
- YMYL区分: **低**（日付演算のみで医学的判断を含まない。ただし修正月齢は早産児の発達評価に使われる文脈があるため、医療判断ではなく「参考値」であることを明記する）。

## 入力仕様

| フィールド名 | 型 | 必須/任意 | バリデーションルール |
|---|---|---|---|
| 生年月日（birthDate） | 日付（YYYY-MM-DD） | 必須 | 基準日以前の日付であること。西暦1900-01-01〜基準日の範囲。未入力・不正形式はエラー |
| 基準日（baseDate） | 日付（YYYY-MM-DD） | 任意（未入力時は端末の今日の日付をデフォルト採用） | 生年月日以降であること。生年月日から起算して過去120年を超える未来日は警告表示（通常利用では発生しない想定） |
| 出産予定日（dueDate） | 日付（YYYY-MM-DD） | 任意（早産児の修正月齢を出したい場合のみ入力） | 生年月日以降の日付であること（出産予定日は実際の出生日と同じか後）。生年月日から+70日（約10週）を超える極端な差がある場合は「入力値をご確認ください」の注意表示（誤入力防止のソフトバリデーション、エラーにはしない） |

## ロジック仕様

すべて「実在の暦（グレゴリオ暦）」上の日付演算とする。ブラックボックス化を避けるため、計算式を結果画面にも表示する。

### 1. 生後日数・生後週数

```
生後日数 = 基準日 − 生年月日   （日数、単純な暦日差）
生後週数 = floor(生後日数 / 7)
端数日数 = 生後日数 mod 7
```

### 2. 月齢（暦月ベース、応当日方式）

「◯か月」は「生まれた日と同じ日（応当日）を迎えるたびに1ヶ月」という暦月ベースの数え方を採用する。応当日が存在しない月（例: 1/31生まれの2月）は、その月の末日を応当日とみなす（月末クランプ方式）。

```
function addMonths(date, n):
    idx = date.year * 12 + (date.month - 1) + n
    y = idx // 12
    m = (idx % 12) + 1
    d = min(date.day, daysInMonth(y, m))
    return Date(y, m, d)

function calcAgeInMonths(birthDate, baseDate):
    # 前提: baseDate >= birthDate
    months = 0
    while addMonths(birthDate, months + 1) <= baseDate:
        months += 1
    remainderDays = baseDate − addMonths(birthDate, months)   # 日数
    return (months, remainderDays)
```

うるう年の2/29生まれの場合も上記 `daysInMonth` により自動的に「非うるう年の2月は28日までしかない」ことが反映され、2/29生まれの子は非うるう年には2/28を応当日として扱う。

### 3. 修正月齢（早産児向け、出産予定日が入力された場合のみ）

修正月齢は「生年月日」の代わりに「出産予定日」を起点にして、同じ `calcAgeInMonths` ロジックを適用する。

```
if dueDate が未入力:
    修正月齢は表示しない
elif baseDate >= dueDate:
    (修正月齢, 修正月齢の端数日数) = calcAgeInMonths(dueDate, baseDate)
else:
    # 出産予定日にまだ達していない（基準日が予定日より前）
    残り日数 = dueDate − baseDate
    修正月齢は「予定日前」として、"修正月齢はまだ0か月未満（予定日まであと{残り日数}日）" を表示
```

早産日数（実齢と修正月齢の差の目安として画面に併記）:
```
早産日数 = dueDate − birthDate   （dueDate >= birthDateの場合のみ正の値）
```

## データ表

暦日ベースの考え方に必要な固定データは以下のみ。出典: グレゴリオ暦のうるう年規則（内閣府・国立天文台 暦要項が典拠とする一般規則）。

| 規則 | 内容 |
|---|---|
| うるう年判定 | 西暦年が4で割り切れる年はうるう年。ただし100で割り切れる年はうるう年ではない。ただし400で割り切れる年はうるう年 |
| 2月の日数 | 平年28日／うるう年29日 |
| 大の月（31日） | 1, 3, 5, 7, 8, 10, 12月 |
| 小の月（30日） | 4, 6, 9, 11月 |
| 月末クランプ規則 | 応当日が存在しない月（例: 4/31, 2/30等）は、その月の末日を応当日とみなす |

## エッジケース・注意事項

- **生年月日が基準日より未来**: 入力エラーとして受け付けない（「生年月日は基準日より前の日付にしてください」）。
- **うるう年2/29生まれ**: 非うるう年の該当月は2/28を応当日とみなす（上記月末クランプ規則に準拠、特別扱いの例外コードは書かず月末クランプの一般規則で自然に処理する）。
- **月末生まれ（1/31, 3/31等）の月またぎ**: 例えば1/31生まれは2ヶ月時点の応当日が2/28（または29）になり、3ヶ月時点の応当日は3/31に戻る。日数の凸凹が発生するのは暦月ベース計算の性質上正常な挙動であり、バグではない。
- **早産児の修正月齢の考え方**: 修正月齢は「出産予定日を起点に数えた月齢」であり、実際の発達評価（乳幼児健診等）では医療者が個別に判断する。本ツールは目安の日付演算のみを提供し、発達の医学的評価は行わない。一般に修正月齢は生後1歳6か月頃までの発達の目安として使われることが多いが、いつまで使うかは医療者の判断によるため、本ツールは年齢上限を設けずに常に計算結果を表示する（利用者の判断に委ねる）。
- **多胎児（双子・三つ子等）**: 本ツールは1人分の計算のみを行う。多胎の場合は子どもごとに個別入力してもらう想定（同一生年月日を複数回入力すれば同じ結果になる）。
- **基準日省略時の挙動**: 未入力の場合は表示側のシステム日付（今日）を採用する。SSR/SSGで生成する場合はキャッシュにより日付がずれないよう、クライアントサイドで当日日付を取得して再計算する。
- **入力範囲外**: 生年月日が極端に古い場合（120年超前など）は本ツールの用途外である旨を表示する（エラーにはしないが、育児向けツールである旨の案内文を出す）。

## YMYL配慮事項

- 本ツールは暦日演算のみを行い、医学的な発達判断・健康診断の代替にはならない旨を固定文言として結果画面下部に表示する。
- 固定文言案: 「この結果は生年月日・出産予定日から計算した目安です。お子さまの発達の評価は乳幼児健診等で医師・保健師にご確認ください。」
- 修正月齢について特に強調する固定文言案: 「修正月齢は早産で生まれたお子さまの発達を考える際の目安の一つとして使われる考え方です。実際の発達評価は医療者にご相談ください。」
- 最終更新日／準拠年度／出典（グレゴリオ暦の一般規則。制度改定の概念がないため出典は暦法の一般知識である旨を明記）の3点セットを表示する。
