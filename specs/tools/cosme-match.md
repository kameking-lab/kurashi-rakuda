# 化粧品「実名・超公平」レコメンド（cosme-match）仕様

作成: 2026-07-20 司令塔（フェイブル）。社長決裁: 実名レコメンド実施（スポンサーなし・超公平）。
実装はオーパス次波（P4-T04）。データ収集はソネット（本仕様のスキーマ準拠）。

## 0. 公平性は「約束」ではなく「仕組み」で担保する

1. **選定基準の完全公開**: 結果は「肌質 × 悩み × 価格帯 × 入手性」のマトリクスに合致する商品の**機械的列挙**（合致数と並び順ルールをページ内に明記。並びは価格昇順→五十音。編集部イチオシ・PR枠・特別扱いの仕組みをコードに持たない）
2. **アフィリエイト全面禁止**: 商品名はテキストのみ。メーカー公式へのリンクも張らない（rel 属性の議論以前に導線ゼロ）。この機能に限り「収益導線ゼロ」を /policy に明文化し、brand-lint の収益タグ検知（DIAG-A8）の対象に data/cosme/ と本ツール UI を含める
3. **「向かない人」「注意点」の必須併記**: データスキーマ上、cautions（注意点）と notSuitedFor（向かない人）が空の商品はビルドエラー（賞賛のみの紹介を機械的に不可能にする）。加えて表示文言の検査（次項）
4. **賞賛のみ検知ゲート**: check:cosme-fairness（新設 lint）: ①全商品に notSuitedFor ≥ 1・cautions ≥ 1 ②禁止語（「神コスメ」「絶対」「一番おすすめ」「これ1本でOK」等の断定・最上級）③同一ブランドが結果上位を独占しない構造か（マトリクス列挙である限り自然に満たすが、データ偏りを警告: 1ブランドが全収録の 30% 超で warn）

## 1. YMYL 線引き

- 成分×肌質の適合は**公的・学会情報の一般論まで**: 日本皮膚科学会（ガイドライン・Q&A）、日本香粧品学会、消費者庁（化粧品の表示・広告）、厚生労働省（薬機法上の効能範囲・医薬部外品の有効成分）。個別商品の「効果」は謳わず、「保湿成分◯◯配合」という**事実記載**と「乾燥肌向けとされる成分」という**一般論**を分離して表示
- 効果の断定・治療効果の示唆は禁止（薬機法の化粧品効能 56 項目の範囲内の表現のみ）。ニキビ・皮膚炎など疾患が疑われる悩みには結果より先に皮膚科受診の案内を表示
- 敏感肌選択時は常時注意: 「新しい化粧品は少量から。異常が出たら使用を中止し皮膚科へ」

## 2. データ構造（data/cosme/。data/seido と分離した新系統）

- `data/cosme/schema.json` を新設（seido-common とは別。verify-seido の対象外だが check-freshness の nextCheckDue 監査対象に**含める**）
- 商品 1 件 = 1 ファイル `data/cosme/products/<brand>-<slug>.json`:
  ```json
  {
    "id": "brand-product",
    "name": "（正式商品名）",
    "brand": "（ブランド名）",
    "category": "kesho-mizu | nyueki | cream | senganryo | hiyakedome",
    "priceYen": 1980, "volume": "200mL",
    "priceBand": "petit | middle | high",
    "availability": ["drugstore", "online", "department"],
    "keyIngredients": [{ "name": "セラミドNP", "role": "hoshitsu", "sourceNote": "全成分表示より" }],
    "skinTypeFit": { "dry": true, "oily": false, "combination": true, "sensitive": null },
    "concernsFit": ["kansou", "keana"],
    "notSuitedFor": ["エタノールに敏感な方（配合上位）"],
    "cautions": ["医薬部外品ではない", "香料あり"],
    "sources": [{ "label": "メーカー公式 商品ページ（全成分・価格）", "url": "https://..." }],
    "checkedAt": "2026-07-20", "nextCheckDue": "2026-10-20",
    "status": "on-sale | renewal-pending | discontinued"
  }
  ```
- `data/cosme/ingredient-skin-map.json`: 成分×肌質の一般論の根拠表（各行に公的・学会出典必須。ここだけが「適合」判定の根拠。商品ファイルの skinTypeFit はこの表から導出し、恣意的な手書き true を lint で禁止 = keyIngredients から導出できない fit はエラー）
- 出典規約: 商品情報は**メーカー公式サイトのみ**（小売・レビューサイト禁止）。source-allowlist にブランド公式ドメインを追記する運用（追記 PR には公式である根拠を記載）

## 3. 鮮度管理

- nextCheckDue 90 日（価格・リニューアル・廃番の周期に合わせる）。check-freshness が超過を検知して Issue 起票（既存機構に相乗り）
- status=discontinued は結果から自動除外（データは履歴として残す）。renewal-pending は「リニューアル情報あり・店頭在庫と成分が異なる場合があります」を併記

## 4. 入力 UI（ツール本体）

1. 肌質: 乾燥 / 脂性 / 混合 / 敏感（「わからない」→ 簡易 4 問診断: 洗顔後のつっぱり・T ゾーンのテカリ・季節差・刺激反応。診断ロジックは一般論出典付き）
2. 悩み: 崩れ / 乾燥 / 毛穴（複数可）
3. 価格帯: プチプラ（〜2,000 円）/ ミドル（〜5,000 円）/ デパコス（5,000 円〜）/ 指定なし
4. 結果: 合致商品を「合致した基準の内訳」付きで列挙（例: 乾燥肌◯=セラミド配合 / 価格帯◯ / ドラッグストアで買える）。各カードに notSuitedFor と cautions を折りたたまず常時表示。0 件のときは正直に 0 件と表示し、未解決クエリ蓄積（specs/ai/03）へ任意送信ボタン
- 入力はすべて端末内。localStorage 保存もしない（肌質は健康関連情報のため保存自体を避ける。診断のたびに選び直す軽さを優先）

## 5. 初期収録スコープと完了条件

- 初期スコープ: 洗顔料・化粧水・乳液/クリーム・日焼け止めの 4 カテゴリ × 各 10〜15 商品（プチプラ中心。ドラッグストア入手性を優先）
- 完了条件: check:cosme-fairness green ／ 全商品にメーカー公式出典 ／ ingredient-skin-map 全行に公的・学会出典 ／ 敏感肌フローの皮膚科誘導表示 ／ 「選定基準の完全公開」セクションがツールページに常設 ／ 検収は司令塔（実装者≠検収者）

## 6. 社長決裁事項

1. /policy への「化粧品紹介は収益導線ゼロ・選定基準公開・機械的列挙」明文化の文面
2. 初期収録ブランドの除外希望の有無（利害関係の宣言。なければ「なし」と記録）
