## 0. このキューの使い方

1 自治体 = 1 タスク。相互依存なしのため**全並列で実行可能**。1 エージェント 3 自治体程度が実績値（所要 8〜13 分／3自治体）。
着手時は §3 の収集手順をそのまま作業指示に貼る。完了したら §1 の表の「状態」を更新する。

**絶対ルール**
- 一次情報（当該自治体の公式サイト）のみ。まとめサイト・保活情報サイト・民間ブログの引用は**禁止**（`scripts/verify-seido.mjs` がドメイン許可リストで機械的に弾く）。
- **推測で数値を埋めない。** 取得できなければ `note` に「未確認: 理由」と明記する。階層を捏造しない。
- 令和8年度版が未公表なら `fiscalYear: 2025` とし `amendments` に `status: "under-review"` で明記する。**令和8年度と偽らない。**

## 2. 収集済みデータから分かった実装上の論点

★ツール実装者（S級ツール #21 保育料計算全国版）は必ず読むこと★

### 2.1 「所得割額をそのまま入力させてはいけない」

階層判定に使う所得割額の前処理が**自治体ごとに違う**。ユーザーが課税明細書の数字をそのまま入れると誤判定する。

| 自治体 | 前処理 |
|---|---|
| 大阪市 | 税額控除前所得割額 **× 6/8**（政令市の税率調整） |
| 名古屋市 | **税源移譲前の税率**で算定した額 |
| 横浜市 | 調整控除額・所得割の調整措置の額**のみ**を控除した額 |
| 札幌市・福岡市・川崎市 | 政令市のため**旧税率6%相当**に換算 |

また、**適用しない控除**も自治体差がある（住宅借入金等特別税額控除・寄附金税額控除・配当控除が典型）。川崎市のみ**定額減税は適用後の金額**で算定する点が他と逆。

→ **設計判断**: 所得割額の直接入力は上級者向けオプションに留め、既定は年収・家族構成からの推計とする。推計値である旨を必ず明示する。

### 2.2 「A/B/C階層は所得割額では判別できない」

生活保護世帯（A）・住民税非課税世帯（B）・所得割非課税世帯（C）はいずれも所得割額0円のため、金額だけでは区別できない。`isWelfare` / `isNonTaxable` フラグと均等割の課税有無で分岐する必要がある。
収集済みデータでは自治体により `incomeMax` の表現が割れている（札幌C1は均等割のみ世帯を含み `incomeMax: 48600`、福岡C1・川崎C1は `incomeMin: 0, incomeMax: 1`）。**この不統一はスキーマ側で吸収せず、ツール側で `isNonTaxable` / `isWelfare` を先に見る実装にすること。**

### 2.3 「無償化により階層表が消えつつある」

東京23区は第1子まで無償化され、練馬区は階層表そのものを廃止（ページは404）。大阪市も令和8年9月から全員無償化。
→ **設計判断**: 「階層表がある」前提の設計にしない。`tiers` が1行のみの自治体、全額0円の自治体を正常系として扱う。無償の自治体では「保育料は0円です。かかるのは給食費・延長保育料です」と表示し、**給食費・延長保育料の案内に価値をシフトする**（世田谷・練馬は給食費も無償という差別化要素がある）。

### 2.4 「施設種別の軸がスキーマにない」

川崎市は施設種別ごとに料金列が3系統（認可保育所等／小規模B型等／家庭的保育等）ある。現スキーマには施設種別の軸がないため**認可保育所列のみ収録**し、`scope.excludes` に明記している。
→ **残タスク**: 小規模保育・家庭的保育の料金差が大きい自治体が増えるなら、スキーマに `facilityType` 軸を追加するか検討する。MVP では認可保育所のみで割り切る。

### 2.5 「0歳児と1・2歳児で金額が違う自治体がある」

大田区・杉並区は原典が0歳児クラスと1・2歳児クラスを別区分にしている。現スキーマの `under3` は1区分のため統合している（現在いずれも0円のため情報欠落なし）。
→ **残タスク**: 有料の自治体で0歳／1・2歳の金額差がある場合、`under3` を `age0` / `age1to2` に分割する必要がある。**中核市の収集開始前に判断すること**（後からのスキーマ変更は全ファイル修正になる）。

### 2.6 「ひとり親世帯等の軽減階層」

横浜（E0〜E5）・大阪（各階層内サブ行）・名古屋（軽減制度）にひとり親等の軽減がある。スキーマに対応フィールドがないため `tiers` には一般世帯の額を収録し、詳細を `bracketBasis.note` に原典表記のまま保存している。
→ **残タスク**: ひとり親はペルソナとして重要（[02_ツール一覧](../docs/02_ツール一覧.md) #59 児童扶養手当）。スキーマに `singleParentTiers` を追加するか、`tiers[].fees` に `singleParent` を足すかを検討する。

## 3. 収集手順（作業指示にそのまま貼る用）

```
リポジトリ C:\Users\kanet\20260522\wo-opus で作業。担当自治体: <自治体名×3>

## 必読
1. data/schema/hoikuryo.schema.json を Read してスキーマに厳密に従う
2. data/schema/seido-common.schema.json の source / valueNode / amendment 定義も Read
3. queue/hoikuryo-backlog.md の §2「実装上の論点」を Read（同じ罠を踏まないため）
4. 収集済みの近い自治体（例 data/seido/hoikuryo/kanagawa-yokohama.json）を Read して書式を揃える

## 出力先
data/seido/hoikuryo/<pref>-<city>.json

## 手順
1. WebSearch を allowed_domains で当該自治体の公式ドメインに絞り「保育料 利用者負担額 令和8年度」等で検索
2. WebFetch で実ページを開く。PDFでテキストが取れない場合は curl で取得し、
   python の PyMuPDF(fitz) で page.find_tables() / get_text('words') を使い
   表構造を座標から復元して読む（結合セルが多く、単純なテキスト抽出では列がずれる。
   pdftotext は日本語PDFでテキストランが混ざるため使わない）。
   スキャン画像PDFの場合は page.get_pixmap(dpi=260) で描画して読む。
   scratchpad ディレクトリを作業用に使う。
3. 階層表の全階層を漏れなく取得。3歳未満/3歳以上 × 保育標準時間/保育短時間 の4象限すべて
4. 境界は「以上/未満」を厳密に。incomeMax は「その値を含まない(未満)」で統一。
   原典が「〜円以下」なら +1 した値を入れ、原典表記を label に残す
5. bracketBasis の taxYearRule（4〜8月は前年度課税、9〜3月は当年度課税 等）と
   deductionsIgnored（住宅ローン控除・寄附金税額控除等を適用しない旨）を原典から必ず取る
6. §2.1 の所得割額の前処理（政令市の税率調整等）を bracketBasis.note に必ず記録する
7. scope.excludes に給食費(副食費)・延長保育料・実費徴収を必ず明記
8. 無償化している場合は freeTuition.localExtension に、多子カウントの年齢上限を
   freeTuition.multiChildPolicy.countingRule に記録

## 品質ルール
- 推測で数値を埋めない。未取得は note に「未確認: 理由」
- 令和8年度版が未公表なら fiscalYear: 2025 + amendments に status:"under-review"
- checkedAt / asOf は作業日。sources[].tier は "municipality"
- PDF出典は url に PDF、landingUrl に掲載元ページを両方入れる
- 検証: node scripts/verify-seido.mjs --only=<id> でエラー0を確認してから完了
- git 操作は一切しない。data/ 以外は触らない
```

**新しい自治体ドメインを追加したら**: `.lg.jp` / `.go.jp` 以外の独自ドメイン（例 `city.nagoya.jp`）は `scripts/verify-seido.mjs` の `ALLOWED_MUNICIPAL_DOMAINS` に追加しないと、一次情報でないと判定されて CI が落ちる。

## 5. 【保育料以外】住民税の非課税限度額（扶養の壁ツール向け）★2026-07-17 解決済み★

**→ `data/seido/juuminzei.json` を新設して決着。** 本節の宿題はすべて解消した。

### 決着の内容

**級地別の金額は、法令のどこにも数値として書かれていない。算式と率から導出されるものだった。** 前任者が「総務省の一次資料で確認できなかった」のは当然で、探し方（数値の literal 検索）が構造的に当たらなかった。

| 条文 | 内容 |
|---|---|
| 地方税法第295条第3項 | 条例委任（「政令で定める基準に従い当該市町村の条例で定める金額以下」） |
| 地方税法施行令第47条の3第1号 | 算式 = **基本額 ×（同一生計配偶者及び扶養親族の数 + 1）+ 10万円**（＋加算額） |
| 同第2号 | 基本額 = **35万円 × 級地別率** を「**参酌して**」定める。級地は「**生活保護法第8条第1項**の規定により厚生労働大臣が定める保護の基準における地域の級地区分」 |
| 同第3号 | 加算額 = **21万円 × 級地別率** を参酌 |
| **地方税法施行規則第9条の21第2項** | **1級地 1.0 ／ 2級地 0.9 ／ 3級地 0.8** ← 級地差の正体 |

単身に適用すると 35万×1.0+10万=**45万円**／35万×0.9+10万=**41.5万円**／35万×0.8+10万=**38万円** となり通説値と一致する。構成要素（35万・21万・10万・1.0/0.9/0.8）はすべて条文で照合済み。

### ★ただし「確定値を出さない」方針は維持する★

条文は「とする」ではなく「**参酌して定める**」であり、**市町村は条例で異なる額を定めうる**。したがって均等割の非課税判定を断定しない方針（[specs/s-tools/02](../specs/s-tools/02-fuyou-kabe-simu.md) §7、`fuyou-kabe.json` の `amount2026: null`）は**正しく、維持すべき**。

### 派生して確定した事項

- **所得割の非課税限度額は級地に依存せず全国一律45万円**（地方税法附則第3条の3）→ **保育料のC階層（所得割非課税）判定は自治体差なしで確定できる**
- **住民税の基礎控除は43万円のまま**（e-Gov `law_revisions` で令和9年1月1日施行版・令和10年1月1日施行版のいずれも「四十三万円」を確認。§5 旧記載の「要確認」を解消）
- **住民税の給与所得控除の最低保障74万円は令和9年度分・令和10年度分**（財務省大綱の原文で裏取り）。★**保育料の令和8年度課税＝令和7年分所得は65万円**であって74万円ではない★
