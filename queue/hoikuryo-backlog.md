- **`tests/hoikuryo.test.ts` の件数**：ハードコード廃止。「data/seido/hoikuryo/*.json と municipalities が過不足なく1対1」「全ファイルが妥当な構造」を**データ由来で動的検査**。件数を書き換える必要はない。
- **`app/lib/tools/registry.json` の hoikuryo sources**：**編集不要**。出典は各自治体の JSON（`sources`）に書き、ツールは選択自治体の出典を動的表示する。registry の sources は代表出典として凍結（増やさない）。
- **この backlog の §1 の表**：**更新不要**（履歴スナップショット）。件数・残数の書き換えも不要。

→ これにより municipalities配列・テスト件数・backlog表・registry の**3〜4-way コンフリクトが消滅**する（並行収集で「自己解消した」報告が不要になる）。新しい流儀・罠の共有だけ §2 の末尾に追記（末尾追加はコンフリクトしにくい）。**§1 以下の表は歴史記録として残すが、以後は編集しない。**

## 0. このキューの使い方

1 自治体 = 1 タスク。相互依存なしのため**全並列で実行可能**。1 エージェント 3 自治体程度が実績値（所要 8〜13 分／3自治体）。
着手時は §3 の収集手順をそのまま作業指示に貼る。**§1 の表は更新不要**（上記の周知参照。data/seido/hoikuryo/ に JSON を置けば自動反映）。

**絶対ルール**
- 一次情報（当該自治体の公式サイト）のみ。まとめサイト・保活情報サイト・民間ブログの引用は**禁止**（`scripts/verify-seido.mjs` がドメイン許可リストで機械的に弾く）。
- **推測で数値を埋めない。** 取得できなければ `note` に「未確認: 理由」と明記する。階層を捏造しない。
- 令和8年度版が未公表なら `fiscalYear: 2025` とし `amendments` に `status: "under-review"` で明記する。**令和8年度と偽らない。**

### 2.1 「所得割額をそのまま入力させてはいけない」

階層判定に使う所得割額の前処理が**自治体ごとに違う**。ユーザーが課税明細書の数字をそのまま入れると誤判定する。

| 自治体 | 前処理 |
|---|---|
| 大阪市 | 税額控除前所得割額 **× 6/8**（政令市の税率調整） |
| 名古屋市 | **税源移譲前の税率**で算定した額（※他市の資料では「名古屋市のみ5.7%」と説明される） |
| 横浜市 | 調整控除額・所得割の調整措置の額**のみ**を控除した額 |
| 札幌市・福岡市・川崎市 | 政令市のため**旧税率6%相当**に換算 |
| 相模原市・京都市・熊本市・広島市 | **6/8を乗じる**旨と乗率が原典に明記されている（保護者向けの換算手順つき） |
| さいたま市・千葉市・堺市・北九州市・神戸市・岡山市・静岡市 | 「6%で算定する」旨の規定はあるが**乗率・端数処理の明文がない**（＝6/8と決め打ちできない。各ファイルに「未確認」と記録） |
| 新潟市・静岡市 | **階層表が「8%課税額」列と「6%換算額」列の2列併記**。データは6%列を採用し、8%列は label に併記 |
| 浜松市 | **6→8%の税率調整の規定が公式サイト・例規のどこにも見当たらない**（未確認。境界が6%前提か8%そのままかも不明） |
| 東京都北区 | 特別区だが前処理あり。調整控除のみ差引き、かつ**政令市在住者は課税標準額に区民税6%を乗じて再計算** |
| 東京都荒川区 | 算式が `父＋母＋各種税額控除` と控除額を**足し戻す**形（課税明細の額をそのまま入れると**過小**判定） |
| 広島市 | **16歳未満の扶養親族が3人以上いる世帯は、2人を超える1人につき22,800円を世帯の所得割合算額から控除**。★所得割額の前処理が世帯構成に依存する初の事例★ |

また、**適用しない控除**も自治体差がある（住宅借入金等特別税額控除・寄附金税額控除・配当控除が典型）。
**定額減税の向き**は自治体で割れる。**適用後**（＝減税を反映した額で算定）は川崎市・さいたま市・京都市・港区・文京区・台東区。札幌市・大阪市等は適用前。**必ず原典を見て判断し、近隣自治体からコピーしないこと。**

→ **設計判断**: 所得割額の直接入力は上級者向けオプションに留め、既定は年収・家族構成からの推計とする。推計値である旨を必ず明示する。

### 2.2 「A/B/C階層は所得割額では判別できない」

生活保護世帯（A）・住民税非課税世帯（B）・所得割非課税世帯（C）はいずれも所得割額0円のため、金額だけでは区別できない。`isWelfare` / `isNonTaxable` フラグと均等割の課税有無で分岐する必要がある。
収集済みデータでは自治体により `incomeMax` の表現が割れている（札幌C1は均等割のみ世帯を含み `incomeMax: 48600`、福岡C1・川崎C1は `incomeMin: 0, incomeMax: 1`）。**この不統一はスキーマ側で吸収せず、ツール側で `isNonTaxable` / `isWelfare` を先に見る実装にすること。**

### 2.3 「無償化により階層表が消えつつある」

東京23区は第1子まで無償化され、練馬区は階層表そのものを廃止（ページは404）。大阪市も令和8年9月から全員無償化。
→ **設計判断**: 「階層表がある」前提の設計にしない。`tiers` が1行のみの自治体、全額0円の自治体を正常系として扱う。無償の自治体では「保育料は0円です。かかるのは給食費・延長保育料です」と表示し、**給食費・延長保育料の案内に価値をシフトする**（世田谷・練馬は給食費も無償という差別化要素がある）。

### 2.4 「施設種別の軸がスキーマにない」

川崎市は施設種別ごとに料金列が3系統（認可保育所等／小規模B型等／家庭的保育等）ある。現スキーマには施設種別の軸がないため**認可保育所列のみ収録**し、`scope.excludes` に明記している。
→ **残タスク**: 小規模保育・家庭的保育の料金差が大きい自治体が増えるなら、スキーマに `facilityType` 軸を追加するか検討する。MVP では認可保育所のみで割り切る。

### 2.5 「0歳児と1・2歳児で金額が違う自治体がある」

大田区・杉並区は原典が0歳児クラスと1・2歳児クラスを別区分にしている。現スキーマの `under3` は1区分のため統合している（現在いずれも0円のため情報欠落なし）。
→ **残タスク**: 有料の自治体で0歳／1・2歳の金額差がある場合、`under3` を `age0` / `age1to2` に分割する必要がある。**中核市の収集開始前に判断すること**（後からのスキーマ変更は全ファイル修正になる）。

### 2.7 「保育利用時間で金額が変わる自治体がある」★2026-07-17 京都市で判明 → 同日スキーマ拡張で根本解決済み★

京都市は**保育標準時間認定の中がさらに保育利用時間（8.5／9／9.5／10／10.5／11時間）で6段階**に分かれ、金額が違う
（最上位階層で 8.5時間 80,300円 に対し 11時間 94,400円＝**最大14,100円/月の差**）。

**→ 2026-07-17 解決。** スキーマに保育利用時間の軸を追加した:

- **スキーマ**: トップレベル `timeBands`（`bands[]`/`defaultBandKey`/`rule`/`sourceId`/`checkedAt`）＋ `tiers[].fees.<age>.standardByBand`（バンドキー→月額）。`fees.standard` には既定バンド（＝最長時間）の額を重複収録する（後方互換）。単一バンドの自治体は `timeBands` 自体を持たない＝影響なし。
- **整合検査**: `scripts/verify-seido.mjs` がバンドキーの一致・全バンドの網羅・`standard`＝既定バンド額を機械検査する（label への押し込み方式の再発防止）。
- **ツール**: `feeOf(tier, age, need, timeBandKey)` でバンド解決。UIは `timeBands` のある自治体×保育標準時間のみ「保育利用時間」の選択を表示（既定は最長時間＝過小表示しない安全側）。テストは `tests/hoikuryo.test.ts` の「★京都市★ 保育利用時間バンド」。
- **今後の収集**: 保育利用時間で金額が分かれる自治体は、京都市方式（`kyoto-kyoto.json`）に倣って収録すること。label への金額列挙は不要。

§2.4 の `facilityType` 軸・§2.5 の 0歳／1・2歳分割は引き続き未解決（中核市の収集開始前に判断）。
UI 側の分岐追加が前提になる。

### 2.8 「2歳児クラスのみ別扱い（青森市＝2歳児以上を独自無償化）」★2026-07-18 スキーマ拡張で解消・後方互換★

★共有ファイル改修の周知（wo-opus 主導）★ 青森市はR8から**2歳児クラス以上を独自無償化し、0・1歳児クラスのみ課税**する。
従来の `under3`（0〜2歳一律）では表現できず収集がブロックされていた。**後方互換なスキーマ拡張で解消済み**（`feat/hoikuryo-age2-class`）:

- **スキーマ**: `ageClasses[].key` に `age2` を追加、`tiers[].fees` に任意の `age2`、`freeTuition.age2Free`（valueNode）を追加。
- **型/ロジック**（`lib/tools/impl/hoikuryo.ts`）: `AgeKey` に `age2`、`isAge2Free(m)`、`FeeBasis` に `age2Free`、`calc` に age2 無償分岐（age3plusFree と同流儀）。
- **UI**（`Hoikuryo.tsx`）: 年齢選択肢は `m.ageClasses` 由来のためデータ側で `age2` を足せば自動で選択肢に出る。結果メッセージに age2Free 分岐を1行追加。
- **後方互換の実証**: 既存69自治体は `age2` を持たず `isAge2Free=false`＝従来どおり `under3` が0〜2歳を一律に扱う。tests/hoikuryo.test.ts の「★年齢クラス拡張 age2★」で「既存69で age2 は未使用・feeOf age2=null・basis≠age2Free」を全数固定。verify-seido 構造検査エラー0。

**★青森市の収集は wo（東日本担当）に引き継ぐ★**（データは wo が収集）。収集時の書き方:
1. `ageClasses` を `[{key:"under3", label:"0・1歳児クラス", ...}, {key:"age2", label:"2歳児クラス", ...}, {key:"age3plus", label:"3〜5歳児クラス", ...}]` にする（under3 のラベルを「0・1歳児クラス」にする点に注意）。
2. `tiers[].fees.under3` には**0・1歳児クラスの課税額のみ**を収める（2歳を含めない）。`fees.age2` は無くてよい（無償のため）。
3. `freeTuition.age2Free` を `{value:true, label:"2歳児クラスは無償です（青森市の独自無償化）", sourceId, checkedAt, verify:{expect:[...]}}` で設定。age3plusFree と同じ流儀。
4. 2歳が**課税**の自治体が将来出た場合は `fees.age2` に階層別額を入れる（age2Free は設定しない）。

### 2.6 「ひとり親世帯等の軽減階層」

横浜（E0〜E5）・大阪（各階層内サブ行）・名古屋（軽減制度）にひとり親等の軽減がある。スキーマに対応フィールドがないため `tiers` には一般世帯の額を収録し、詳細を `bracketBasis.note` に原典表記のまま保存している。
→ **残タスク**: ひとり親はペルソナとして重要（[02_ツール一覧](../docs/02_ツール一覧.md) #59 児童扶養手当）。スキーマに `singleParentTiers` を追加するか、`tiers[].fees` に `singleParent` を足すかを検討する。

## 5. 【保育料以外】住民税の非課税限度額（扶養の壁ツール向け）★2026-07-17 解決済み★

**→ `data/seido/juuminzei.json` を新設して決着。** 本節の宿題はすべて解消した。

### 決着の内容

**級地別の金額は、法令のどこにも数値として書かれていない。算式と率から導出されるものだった。** 前任者が「総務省の一次資料で確認できなかった」のは当然で、探し方（数値の literal 検索）が構造的に当たらなかった。

| 条文 | 内容 |
|---|---|
| 地方税法第295条第3項 | 条例委任（「政令で定める基準に従い当該市町村の条例で定める金額以下」） |
| 地方税法施行令第47条の3第1号 | 算式 = **基本額 ×（同一生計配偶者及び扶養親族の数 + 1）+ 10万円**（＋加算額） |
| 同第2号 | 基本額 = **35万円 × 級地別率** を「**参酌して**」定める。級地は「**生活保護法第8条第1項**の規定により厚生労働大臣が定める保護の基準における地域の級地区分」 |
| 同第3号 | 加算額 = **21万円 × 級地別率** を参酌 |
| **地方税法施行規則第9条の21第2項** | **1級地 1.0 ／ 2級地 0.9 ／ 3級地 0.8** ← 級地差の正体 |

単身に適用すると 35万×1.0+10万=**45万円**／35万×0.9+10万=**41.5万円**／35万×0.8+10万=**38万円** となり通説値と一致する。構成要素（35万・21万・10万・1.0/0.9/0.8）はすべて条文で照合済み。

### ★ただし「確定値を出さない」方針は維持する★

条文は「とする」ではなく「**参酌して定める**」であり、**市町村は条例で異なる額を定めうる**。したがって均等割の非課税判定を断定しない方針（[specs/s-tools/02](../specs/s-tools/02-fuyou-kabe-simu.md) §7、`fuyou-kabe.json` の `amount2026: null`）は**正しく、維持すべき**。

### 派生して確定した事項

- **所得割の非課税限度額は級地に依存せず全国一律45万円**（地方税法附則第3条の3）→ **保育料のC階層（所得割非課税）判定は自治体差なしで確定できる**
- **住民税の基礎控除は43万円のまま**（e-Gov `law_revisions` で令和9年1月1日施行版・令和10年1月1日施行版のいずれも「四十三万円」を確認。§5 旧記載の「要確認」を解消）
