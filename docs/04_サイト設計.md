### 第一候補の決定理由（社長承認待ち）

**くらしのラクダ（kurashi-rakuda.jp）**
1. ブランドの約束「あなたの荷物（生活の負担）を代わりに背負う」を動物一体で表現でき、完全無料・広告臭ゼロの世界観と合う
2. 指名検索されやすい固有名詞（「暮らし 便利」系の一般名詞サイト名は指名検索が育たない）
3. .jp / .com とも空き。商標面でも「くらしのラクダ」完全一致の既存サービスは確認されず（出願前に商標調査は別途実施のこと）

### ドメイン費用（2026年7月調査）

- .jp: 年額約2,000〜3,900円（Xserverドメイン 1,980円/年が表示額=請求額で最安級。お名前.com は「サービス維持調整費」上乗せで実質約3,900円/年）
- .com: 年額約1,500〜1,800円（Cloudflare Registrar US$10.46 が卸値最安。ただし .jp は取扱なし）
- **推奨**: kurashi-rakuda.jp を Xserverドメインで取得（初回0円キャンペーン対象なら0円、更新1,980円/年）。ブランド保護で kurashi-rakuda.com も Cloudflare で押さえる（合計 年額約3,600円）

## 2. マスコット案

- **名前案**: 「ラクダさん」または「らっく」（公募/SNS投票は Phase 3 で検討）
- キャラクター設定: ユーザーの「生活の荷物」をコブに載せて代わりに運ぶラクダ。急かさない・売りつけない・いつでも無料で手伝う、が性格設定（＝ブランド哲学の擬人化）
- 展開: anzen-ai のマスコット全サイト展開ノウハウを転用。ツールの結果画面でひとこと補足（「この金額は2026年度の税率で計算したよ」）、404ページ、ローディング、SNS アイコン
- トーン: 癒し系・脱力系。煽り文言との構造的相性が悪いキャラにすることで、広告臭ゼロを内側からも担保する

## 3. 情報アーキテクチャ（IA）

### 3.1 URL 設計

```
/                       … トップ（検索窓＋ライフステージ導線＋よく使われるツール）
/tools/                 … ツール一覧（全ツール横断・絞り込み）
/tools/[category]/[slug]  … 各ツール（例: /tools/money/ikukyu-kyufu-sim）
/guide/[category]/[slug]  … 記事（制度解説/ツール併走/段取り型）
/stage/[stage]/           … ライフステージ別ハブ（下記6ステージ）
/week/[n]  /month/[n]     … 妊娠週数別・月齢別（プログラマティックSEO）
/about /policy /sources /contact … 運営者情報・紹介ポリシー・出典一覧・問い合わせ
```

- カテゴリ7種（[02_ツール一覧](02_ツール一覧.md) と1:1）: `pregnancy` 妊娠出産 / `childcare` 子育て / `kaji` 家事・料理 / `money` お金 / `health` 健康・美容 / `career` 仕事・キャリア / `care` 介護
- ライフステージ6種: 妊活 `pre-pregnancy` / 妊娠中 `pregnant` / 育児 `raising` / 復職・仕事 `working` / 介護 `caring` / シングル・自分の暮らし `solo`
- **カテゴリ（=悩みの種類）とステージ（=人生の時期）の2軸ナビ**が本サイトの IA の核。競合は年齢/月齢の1軸のみが多く、ダブルケア（育児×介護）のような複合状況を拾えない

### 3.2 回遊設計（最小限）

- ツール結果画面下に「同じ悩みの人が次に使うツール」を最大3件だけ表示（solves メタデータから自動生成）
- それ以外の回遊装置（人気ランキング常設・無限スクロール・レコメンド枠）は置かない。「最短で解決して立ち去ってもらう」が善（[00_企画書](00_企画書.md) §6）

## 4. デザイン方針 —「広告臭ゼロ」を体現する UI 原則

### 4.1 UI 十原則

1. **ファーストビューに答え**: ツールは入力フォームがファーストビュー内。記事は結論を最初の段落に
2. **広告枠が存在しないレイアウト**: サイドバー広告・記事内広告・追従バナーの「枠」自体を設計に持たない
3. **ポップアップ全面禁止**: 通知許可・アプリ誘導・メルマガ登録のモーダルを一切出さない
4. **登録不要**: 全機能がログインなし。保存系は localStorage
5. **タップ領域48px以上・入力は最少ステップ**: 片手抱っこ・通勤電車で使える（ペルソナ1・2）
6. **数字は大きく、根拠は必ず添える**: 結果金額の直下に計算式と準拠年度・出典（信頼の可視化）
7. **文字サイズ16px以上・平易文**: ペルソナ5（52歳）が眼鏡なしで読める
8. **色は落ち着いた低彩度基調**: 砂色（ラクダ）×深緑系を基調に、強調色は結果表示のみに使用。セール感のある赤・黄の多用禁止
9. **煽り UI 禁止**: カウントダウン、点滅、バッジ「HOT」「急上昇」等を使わない
10. **オフライン耐性**: 一度開いたツールは通信不安定でも動く（クライアント計算の恩恵。地下鉄・授乳中の暗い部屋で使える）

### 4.2 品質基準

- Lighthouse Performance / Accessibility / Best Practices / SEO すべて 90 以上（CI で強制。[05_技術構成](05_技術構成.md) §8）
- WCAG 2.1 AA 相当のコントラスト・フォーカスリング
- ダーク対応: `prefers-color-scheme` 準拠（深夜授乳・深夜検索のペルソナ4対応）

## 5. トップページ構成（ワイヤ概要）

```
[ロゴ: くらしのラクダ]  [サイト内検索窓（最重要・最大面積）]
[ライフステージ6タイル: 妊活/妊娠中/育児/復職・仕事/介護/自分の暮らし]
[カテゴリ7導線]
[「今日をラクにする」: 季節・時期文脈のツール3件（例: 7月=夏の電気代、児童手当現況届）]
[新着・更新されたツール（改定追随の証明として更新日を見せる）]
[フッター: 運営者情報/編集ポリシー/紹介ポリシー/出典一覧]
```

- 「今日をラクにする」枠は編集部おすすめ＝広告枠に見えないよう、選定理由（時期文脈）を必ず添える
