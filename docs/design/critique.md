# UI格上げレビュー（2026-07-18）

## 対象と方法

本番 `https://kurashi-rakuda.vercel.app` をデスクトップ幅1440pxで実撮影し、トップ、ツール一覧、ツール詳細、記事、ガイド一覧、固定ページを比較した。撮影原本は `docs/design/shots/production-before-*.png` に保存した。

## 率直な現状診断

- 第一印象が「よく整った個人制作」に留まる。情報は正確だが、ブランド固有の編集思想や視覚資産が一画面目から伝わらない。
- 角丸カード、1px罫線、淡いベージュ背景がほぼ全セクションで反復され、重要度や用途が違っても同じ部品に見える。長いページほど「カードの棚」になる。
- 見出し、本文、補足のサイズ差が小さく、和文の太さもほぼ二段階しかない。ページを遠目に見たときの骨格が弱い。
- 最大幅、2列グリッド、カード間隔は整っているが、画面幅に対して安全側へ寄りすぎている。大胆な余白、全幅の色面、左右非対称、編集的な切り替えがない。
- ヘッダーのマスコット＋文字は仮ロゴに見える。字間、ベースライン、マークと名称の比率に設計感がなく、企業サイトの「署名」になっていない。
- カテゴリは記号と色だけで、妊娠、育児、家事、お金などの世界観を視覚的に識別できない。記号体系も○・◇・⌂・¥・+・□・∞で出自が揃っていない。
- ヒーローのらっくは可愛いが、単体立ち絵を置いただけで「暮らしの荷物を軽くする」という物語がない。背景もUIグラデーションに留まり、キービジュアルとして弱い。
- トップの信頼情報は4等分の小箱で、数字の説得力よりダッシュボード感が先に立つ。「根拠がある」ことを編集表現で見せ切れていない。
- ツール一覧は64件すべてが同じ密度で続き、探す体験より在庫表に近い。カテゴリの導入、余白、特集的な抜き、視覚的ランドマークが不足している。
- 記事は読みやすいが、制度記事、段取り記事、ツール併走記事の差がヒーロー色に偏る。本文に引用、数字、要点、関連行動の編集パターンが少ない。
- 固定ページは誠実だが「規約文を綺麗な箱へ入れた」印象。運営姿勢を象徴する数字、工程、署名、更新責任の見せ方が弱い。
- フッターはリンク一列と一文だけで終わり、サイト全体像、カテゴリ、信頼情報、運営情報を最後に再構成する役割を果たしていない。
- モバイルLCP対策でヒーローマスコットを非表示にしたため、最重要接点でブランド資産が消えている。性能とブランドがトレードオフになっている。

## 大手コンシューマ向けサイトから抽出した原則

調査対象は、任天堂の企業情報、サントリー公式、クックパッド企業サイト、無印良品ネットストア。固有の構図、文言、素材は転用せず、以下の設計原則だけを採用する。

- セクション構成には「静かな導入→大きな物語→機能的な一覧→信頼の証拠→次の行動」という密度差を作る。同じカードグリッドを連続させない。
- 一画面内で主見出し、説明、ラベル、補足の差を明確にし、見出しは文字サイズだけでなく行長、改行位置、余白で階層化する。
- グリッドは揃えるが、すべてを同じ列数にしない。全幅の色面、片側に余白を残す非対称構成、数字の帯、横スクロールなど用途に応じてリズムを変える。
- 写真やイラストは装飾ではなくセクションの意味を担わせる。少数でも、キービジュアルとカテゴリ資産の品質・統一感を優先する。
- アイコンは線幅、角、塗り、視覚サイズを統一し、文字記号の寄せ集めにしない。
- フッターはサイトマップ、企業姿勢、問い合わせ、規約、カテゴリを再編集した「最後の案内所」として設計する。
- モバイルはデスクトップの縮小版にしない。主要ナビを親指で操作できる位置・大きさにし、横断導線は横スクロール、本文は一列、装飾は軽量なまま残す。
- CTAは数を増やさず、文脈の終点に一つ置く。色で煽らず、余白、矢印、動詞で次の行動を明確にする。
- 信頼は「安心」「高品質」と自称せず、出典、更新日、検査工程、広告ゼロなど検証可能な事実を大きく見せる。

参考: [任天堂 会社情報](https://www.nintendo.co.jp/corporate/index.html)、[サントリー公式](https://www.suntory.co.jp/)、[クックパッド株式会社](https://info.cookpad.com/)、[無印良品ネットストア](https://www.muji.com/jp/ja)

## 採用する設計方針

- ブランド署名を独立したSVGワードマークへ統一する。
- 深緑を信頼、砂色を生活、珊瑚色を人の温度、青緑を情報整理の役割へ分ける。
- トップは「物語のあるヒーロー」「編集された入口」「数字で見る信頼」「全件一覧」の順に密度を変える。
- 64件のSSRは維持しつつ、画面外一覧は既存の `content-visibility` 最適化を継続する。
- モバイルでもらっくを表示し、専用の軽量画像と正確な `sizes` でLCPを守る。
