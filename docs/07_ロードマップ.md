# 07_ロードマップ — くらしのラクダ（仮称）

作成日: 2026-07-17 ／ 原則: 各 Phase の完了条件は PV・収益ではなく「使えるツール数」「解決できる悩み数」で定義する。

「解決できる悩み数」の定義: 「◯◯がわからない/決められない/計算できない」という具体的な悩み1種に対し、ツールまたは記事が直接の答えを返せる状態を1とカウントする（各ツール・記事のフロントマターに `solves:` として明記し、機械集計する）。

---

## Phase 0: 企画・基盤（本フェーズ）

- 内容: 企画ドキュメント確定、サイト名・ドメイン決定、リポジトリ作成
- 完了条件:
  - [x] docs/00〜08 + BACKLOG.md が揃っている
  - [ ] 社長が決定事項（サイト名・ドメイン・優先ツールTOP10・MVP範囲）を承認
  - [ ] ドメイン取得

## Phase 1: MVP —「妊娠・子育て・お金」の中核20ツール

- 期間目安: 開始から6〜8週
- 内容:
  - 優先度 S のツール20個（[02_ツール一覧](02_ツール一覧.md) の S ランク）を実装
  - サイト基盤: トップ、カテゴリページ、ツールテンプレート、検索（Pagefind）、運営者情報・編集ポリシー・紹介ポリシー公開
  - 制度解説記事 60 本（記事工場転用の初回バッチ）
  - GA4/GSC 接続、VPS 自律改善エンジン接続
- 完了条件（すべて数えられるもの）:
  - 使えるツール: **20個**（全ツール: テスト100%・出典明記・スマホ実機確認済み）
  - 解決できる悩み: **100種以上**（solves 集計）
  - 制度解説記事: 60本（CI 忠実性ゲート通過済みのみ）
  - 全ページ Lighthouse Performance/Accessibility 90 以上
  - 紹介リンク数: **0 でもよい**（無理に置かない。置く場合は全件チェックリスト通過）

## Phase 2: 拡張 — ライフステージ全域カバー

- 期間目安: Phase 1 完了から12週
- 内容:
  - ツールを 60 個まで拡張（妊活・仕事キャリア・介護・健康カテゴリを本格追加）
  - プログラマティック SEO 第1弾（週数別43p・月齢別37p・保育料 政令市/中核市62p）
  - Web Push（予防接種・制度改定のリマインダー）
  - 保存系機能（localStorage: よく使うツール・入力値の記憶）
  - kurashi-keisan / anzen-ai との相互送客リンク稼働
- 完了条件:
  - 使えるツール: **60個**
  - 解決できる悩み: **300種以上**
  - プログラマティックページ 140p 以上がインデックス済み（GSC 確認）
  - リマインダー機能が3系統（予防接種/妊婦健診/制度改定）動作

## Phase 3: SNS 連携・自動化 —「勝手に育つサイト」化

- 期間目安: Phase 2 完了から12週
- 内容:
  - ツールを 100 個以上へ（[02_ツール一覧](02_ツール一覧.md) 全カテゴリ完遂）
  - instagram/note 自動運用に本サイトアカウントを追加（ツール紹介の自動投稿、PDCA エンジン接続）
  - GSC クエリ→新ツール候補の自動起票（自律改善エンジン拡張）
  - 保育料の全国自治体展開（段階リリース）
  - RAG 相談機能（kakomon-ai 転用）の費用対効果審査 → 採否決定
- 完了条件:
  - 使えるツール: **100個以上**
  - 解決できる悩み: **500種以上**
  - SNS 自動投稿が週次で無人稼働、投稿→サイトの導線が機能している
  - 制度改定の自動追随（改定チェック日超過の自動検知→更新）が1サイクル実績を持つ

## Phase 4 以降（構想のみ・未計画）

- ダブルケア（子育て×介護）特集領域の深掘り、自治体データの網羅、多言語化（在日外国人ママ）
- 着手判断は Phase 3 完了時に「ユーザーの声」の蓄積を見て行う

## 撤退・軌道修正の基準

- 各 Phase 終了時にブランド哲学セルフ監査を実施: 「これで稼いでるんだな」と見えるページが1つでもあれば、新規開発を止めてでも先に直す
- ツールの利用が極端に少ない場合も削除はせず改善（完全無料サイトの在庫は負債ではなく資産。ただし制度改定に追随できないものは非公開にする）
