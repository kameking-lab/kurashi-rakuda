# 12_パーソナライズ設計 — 対象属性メタと「並べ替え・ハイライト」基盤

任意プロフィールに基づいて関連ツール・記事を上位表示する機能の**土台（データ構造）**を定義する。
UI 実装は Codex が担当する。本書はデータ層の契約書であり、Codex はこの契約に沿って
並べ替え・ハイライトの見た目を作る。

## 0. 絶対原則（SEO を殺さない）

1. **パーソナライズは「並べ替え・ハイライト」であって「非表示」ではない。**
   全ツール/記事は audience の値に関わらず、常に URL 不変で存在し、一覧にも掲載され、
   クロール可能でなければならない。audience を根拠にページやリンクを DOM から消してはならない。
2. **プロフィール未設定・Googlebot には全件を既定順で表示する。**
   並べ替えは「プロフィールがあるときだけ」効く上乗せ。既定（未設定）挙動は現状と同一。
3. **プロフィール情報は localStorage 完結・サーバ送信ゼロ・クッキーレス。**
   「この端末だけに保存、誰にも送られない」を UI に明記できる設計にする（プライバシーポリシー第3条の維持）。
   SSR/SSG が出力する HTML はプロフィール非依存（全ユーザー共通）で、並べ替えはクライアントで後付けする。
4. **audience はページの内容・出典・meta・JSON-LD を一切変えない。** 表示順・強調の手がかりにのみ使う。

## 1. スキーマ（`app/lib/audience.ts` が単一ソース）

各ツール（`app/lib/tools/registry.json`）と各記事（`content/articles/*.md` の frontmatter）に
`audience` オブジェクトを持たせる。

```ts
interface Audience {
  universal: boolean;        // 属性非依存＝誰にでも関係（調味料換算・炊飯水など）
  lifeStages: LifeStage[];   // 関係するライフステージ（複数可）
  lifeEvents: LifeEvent[];   // 関係するライフイベント（複数可）
  childAgeBands: ChildAgeBand[]; // 子ども関連の対象年齢帯（複数可）
  gender: "female" | "male" | null;  // 性別依存のみ設定。null=不問（既定）
}
```

### 語彙（controlled vocabulary）

- **LifeStage**: `pregnancy`（妊娠中）/ `newborn`（新生児・0〜1歳ごろ）/ `infant`（乳児）/
  `toddler`（幼児・1〜6歳ごろ）/ `schoolAge`（学童）/ `adolescent`（思春期）/ `adult`（成人）/ `senior`（シニア）
- **LifeEvent**: `pregnant`（妊娠・妊活中）/ `parenting`（子育て中）/ `caregiving`（介護中）/ `working`（就労中）
- **ChildAgeBand**: `prenatal`（胎児）/ `age0_1` / `age1_3` / `age3_6` / `age6_12` / `age12_18`
- **Gender**: `female` / `male` / `null`

### 整合ルール（`validateAudience` と CI が強制）

- `universal:true` のとき `lifeStages`/`lifeEvents`/`childAgeBands` は空・`gender` は `null`。
- `universal:false` のとき `lifeStages` か `lifeEvents` を最低1つ持つ。
- `childAgeBands` に `prenatal` があれば `lifeEvents` に `pregnant` を含む。
- `childAgeBands` に `prenatal` 以外があれば `lifeEvents` に `parenting` を含む。
- 各配列は既知の語彙のみ・重複なし。

### 設計判断（なぜこの粒度か）

- **hasChildren は導出**: `lifeEvents` に `parenting` があるか `childAgeBands` が空でなければ「子がいる人向け」。
  重複フィールドを持たない。
- **ageRange（本人の年齢帯）は lifeStages の `adult`/`senior` で代替**。数値レンジは持たない（過度な精密化を避ける）。
- **広めに含める**: 迷ったら関係しうる属性を足す。パーソナライズは「上位表示」であり、
  取りこぼしても全件は見えるので、狭めるより広める方が安全。

## 2. 付与ルール（assignment）

- **ツール（59件）**: 各ツールの性質から手作業でマッピング（`registry.json` に直書き）。
  例: 離乳食の量 → `infant` / `parenting` / `age0_1,age1_3`。扶養の壁 → `adult` / `working`。
  介護自己負担 → `adult,senior` / `caregiving`。調味料換算・炊飯水・レンジワット・冷凍保存・
  作り置き・洗濯表示・レシピ人数・味付け黄金比・献立・買い物リスト → `universal`。
- **記事（97件）**:
  - 併走型（tool-narisou）は併走ツールの audience を継承する（1ツール1記事の対応が正）。
  - 制度解説・段取り型はカテゴリ既定 + タイトル/要約/solves のキーワードで属性を union（広めに）。
- **新規ツール/記事を追加する人へ**: `audience` は必須。CI（`check:audience`）が欠落・不正を
  ブロックする。上記語彙と整合ルールに沿って手で付ける（registry.json はビルド時に型検査されないため CI が唯一の砦）。

## 3. カテゴリ再編案（タスク2）

現行の7カテゴリ（`TOOL_CATEGORIES`）は URL 構造（`/tools/{category}/...`）と 1:1 で、
**SEO・被リンク・sitemap の安定のため変更しない**（カテゴリ URL は不変を保証）。
再編は「URL を変える」のではなく「**横断の見せ方（軸）を足す**」で実現する。

### 3.1 追加する軸：ライフステージ軸

カテゴリ（テーマ軸）に直交する「ライフステージ軸」を UI のナビゲーションとして追加する。
ツール/記事は `audience.lifeStages` によって複数のステージに同時に属せる（多対多）。

| ライフステージ軸（表示ラベル） | 対応 lifeStages | 主に集まるツール例 |
| --- | --- | --- |
| 妊活・妊娠 | pregnancy | 出産予定日・妊婦健診・戌の日・不妊治療 |
| 産後・0〜1歳 | newborn, infant | 授乳ミルク量・月齢・産後手続き・保活 |
| 乳幼児（1〜6歳） | infant, toddler | 離乳食・保育料・無償化・予防接種 |
| 学童（小学生） | schoolAge | 小1の壁・学童・児童手当・身長予測 |
| 働く・お金 | adult（＋working） | 扶養の壁・時短給料・復職手取り・パート収入 |
| 介護 | senior（＋caregiving） | 介護自己負担・要介護認定・施設費用 |
| 暮らし全般（万人向け） | universal | 調味料換算・炊飯水・洗濯表示・献立 |

- ライフステージ軸は `/tools`・`/guide` 内の**表示上のグルーピング／絞り込みタブ**として提供する
  （URL を新設する場合も、既存の `/tools/{category}/{slug}` を正 canonical に保ち、軸ページは
  補助導線＝内部リンク集約として設計する。孤立ページを生まない）。
- 実装分担: 分類データ（`audience`）は本 PR で用意済み。軸の UI・ルーティングは Codex が担当。

### 3.2 プロフィール項目（localStorage）

Codex が作る設定 UI で保存する想定のプロフィール（すべて任意・未設定可）:

```ts
interface Profile {
  lifeStages?: LifeStage[];   // いま自分/子が該当する段階
  lifeEvents?: LifeEvent[];   // いまの状況（妊娠中/子育て中/介護中/就労中）
  childAgeBands?: ChildAgeBand[]; // 子の年齢帯
  gender?: "female" | "male"; // 任意
}
```

## 4. 並べ替え・ハイライトの契約（ロジックの土台）

土台の純関数は `lib/personalize.ts`（実装済み。単体テスト `tests/personalize.test.ts`）に置く。契約:

- `scoreAudience(profile, audience): number` — プロフィールと audience の一致度スコア。
  `universal` は常に基礎点（誰にでも出る）。lifeStages/lifeEvents/childAgeBands/gender の一致で加点。
- `sortByAudience(items, profile): T[]` — **安定ソート**でスコア降順に並べ替える。
  **要素を1つも落とさない**（全件を返す。並べ替えのみ）。profile 未設定なら入力順のまま返す。
- `isHighlighted(profile, audience): boolean` — 強調表示（バッジ等）の判定。UI は Codex。

これらは純関数・副作用なし・DOM 非依存。SSR は既定順（未設定）で出力し、
クライアントで `Profile` を localStorage から読んで並べ替え・強調を後付けする（SEO 用 HTML は不変）。

## 5. CI（メタ検証）

- `npm run check:audience`（`.github/scripts/check-audience.mjs`）: 全ツール・全記事が
  妥当な `audience` を持つことを検証（`package.json` の `check` と `ci.yml` に組込済み）。
- `tests/audience.test.ts`: 本体の `validateAudience` で全データを網羅検証（.mjs と型側の乖離防止）。
